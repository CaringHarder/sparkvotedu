  <name>Task 1: Add account management functions to admin DAL and create server actions</name>
  <files>
    src/lib/dal/admin.ts
    src/actions/admin.ts
    prisma/schema.prisma
  </files>
  <action>
1. In `prisma/schema.prisma`, add a `deactivatedAt` field to the Teacher model:
   ```
   deactivatedAt  DateTime?  @map("deactivated_at")
   ```
   Place it after the `role` field. A non-null value means the account is deactivated. Run `npx prisma db push`.

2. Add to `src/lib/dal/admin.ts`:

   a. `deactivateTeacherAccount(teacherId: string)`:
      - Set `deactivatedAt` to `new Date()` via Prisma update
      - Also ban the user in Supabase Auth to block login: use `createAdminClient()` from `@/lib/supabase/admin`, get the teacher's supabaseAuthId, call `admin.auth.admin.updateUserById(supabaseAuthId, { ban_duration: 'none' })` -- actually use `banUser` approach: set `banned: true` via `admin.auth.admin.updateUserById(id, { ban_duration: '876000h' })` (100 years = effectively permanent)
      - Return the updated teacher

   b. `reactivateTeacherAccount(teacherId: string)`:
      - Set `deactivatedAt` to null via Prisma update
      - Unban in Supabase: `admin.auth.admin.updateUserById(supabaseAuthId, { ban_duration: 'none' })`
      - Return the updated teacher

   c. `overrideTeacherTier(teacherId: string, tier: string)`:
      - Validate tier is one of: 'free', 'pro', 'pro_plus' (from SubscriptionTier type)
      - Update teacher's `subscriptionTier` via Prisma
      - Return the updated teacher

   d. `createTeacherWithTempPassword(data: { name: string, email: string, tier: string })`:
      - Generate a temporary password: 12 chars, mix of uppercase, lowercase, digits (use crypto.randomBytes or similar)
      - Create Supabase Auth user via admin client: `admin.auth.admin.createUser({ email, password: tempPassword, email_confirm: true, user_metadata: { name } })`
      - Create Prisma Teacher record with the supabaseAuthId from the created user, the given name, email, and tier
      - Return `{ teacher, tempPassword }` -- the temp password is shown once to the admin and never stored

3. Create `src/actions/admin.ts` with server actions:

   a. `deactivateTeacherAction(teacherId: string)`:
      - Call `getAuthenticatedAdmin()` -- return error if not admin
      - Call `deactivateTeacherAccount(teacherId)` from DAL
      - Revalidate `/admin` path
      - Return `{ success: true }`

   b. `reactivateTeacherAction(teacherId: string)`:
      - Same auth check
      - Call `reactivateTeacherAccount(teacherId)`
      - Revalidate, return success

   c. `overrideTierAction(teacherId: string, tier: string)`:
      - Auth check
      - Validate tier is valid SubscriptionTier
      - Call `overrideTeacherTier(teacherId, tier)`
      - Revalidate, return success

   d. `createTeacherAccountAction(formData: FormData)`:
      - Auth check
      - Parse name, email, tier from FormData
      - Validate inputs (name required, email valid, tier valid)
      - Call `createTeacherWithTempPassword({ name, email, tier })`
      - Return `{ success: true, tempPassword }` -- the temp password is returned to the UI for display

   All actions follow the established pattern: 'use server' at top, auth -> validate -> DAL -> revalidate -> return.
  </action>

    - All 4 server actions are exported
