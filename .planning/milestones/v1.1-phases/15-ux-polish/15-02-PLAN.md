  <files>src/components/landing/hero-section.tsx</files>
  <action>
Inspect and fix the hero section's logo display and background color rendering.

**Logo sizing check and fix:**

The hero logo currently uses:
```tsx
<Image
  src="/logo-horizontal.png"
  width={320}
  height={80}
  className="h-auto w-56 sm:w-72 lg:w-80"
/>
```

This renders the logo at: 224px (mobile), 288px (sm), 320px (lg). The `width={320} height={80}` props set the intrinsic aspect ratio. The CSS `w-*` classes control display size.

**Verify the aspect ratio is correct:**
- The logo file is `logo-horizontal.png`. The width/height props should match the actual image aspect ratio.
- If the image's natural dimensions differ significantly from 320x80 (4:1), update the width/height props to match (these are just hints for the layout engine).
- Keep the responsive display sizes as-is (w-56 / w-72 / w-80) since they provide good visual hierarchy.

**Background color check and fix:**

The hero section uses: `bg-gradient-to-b from-brand-blue via-brand-blue to-brand-blue-dark`. This creates a subtle gradient from brand-blue to brand-blue-dark. The brand colors are defined in globals.css using oklch values.

Check these specific issues:
1. The hero section gradient should be a smooth blue gradient without visible banding or harsh transitions
2. The `HowItWorksSection` uses `bg-background` -- confirm no jarring transition from the hero's blue to the white/light background
3. The `UseCasesSection` uses `bg-muted/30` and `PricingSection` uses `bg-brand-blue-light/30` -- these should create alternating subtle backgrounds
4. The `FinalCTASection` uses `bg-gradient-to-b from-brand-blue/5 to-brand-blue/15` which should be a very subtle blue tint

**If the hero-to-background transition is jarring:**
Add a very subtle gradient tail at the bottom of the hero section, e.g., change the gradient to include a transition toward background color at the bottom. Alternatively, add a very thin gradient divider.

**If logo aspect ratio appears wrong** (stretched or squished):
Update the `width` and `height` props on the Image component to better match the actual file dimensions. The `h-auto` class already ensures the height scales proportionally to width, so just ensure the ratio hint is reasonable.

**Do NOT:**
- Change brand color values in globals.css (those are established branding)
- Resize the logo image file itself
- Remove existing responsive breakpoints
- Change section ordering or content
