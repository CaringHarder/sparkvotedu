    **Sports bracket creation DAL** -- Create `src/lib/dal/sports.ts`:

    Import `prisma`, `getProvider`, types, `resolveTeamLogoUrl`, `broadcastBracketUpdate`, `broadcastActivityUpdate`.

    `createSportsBracketDAL(teacherId: string, input: { tournamentId: string, season: number, sessionId: string })`:
    1. Fetch tournament games from provider: `const provider = getProvider()` then `provider.getTournamentGames(input.tournamentId, input.season)`.
    2. Extract unique teams from games (build Set from homeTeam/awayTeam).
    3. Determine gender from tournamentId string (contains 'mens' or 'womens').
    4. Separate First Four games (round === 0 or round === -1) from main bracket games.
    5. Determine bracket size: 64 for standard March Madness.
    6. Determine total rounds: `Math.ceil(Math.log2(64))` = 6.
    7. Create bracket name: e.g., "NCAA March Madness 2026 - Men's" or "NCAA March Madness 2026 - Women's".

    Use a Prisma `$transaction` with 30s timeout (matching 07-16 pattern) to:
    a. Create the Bracket record with:
       - `bracketType: 'sports'`
       - `status: 'draft'` (teacher activates when ready)
       - `predictionStatus: null` (set when predictions open)
       - `predictiveResolutionMode: 'auto'` (sports brackets auto-resolve from API)
       - `externalTournamentId: input.tournamentId`
       - `dataSource: 'sportsdataio'`
       - `lastSyncAt: new Date()`
       - `sportGender: gender`
       - `sessionId: input.sessionId`
       - `size: 64`
       - `maxEntrants: 68` (includes First Four teams)
       - `playInEnabled: true` (for First Four)
       - `viewingMode: 'advanced'`
       - `showSeedNumbers: true`

    b. Create BracketEntrant records for each team:
       - `name: team.shortName`
       - `seedPosition`: Use team.seed if available, otherwise assign incrementally
       - `externalTeamId: team.externalId`
       - `logoUrl: resolveTeamLogoUrl(team.logoUrl, team.abbreviation)`
       - `abbreviation: team.abbreviation`

    c. Create Matchup records for each game:
       - Map SportsDataIO game round numbers to bracket round numbers
       - `round`: Game round (1-6 for main bracket, 0 for First Four)
       - `position`: Derive from game's displayOrder within its round
       - `entrant1Id` / `entrant2Id`: Lookup from entrant records by externalTeamId matching homeTeam/awayTeam
       - `nextMatchupId`: Wire using game's relationship data (the game whose previousHomeGameId or previousAwayGameId matches this game's ID)
       - `externalGameId: game.externalId`
       - `homeScore: game.homeScore`
       - `awayScore: game.awayScore`
       - `gameStatus: game.status`
       - `gameStartTime: new Date(game.startTime)`
       - `isBye: false`
       - `bracketRegion`: Map from game.bracket string ('East', 'West', 'South', 'Midwest' -> store as-is; Final Four/Championship -> null)
       - For First Four games: set `round: 0` and wire `nextMatchupId` to the R1 matchup they feed into

    d. Wire nextMatchupId after all matchups created (two-pass approach matching existing createMatchupsInTransaction pattern):
       - First pass: create all matchups without nextMatchupId
       - Second pass: use SportsDataIO's previousHomeGameId/previousAwayGameId to build the advancement chain
       - For each game that has a previousHomeGameId or previousAwayGameId, find the matchup with that externalGameId and set its nextMatchupId to the current game's matchup ID

    e. If any games are already completed (IsClosed), set winnerId on those matchups (advance the already-decided games).

    Return the created bracket with entrants and matchups.

    `getActiveSportsBrackets()`:
    Query all brackets where `bracketType: 'sports'` and `status` is not 'completed'. Include matchups. Used by cron sync.

    `syncBracketResults(bracketId: string, games: SportsGame[])`:
    1. Fetch all matchups for the bracket that have an externalGameId.
    2. For each matchup, find the corresponding game by externalGameId.
    3. If game data found:
       - Update `homeScore`, `awayScore`, `gameStatus` on the matchup.
       - If `game.isClosed && game.winnerId && !matchup.winnerId`:
         - Look up the entrant whose `externalTeamId` matches `game.winnerId`.
         - Set `winnerId` on the matchup.
         - Propagate the winner to the next matchup (same pattern as existing advanceMatchup): set entrant1Id or entrant2Id on nextMatchup based on position parity (odd -> entrant1, even -> entrant2).
       - Update `lastSyncAt` on the bracket.
    4. After all updates, broadcast: `broadcastBracketUpdate(bracketId, 'bracket_update')`.
