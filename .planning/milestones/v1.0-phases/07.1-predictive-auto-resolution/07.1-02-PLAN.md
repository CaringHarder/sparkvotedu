<task type="auto">
  <name>Task 1: Extend broadcast types and DAL functions</name>
  <files>
    src/lib/realtime/broadcast.ts
    src/lib/dal/prediction.ts
  </files>
  <action>
    1. In broadcast.ts:
       - Add 'reveal_round', 'reveal_complete', 'results_prepared' to BracketUpdateType union

    2. In prediction.ts DAL, update VALID_PREDICTION_TRANSITIONS for auto mode:
       - Add auto-mode transitions. The existing transitions (draft -> predictions_open -> active -> completed) remain for manual/vote_based modes.
       - For auto mode, add these additional valid transitions:
         predictions_open -> tabulating
         tabulating -> previewing
         previewing -> revealing
         revealing -> completed
         previewing -> predictions_open (reopen safety net)
       - Update updatePredictionStatusDAL to check bracket.predictiveResolutionMode and use mode-specific transitions.
       - When transitioning to 'tabulating', do NOT auto-open matchups for voting (that's the manual/vote_based behavior).
       - Keep bracket.status as 'active' throughout the auto flow (only set to 'completed' at the end).

    3. Add DAL function: tabulateBracketPredictions(bracketId, teacherId)
       - Verify bracket ownership, bracketType='predictive', predictiveResolutionMode='auto', predictionStatus='predictions_open' or 'tabulating'
       - Transition predictionStatus to 'tabulating'
       - Fetch all predictions from DB: prisma.prediction.findMany({ where: { bracketId } })
       - Fetch all matchups from DB: prisma.matchup.findMany({ where: { bracketId }, include entrant data })
       - Build TabulationInput[] from matchups
       - Call tabulatePredictions(predictions, matchups, totalRounds) from predictive.ts
       - Write results: for each TabulationResult with winnerId, update matchup.winnerId (keep status 'pending')
       - For resolved matchups, also propagate winner to next matchup entrant slot (odd position -> entrant1Id, even -> entrant2Id) via advanceMatchupWinner pattern
       - Transition predictionStatus to 'previewing'
       - Return { results: TabulationResult[], unresolvedCount: number } where unresolvedCount = ties + no_predictions

    4. Add DAL function: overrideMatchupWinnerDAL(bracketId, teacherId, matchupId, winnerId)
       - Verify ownership, predictionStatus='previewing'
       - Update matchup.winnerId
       - Re-tabulate all downstream rounds: clear winnerId for all matchups in later rounds, re-run tabulatePredictions for affected rounds only
       - Return { success: true }
       - IMPORTANT: After override, winnerId must propagate to next matchup slot. Then re-tabulate downstream rounds because entrants changed.

    5. Add DAL function: releaseResultsDAL(bracketId, teacherId)
       - Verify ownership, predictionStatus='previewing'
       - Verify ALL matchups have winnerId set (no ties or no_predictions remaining)
       - If unresolved matchups remain, return { error: 'All matchups must have winners before release' }
       - Transition predictionStatus to 'revealing'
       - Set revealedUpToRound to 0 (no rounds revealed yet)
       - Return { success: true }

    6. Add DAL function: revealRoundDAL(bracketId, teacherId, round)
       - Verify ownership, predictionStatus='revealing'
       - Verify round is the next round to reveal (revealedUpToRound + 1)
       - Update all matchups for that round from 'pending' to 'decided'
       - Update bracket.revealedUpToRound to round
       - If this is the last round, broadcast 'reveal_complete'
       - Otherwise broadcast 'reveal_round' with { round }
       - Return { success: true, isLastRound: boolean }

    7. Add DAL function: reopenPredictionsDAL(bracketId, teacherId)
       - Verify ownership, predictionStatus='previewing'
       - Clear all non-bye matchup winnerId values
       - Clear all non-bye matchup entrant slots for rounds > 1 (reset propagated entrants)
       - Transition predictionStatus back to 'predictions_open'
       - Set revealedUpToRound to null
       - Return { success: true }
  </action>

<task type="auto">
  <name>Task 2: Server actions for auto-resolution lifecycle</name>
  <files>src/actions/prediction.ts</files>
  <action>
    Add 5 new server actions following the existing pattern (auth -> validate -> feature gate -> DAL -> revalidate -> return):

    1. prepareResults(input: unknown)
       - Auth: getAuthenticatedTeacher
       - Validate: prepareResultsSchema
       - Gate: canUseBracketType(tier, 'predictive')
       - DAL: tabulateBracketPredictions(bracketId, teacherId)
       - Broadcast: broadcastBracketUpdate(bracketId, 'results_prepared', { unresolvedCount })
       - Revalidate: /brackets/{bracketId}
       - Return: { success, results, unresolvedCount } or { error }

    2. overrideWinner(input: unknown)
       - Auth: getAuthenticatedTeacher
       - Validate: overrideMatchupWinnerSchema
       - Gate: canUseBracketType(tier, 'predictive')
       - DAL: overrideMatchupWinnerDAL(bracketId, teacherId, matchupId, winnerId)
       - Revalidate: /brackets/{bracketId}
       - Return: { success } or { error }

    3. releaseResults(input: unknown)
       - Auth: getAuthenticatedTeacher
       - Validate: prepareResultsSchema (same shape -- just bracketId)
       - Gate: canUseBracketType(tier, 'predictive')
       - DAL: releaseResultsDAL(bracketId, teacherId)
       - Broadcast: broadcastBracketUpdate(bracketId, 'prediction_status_changed', { predictionStatus: 'revealing' })
       - Revalidate: /brackets/{bracketId}
       - Return: { success } or { error }

    4. revealNextRound(input: unknown)
       - Auth: getAuthenticatedTeacher
       - Validate: revealRoundSchema
       - Gate: canUseBracketType(tier, 'predictive')
       - DAL: revealRoundDAL(bracketId, teacherId, round)
       - Revalidate: /brackets/{bracketId}
       - Return: { success, isLastRound } or { error }
       - Note: broadcast is handled inside the DAL function

    5. reopenPredictions(input: unknown)
       - Auth: getAuthenticatedTeacher
       - Validate: prepareResultsSchema
       - Gate: canUseBracketType(tier, 'predictive')
       - DAL: reopenPredictionsDAL(bracketId, teacherId)
       - Broadcast: broadcastBracketUpdate(bracketId, 'prediction_status_changed', { predictionStatus: 'predictions_open' })
       - Revalidate: /brackets/{bracketId}
       - Return: { success } or { error }

    Also update the existing updatePredictionStatus action to pass through to the updated DAL which now handles mode-specific transitions.

    All actions follow the try/catch pattern with non-blocking broadcast (.catch(console.error)).
  </action>
