  <action>
Three-part fix for stale override entrants:

**Part A: DAL (src/lib/dal/prediction.ts)**
Change `overrideMatchupWinnerDAL` return type from `Promise<{ success: true } | { error: string }>` to `Promise<{ success: true; results: TabulationResult[] } | { error: string }>`.

At the end of the function (line ~786), instead of `return { success: true }`:
- After the downstream re-tabulation loop completes, re-fetch ALL matchups one final time
- Build TabulationResult[] from the final matchup state (each matchup -> { matchupId, round, winnerId, status based on winnerId presence, voteCount: 0, totalVotes: 0 })
- Return `{ success: true, results }` with the full results array
- For the no-downstream-rounds early path (when downstreamRounds.length === 0), also fetch current matchups and build results the same way

Import TabulationResult type if not already imported.

**Part B: Server Action (src/actions/prediction.ts)**
In `overrideWinner` function (line ~270), change `return { success: true }` to pass through the results:
```
return { success: true, results: result.results }
```

**Part C: Client Component (src/components/bracket/predictive-bracket.tsx)**
In `handleOverrideWinner` (lines 137-149), replace the current implementation that calls `prepareResults()` after override:

Instead of:
```
const result = await overrideWinner(...)
if (result && 'success' in result) {
  const refreshed = await prepareResults(...)
  ...
}
```

Do:
```
const result = await overrideWinner({ bracketId: bracket.id, matchupId, winnerId })
if (result && 'results' in result && result.results) {
  setTabulationResults(result.results as TabulationResult[])
  setUnresolvedCount(
    (result.results as TabulationResult[]).filter(
      (r: TabulationResult) => !r.winnerId && r.status !== 'bye'
    ).length
  )
}
```

This eliminates the blocked `prepareResults()` call and uses the fresh data returned directly from the override.
