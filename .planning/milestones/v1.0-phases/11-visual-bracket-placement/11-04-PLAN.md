    Add state: `const [placementMode, setPlacementMode] = useState<'list' | 'visual'>('list')`

    In the Step 2 section (after the tab buttons for Manual/CSV/Topics and after the entrant counter), add:
    - The PlacementModeToggle component (only show when entrants.length > 0, since you need entrants before visual placement makes sense)
    - When mode is 'list': show the existing EntrantList component (current behavior, unchanged)
    - When mode is 'visual': show either PlacementBracket (for SE, DE, predictive) or PlacementMatchupGrid (for RR) based on bracketType

    For the visual mode components:
    ```typescript
    {placementMode === 'visual' && entrants.length > 0 && (
      bracketType === 'round_robin' ? (
        <PlacementMatchupGrid
          entrants={entrants}
          entrantCount={size}
          onEntrantsChange={handleReorder}
        />
      ) : (
        <PlacementBracket
          entrants={entrants}
          bracketSize={byeInfo?.bracketSize ?? size}
          entrantCount={size}
          onEntrantsChange={handleReorder}
        />
      )
    )}
    ```

    Note: `handleReorder` already handles the seedPosition reassignment logic. The visual placement components call `onEntrantsChange` with the full updated entrants array, and handleReorder normalizes seedPositions. BUT -- for visual placement, we do NOT want to renumber seedPositions sequentially (that would undo the visual placement). We need to update `handleReorder` to accept the entrants as-is when in visual mode, since the placement functions already set correct seedPositions.

    Add a new handler for visual placement:
    ```typescript
    const handleVisualPlacement = useCallback((updatedEntrants: FormEntrant[]) => {
      setEntrants(updatedEntrants)
    }, [])
    ```

    Use `handleVisualPlacement` for the visual mode components and keep `handleReorder` for the list mode EntrantList.

    Important: The Manual/CSV/Topics input tabs should still be visible in visual mode (at the top) so teachers can add more entrants. The mode toggle only switches the view of the entrant list/placement below the input area.

    Preserve the bye helper text and entrant counter in both modes.
