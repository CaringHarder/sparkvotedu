    1. Install @dnd-kit/react and @dnd-kit/dom:
       ```bash
       npm install @dnd-kit/react @dnd-kit/dom
       ```

    2. Create `placement-provider.tsx` as a 'use client' component that wraps children in DragDropProvider from @dnd-kit/react.

    The provider handles the onDragEnd event:
    - Extract source (dragged item) and target (drop zone) from the event
    - Source data contains: `{ type: 'entrant' | 'slot', entrantId?: string, slotIndex?: number }`
    - Target data contains: `{ type: 'slot' | 'pool', slotIndex?: number }`
    - **Case 1 — entrant→slot (pool to bracket):** source.type='entrant' + target.type='slot' → call `placeEntrantInSlot` from placement.ts. If target slot is occupied, the existing entrant is displaced back to pool.
    - **Case 2 — slot→slot (bracket swap):** source.type='slot' + target.type='slot' → call `swapSlots` from placement.ts. Both occupied slots exchange their entrants directly. This enables dragging from one OCCUPIED slot to ANOTHER occupied slot within the bracket for direct swap.
    - **Case 3 — drop on pool:** source dropped on target.type='pool' → no-op (pool is just the unplaced view)
    - Call the `onEntrantsChange(newEntrants)` callback prop with updated entrants array

    Props interface:
    ```typescript
    interface PlacementProviderProps {
      entrants: PlacementEntrant[]
      bracketSize: number
      onEntrantsChange: (entrants: PlacementEntrant[]) => void
      children: React.ReactNode
    }
    ```

    Also provide a context for the "selected entrant" state (click-to-place fallback):
    ```typescript
    interface PlacementContextValue {
      selectedEntrantId: string | null
      setSelectedEntrantId: (id: string | null) => void
      handleSlotClick: (slotIndex: number) => void
      isDragging: boolean
    }
    ```

    Use createContext and useContext. The `handleSlotClick` implements click-to-place: if an entrant is selected and user clicks a slot, place the entrant there (using placeEntrantInSlot). If no entrant selected, clicking a placed slot selects that entrant for moving.

    Track isDragging state by listening to onDragStart/onDragEnd events to enable glow effects on slots.
  </action>

    Props:

    ```typescript
      entrants: PlacementEntrant[]  // All entrants
      placedSlots: Map<number, string>  // slotIndex -> entrantId for placed entrants
      bracketSize: number
    }
    ```

    The pool shows entrants that are NOT in `placedSlots`. Each pool entrant is a draggable item using `useDraggable` from @dnd-kit/react with `data: { type: 'entrant', entrantId: entrant.id }`.

    Layout: Vertical list on desktop (left sidebar style), horizontal scrollable row on mobile.
    - Each entrant shows: seed badge (numbered, bg-muted rounded), entrant name
    - `isDragSource` state from useDraggable applies opacity-50
    - On click: toggle `selectedEntrantId` in PlacementContext (click-to-place)
    - Selected entrant has ring-2 ring-primary visual indicator
    - Show an "Auto Seed" button at the top that calls autoSeed and then onEntrantsChange
    - Show a clear "X" button on each placed entrant. Clicking X resets that entrant's seedPosition to its auto-seed position (index + 1), effectively moving it back to the pool view. There is no separate pool data structure -- the pool is a UI view of entrants not manually placed.

    2. Create `placement-slot.tsx` -- 'use client' component:

    Props:
    ```typescript
    interface PlacementSlotProps {
      slotIndex: number
      expectedSeed: number  // The seed number this slot expects in standard seeding
      placedEntrant: PlacementEntrant | null
      isBye: boolean
      matchupPosition: number  // 1-indexed matchup position in R1
      slotPosition: 1 | 2  // Top (1) or bottom (2) slot in the matchup
    }
    ```

    **Every slot uses `useDroppable`** from @dnd-kit/react with `data: { type: 'slot', slotIndex }` — this makes every slot (empty, placed, or bye) a valid drop target at all times.

    **Occupied slots (placed entrant or bye) ALSO use `useDraggable`** from @dnd-kit/react with `data: { type: 'slot', slotIndex, entrantId }`. This dual draggable+droppable setup is what enables direct slot-to-slot swapping: an occupied slot can be dragged OUT (useDraggable) and another entrant can be dragged INTO it (useDroppable) simultaneously. When a user drags from occupied slot A to occupied slot B, slot A acts as drag source and slot B acts as drop target — the provider's onDragEnd Case 2 (slot→slot) fires and swaps them.

    Visual states:
    - **Empty**: Only droppable (not draggable). Dashed border (`border-dashed border-muted-foreground/30`), dimmed seed label ("Seed {expectedSeed}" in `text-muted-foreground/50 text-sm`), on drag active: glow ring (`ring-2 ring-primary/50 shadow-[0_0_8px_rgba(var(--primary),0.3)]`)
    - **Placed entrant**: Both draggable AND droppable. Solid border, seed badge (muted bg, showing ORIGINAL seed number), entrant name, small X button to reset seedPosition to auto-seed value (moving entrant back to pool view). On click: select this entrant for moving via PlacementContext. `useDraggable` data: `{ type: 'slot', slotIndex, entrantId: placedEntrant.id }`.
    - **Bye**: Both draggable AND droppable. 'BYE' label in italic muted style (`italic text-muted-foreground`), muted bg. Teachers can move byes per locked decision. `useDraggable` data: `{ type: 'slot', slotIndex, isBye: true }`.

    On click (when not dragging): call `handleSlotClick(slotIndex)` from PlacementContext for click-to-place fallback.

    Use `isDropTarget` from useDroppable to conditionally apply glow styles.

    ```typescript
    }
    ```

    This is NOT an SVG component. It is a pure HTML/CSS layout showing Round 1 matchup pairings.

    Implementation:
    1. Compute bracket size from `calculateBracketSizeWithByes(entrantCount).bracketSize` if entrantCount is not power-of-2.
    2. Use `buildSlotMap(bracketSize)` to get the slot order.
    3. Compute R1 matchups: `bracketSize / 2` matchups, each with 2 slots.
    4. For each matchup position (1-indexed), render two PlacementSlot components (top seed slot, bottom seed slot) inside a matchup container.
    5. Use `getByeSlots(entrantCount, bracketSize)` to determine which slots are byes.
    6. Build a `placedSlots` Map from the entrants array: for each entrant, find its slot via `seedToSlot(entrant.seedPosition, bracketSize)`.

    Layout:
