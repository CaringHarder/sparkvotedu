    }

    **How it works:**
    RR does not use bracket slots -- it uses the circle method to generate a schedule from seed numbers. Changing an entrant's seed changes which opponents they face in each round. The matchup grid is a READ-ONLY visualization of the current seed assignment, not a drag target.

    For RR visual placement, the interaction model is different from tree-based brackets:
    - The EntrantPool shows all entrants in their current seed order
    - The matchup grid shows the resulting schedule from `generateRoundRobinRounds(entrantCount)`
    - Teachers reorder entrants in the pool (drag to reorder within the pool, similar to the existing list reorder)
    - The grid live-updates to show how the new seed order changes matchup pairings
    - Each matchup cell shows entrant names (looked up from entrants array by seed number)

    This is effectively a visual preview of the round-robin schedule with drag-to-reorder in the pool.

    Implementation:
    1. Call `generateRoundRobinRounds(entrantCount)` to get the schedule
    2. For each round, render a row of matchup pair cards
    3. Each matchup card shows:
       - Entrant 1 name (looked up: `entrants.find(e => e.seedPosition === matchup.entrant1Seed)?.name`)
       - "vs" divider
       - Entrant 2 name (looked up similarly)
       - Both names in bordered rounded containers
    4. Round labels: "Round 1", "Round 2", etc.

    Layout:
    - Left sidebar: EntrantPool with drag-to-reorder (use @dnd-kit/react sortable for list reorder within the pool)
    - Main area: Read-only matchup grid organized by round
    - Wrap in PlacementProvider for consistent context

    Since RR max is 8 entrants (max 28 matchups across 7 rounds), the grid fits on screen without navigation.

    For the pool reorder in RR mode, use `useSortable` from @dnd-kit/react if available, or implement simple drag-reorder similar to existing entrant-list.tsx native DnD. The key difference from tree-based placement: in RR, users reorder the pool list (changing seedPositions), not drag to specific slots.

    Actually, since we want to keep the DnD library consistent and the research says @dnd-kit/react has sortable capabilities: use useDraggable + useDroppable within the pool to enable reordering. Or more simply: for RR mode, the visual placement IS the reorder list plus a live matchup preview grid. The existing list reorder already works for this. The value-add of "visual placement" for RR is seeing the matchup grid update in real-time.

    Simplified approach:
    - The pool IS a reorderable list (using existing drag pattern or @dnd-kit sortable)
    - The matchup grid is a computed read-only visualization that re-renders when entrants change
    - No droppable slots in the grid itself
    - Wrap everything in a flex layout: pool list on left, matchup grid on right
    - Mobile: pool on top, grid below

    Style the matchup grid:
    - Each round is a section with "Round N" header
    - Matchup pairs in a horizontal row: `flex flex-wrap gap-3`
    - Each matchup pair is a card: `border rounded-lg p-3` with two entrant names and "vs"
    - Odd-entrant rounds show "(BYE)" for the sitting-out entrant
    - Use `text-sm` for entrant names, `text-xs text-muted-foreground` for "vs"
