<task type="auto">
  <name>Task 2: Configure Prisma v7 with Teacher model, push schema to database, and create Supabase client utilities</name>
  <files>
    prisma.config.ts

    src/lib/prisma.ts

  <action>
    1. Create `prisma.config.ts` at project root:
       ```typescript
       import 'dotenv/config'
       import { defineConfig, env } from 'prisma/config'

       export default defineConfig({
         schema: 'prisma/schema.prisma',
         migrations: {
           path: 'prisma/migrations',
         },
         datasource: {
           url: env('DIRECT_URL'),
         },
       })
       ```

    2. Create `prisma/schema.prisma`:
       - Use `prisma-client-js` generator (NOT `prisma-client` -- Turbopack compatibility issue per research).
       - Output to `./generated/prisma`.
       - PostgreSQL datasource with NO url/directUrl (Prisma v7 uses prisma.config.ts for URLs).
       - Teacher model with fields:
         - id: String @id @default(uuid())
         - email: String @unique
         - name: String?
         - avatarUrl: String? @map("avatar_url")
         - subscriptionTier: String @default("free") @map("subscription_tier")
         - stripeCustomerId: String? @map("stripe_customer_id")
         - supabaseAuthId: String @unique @map("supabase_auth_id")
         - createdAt: DateTime @default(now()) @map("created_at")
         - updatedAt: DateTime @updatedAt @map("updated_at")
         - @@map("teachers")

    3. Run `npx prisma generate` to generate the client into prisma/generated/prisma/.

    4. Run `npx prisma db push` to push the Teacher model schema to the Supabase PostgreSQL database. This creates the `teachers` table. The user_setup step has already provided real Supabase credentials in .env.local before this plan executes, so the database connection will be live.

       NOTE: If `npx prisma db push` fails due to invalid credentials (placeholder values still in .env.local), STOP and prompt the user to configure real Supabase credentials in .env.local first, then retry. This is a hard requirement -- the database must exist for downstream plans (DAL, auth) to function.

    5. Create `src/lib/prisma.ts` -- singleton PrismaClient with PrismaPg adapter:
       ```typescript
       import { PrismaClient } from '../../prisma/generated/prisma/client'
       import { PrismaPg } from '@prisma/adapter-pg'

       const globalForPrisma = globalThis as unknown as {
         prisma: PrismaClient | undefined
       }

       function createPrismaClient() {
         const adapter = new PrismaPg({
           connectionString: process.env.DATABASE_URL,
         })
         return new PrismaClient({ adapter })
       }

       export const prisma = globalForPrisma.prisma ?? createPrismaClient()

       if (process.env.NODE_ENV !== 'production') {
         globalForPrisma.prisma = prisma
       }
       ```

    6. Create `src/lib/supabase/server.ts` -- Server-side Supabase client using @supabase/ssr with getAll/setAll cookie pattern (per research). Use `createServerClient` from `@supabase/ssr`. Use `cookies()` from `next/headers`. Include try/catch in setAll for Server Component context.

    7. Create `src/lib/supabase/client.ts` -- Browser Supabase client using `createBrowserClient` from `@supabase/ssr`. Export a `createClient()` function.

    8. Create `src/lib/supabase/admin.ts` -- Service role client for admin operations. Use `createClient` from `@supabase/supabase-js` with the service role key. Include `auth: { autoRefreshToken: false, persistSession: false }` options.

    9. Create `src/lib/utils/validation.ts` -- Zod schemas for auth forms:
       - `signUpSchema`: email (valid email), password (min 8 chars), name (min 1 char)
       - `signInSchema`: email (valid email), password (min 1 char)
       - `forgotPasswordSchema`: email (valid email)
       - `updatePasswordSchema`: password (min 8 chars), confirmPassword (must match password)
       Export all schemas and their inferred TypeScript types.

    10. Create `src/types/database.ts` -- Re-export Prisma generated types for convenience. Export the Teacher type.

    IMPORTANT: The import path for PrismaClient must be `../../prisma/generated/prisma/client` (relative from src/lib/) or use a path alias. Configure a `@prisma/` path alias in tsconfig.json if needed.
  </action>
