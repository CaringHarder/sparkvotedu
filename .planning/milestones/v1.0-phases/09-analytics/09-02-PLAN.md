**Analytics Hub Page** (`src/app/(dashboard)/analytics/page.tsx`):
- Server component
- Auth: `getAuthenticatedTeacher()` -> redirect to /login if null
- Fetch all teacher's brackets and polls in parallel:
  - `prisma.bracket.findMany({ where: { teacherId: teacher.id }, select: { id, name, bracketType, status, size, createdAt, _count: { select: { matchups: { where: { isBye: false } } } } }, orderBy: { updatedAt: 'desc' } })`
  - `prisma.poll.findMany({ where: { teacherId: teacher.id }, select: { id, question, pollType, status, createdAt, _count: { select: { votes: true } } }, orderBy: { updatedAt: 'desc' } })`
- For each bracket, also fetch quick participation count (unique voter count) using a batched approach:
  - Fetch all vote counts grouped by bracketId in one query: `prisma.vote.groupBy({ by: ['matchupId'], where: { matchup: { bracket: { teacherId: teacher.id } } }, _count: { id: true } })` -- but this is complex. Simpler approach: just show the `_count.matchups` as matchup count and `status` badge. Don't fetch per-bracket participation on the hub page (too many queries). Instead, link to the per-bracket analytics page for detail.
- Layout:
  - Page title: "Analytics" with BarChart3 icon
  - Two sections: "Brackets" and "Polls"
  - Each section shows a list/grid of cards (reuse card.tsx pattern)
  - Bracket card: name, type badge (matching bracket-list pattern: violet for type), status badge, matchup count, "View Analytics" link to `/brackets/{id}/analytics`
  - Poll card: question (truncated), type badge, status badge, vote count from _count.votes, "View Analytics" link to `/polls/{id}/analytics`
  - Filter: only show brackets/polls that have been active or completed (exclude drafts -- no analytics for drafts)
  - Empty state: "No brackets with analytics data yet" / "No polls with analytics data yet"
- Use `grid grid-cols-1 gap-4 md:grid-cols-2 lg:grid-cols-3` for cards
