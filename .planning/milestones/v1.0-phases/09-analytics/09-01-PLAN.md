  <action>
Create `src/lib/dal/analytics.ts` with the following functions. All functions follow existing DAL patterns: use prisma directly, no auth checks (enforced in server components/actions), teacher ownership verified by the calling page.

**getBracketParticipation(bracketId: string)**
Returns `{ uniqueParticipants: number, totalVotes: number, totalMatchups: number, sessionParticipantCount: number }`.
- Count unique participantIds from Vote where matchup.bracketId = bracketId using `prisma.vote.findMany` with `distinct: ['participantId']` and select only participantId
- Count total votes with `prisma.vote.count` where matchup.bracketId = bracketId
- Count total non-bye matchups with `prisma.matchup.count` where bracketId and isBye=false
- Get session participant count: first find the bracket's sessionId, then count StudentParticipant where sessionId matches (for participation rate calculation). If no sessionId, return 0.
- Use Promise.all for parallel queries

**getBracketVoteDistribution(bracketId: string)**
Returns `Record<string, { matchupLabel: string, round: number, position: number, entrant1: { id: string, name: string, votes: number }, entrant2: { id: string | null, name: string | null, votes: number }, winnerId: string | null, totalVotes: number }>` keyed by matchupId.
- Fetch all non-bye matchups for the bracket with entrant1 and entrant2 names included (using `include: { entrant1: { select: { id, name } }, entrant2: { select: { id, name } } }`)
- Fetch all votes for the bracket in one query using `prisma.vote.groupBy({ by: ['matchupId', 'entrantId'], where: { matchup: { bracketId } }, _count: { id: true } })` -- this avoids N+1
- Build the nested result by iterating matchups and looking up vote counts from the groupBy result
- matchupLabel format: "R{round} M{position}" (e.g., "R1 M1")
- Sort matchups by round asc, position asc

**getPollParticipation(pollId: string)**
Returns `{ uniqueParticipants: number, totalVotes: number, sessionParticipantCount: number }`.
- Count unique participantIds from PollVote where pollId, rank=1 (rank=1 for simple, counts unique voters for ranked)
- Count total votes with rank=1 (simple poll vote count)
- Get poll's sessionId, then count session participants if sessionId exists
- Use Promise.all

**getPollVoteDistribution(pollId: string, pollType: string)**
Returns `{ options: Array<{ optionId: string, optionText: string, votes: number, percentage: number }>, totalVotes: number }`.
- For simple polls: reuse existing getSimplePollVoteCounts pattern but include option text by joining with PollOption
- For ranked polls: return Borda scores using existing getRankedPollVotes + computeBordaScores from `@/lib/poll/borda`
- Fetch poll options with text for label mapping
- Calculate percentage for each option (guard division by zero with `total > 0 ? Math.round((count / total) * 100) : 0`)

**getPredictiveAnalytics(bracketId: string)**
Returns `{ predictionCount: number, uniquePredictors: number, scores: PredictionScore[] }`.
- Count total predictions with `prisma.prediction.count({ where: { bracketId } })`
- Count unique predictors with `prisma.prediction.findMany` with `distinct: ['participantId']`
- Reuse existing `scoreBracketPredictions(bracketId)` from `@/lib/dal/prediction.ts` for the scores
- Use Promise.all for the counts, then call scoreBracketPredictions

All functions should handle bracket types appropriately:
- SE and DE brackets: use Vote-based analytics
- Round-robin: use Vote-based analytics (RR uses votes too)
- Predictive: use Prediction-based analytics via getPredictiveAnalytics
