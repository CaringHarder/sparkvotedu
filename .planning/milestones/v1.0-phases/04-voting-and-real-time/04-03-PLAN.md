<objective>
Build the server actions for voting and bracket advancement, client-side hooks for real-time updates and optimistic voting, and the polling fallback API endpoint. This connects the data layer (Plan 01) and advancement engine (Plan 02) to the frontend through server actions and reactive hooks.

Purpose: These actions and hooks are the bridge between the data layer and the UI. Every vote cast, every matchup advanced, and every real-time update flows through these files. The transport fallback ensures the app works on school networks that block WebSockets.
Output: Server actions for vote/advance operations, hooks for real-time state and optimistic voting, API route for polling fallback.
</objective>

  <name>Task 1: Vote casting and bracket advancement server actions</name>
  <files>src/actions/vote.ts, src/actions/bracket-advance.ts</files>
  <action>
    **1. Create src/actions/vote.ts** with `'use server'` directive:

    **`castVote(input: unknown)`:**
    - Parse input with `castVoteSchema` from validation.ts
    - Fetch the matchup to get `status` and `bracketId`: `prisma.matchup.findUnique({ where: { id: matchupId }, select: { status: true, bracketId: true, bracket: { select: { sessionId: true } } } })`
    - Check matchup status is "voting" (return `{ error: 'Matchup is not open for voting' }` if not)
    - Check participant is not banned: `prisma.studentParticipant.findUnique({ where: { id: participantId }, select: { banned: true } })` (return error if banned)
    - Call `castVoteDAL(matchupId, participantId, entrantId)` from vote DAL
    - Get updated counts: `getVoteCountsForMatchup(matchupId)` and compute total
    - Fire `broadcastVoteUpdate(bracketId, matchupId, counts, total)` non-blocking (`.catch(console.error)`)
    - Return `{ success: true, votedEntrantId: entrantId }`
    - Wrap in try/catch, return `{ error: 'Failed to cast vote' }` on exception

    **2. Create src/actions/bracket-advance.ts** with `'use server'` directive:

    **`advanceMatchup(input: unknown)`:**
    - Parse with `advanceMatchupSchema`
    - Call `getAuthenticatedTeacher()` from auth DAL (return error if not authenticated)
    - Verify bracket ownership: `prisma.bracket.findFirst({ where: { id: bracketId, teacherId: teacher.id } })`
    - Call `advanceMatchupWinner(matchupId, winnerId, bracketId)` from advancement engine
    - Broadcast: `broadcastBracketUpdate(bracketId, 'winner_selected', { matchupId, winnerId })`
    - Check if bracket is complete via `isBracketComplete(bracketId)`. If complete, broadcast `bracket_completed` event
    - If bracket has sessionId, also call `broadcastActivityUpdate(sessionId)`
    - `revalidatePath('/brackets/${bracketId}')` and `revalidatePath('/brackets/${bracketId}/live')`
    - Return `{ success: true }`

    **`undoAdvancement(input: unknown)`:**
    - Parse with `z.object({ bracketId: z.string().uuid(), matchupId: z.string().uuid() })`
    - Auth check + ownership check
    - Call `undoMatchupAdvancement(matchupId, bracketId)` from advancement engine
    - Broadcast: `broadcastBracketUpdate(bracketId, 'round_advanced', { action: 'undo', matchupId })`
    - Revalidate and return

    **`openMatchupsForVoting(input: unknown)`:**
    - Parse with `openVotingSchema`
    - Auth check + ownership check
    - Call `openMatchupsForVoting(matchupIds)` from vote DAL (batch status update)
    - Broadcast: `broadcastBracketUpdate(bracketId, 'voting_opened', { matchupIds })`
    - Revalidate and return

    **`batchAdvanceRound(input: unknown)`:**
    - Parse with `z.object({ bracketId: z.string().uuid(), round: z.number().int().positive() })`
    - Auth check + ownership check
    - Get all matchups in this round that have winnerId but status is "voting"
    - For each, call advanceMatchupWinner and set status to "decided"
    - Broadcast: `broadcastBracketUpdate(bracketId, 'round_advanced', { round })`
    - Check bracket completion, revalidate, return count

    **`updateBracketVotingSettings(input: unknown)`:**
    - Parse with `updateBracketVotingSettingsSchema`
    - Auth check + ownership check
    - Update bracket with the provided fields (viewingMode, showVoteCounts, votingTimerSeconds)
    - Return updated bracket

    Follow the established pattern from `src/actions/bracket.ts`: Zod parse -> auth check -> DAL call -> revalidatePath -> return result.

<task type="auto">
  <name>Task 2: Real-time hooks, optimistic vote hook, transport fallback, and bracket state API</name>
  <files>src/hooks/use-realtime-bracket.ts, src/hooks/use-vote.ts, src/hooks/use-transport-fallback.ts, src/app/api/brackets/[bracketId]/state/route.ts</files>
  <action>
    **1. Create src/hooks/use-realtime-bracket.ts** (`'use client'`):

    Hook: `useRealtimeBracket(bracketId: string, batchIntervalMs = 2000)`

    - Uses `useMemo(() => createClient(), [])` for Supabase client (established pattern from use-realtime-activities.ts)
    - Maintains `voteCounts` state: `Record<string, { [entrantId: string]: number; total: number }>`
    - Uses `useRef` for `pendingUpdates` accumulator (batching)
    - Sets up an interval that flushes `pendingUpdates.current` into `voteCounts` state every `batchIntervalMs`
    - Subscribes to `bracket:${bracketId}` channel:
      - `vote_update` event: accumulates into `pendingUpdates.current` (NOT direct setState)
      - `bracket_update` event: immediate action -- calls `fetchBracketState()` for structural changes (winner_selected, round_advanced, voting_opened, bracket_completed)
    - `fetchBracketState()` callback: fetches from `/api/brackets/${bracketId}/state`, updates `matchups` state
    - Maintains `bracketCompleted` boolean state, set to true on `bracket_completed` event
    - Cleanup: remove channel on unmount, clear interval
    - Returns `{ voteCounts, matchups, bracketCompleted, refetch: fetchBracketState }`

    **2. Create src/hooks/use-vote.ts** (`'use client'`):

    Hook: `useVote(matchupId: string, participantId: string, initialVote: string | null)`

    - Uses React 19 `useOptimistic` hook: `const [optimisticVote, setOptimisticVote] = useOptimistic(confirmedVote)`
    - `confirmedVote` is a `useState` initialized from `initialVote`
    - `error` state for displaying vote errors
    - `submitVote(entrantId: string)`:
      - Clears error
      - Wraps in `startTransition(async () => { ... })`
      - Sets optimistic vote immediately: `setOptimisticVote(entrantId)`
      - Calls `castVote({ matchupId, participantId, entrantId })` from actions/vote.ts
      - On success: `setConfirmedVote(entrantId)`
      - On error result: `setError(result.error)` -- useOptimistic automatically reverts when transition ends
    - Returns `{ votedEntrantId: optimisticVote, isPending: optimisticVote !== confirmedVote, error, submitVote }`

    **3. Create src/hooks/use-transport-fallback.ts** (`'use client'`):

    Hook: `useTransportFallback(bracketId: string, onData: (data: unknown) => void, pollIntervalMs = 3000)`

    - `transport` state: `'websocket' | 'polling'`
    - On mount, attempts to subscribe to `bracket:${bracketId}` Supabase channel
    - Sets a 5-second timeout. If the channel does not reach `SUBSCRIBED` status within 5 seconds, calls `switchToPolling()`
    - `switchToPolling()`:
      - Sets `transport` to `'polling'`
      - Removes the Supabase channel
      - Starts polling: `setInterval(() => fetch('/api/brackets/${bracketId}/state').then(r => r.json()).then(onData), pollIntervalMs)`
      - Immediately fetches once on switch
    - Returns `{ transport, switchToPolling }`
    - Cleanup: remove channel, clear interval, clear timeout on unmount

    **4. Create src/app/api/brackets/[bracketId]/state/route.ts:**

    - GET handler: extracts `bracketId` from params
    - Fetches bracket with matchups, entrants, and vote counts:
      ```
      prisma.bracket.findUnique({
        where: { id: bracketId },
        include: {
          entrants: { orderBy: { seedPosition: 'asc' } },
          matchups: {
            include: { entrant1: true, entrant2: true, winner: true },
            orderBy: [{ round: 'asc' }, { position: 'asc' }],
          },
        },
      })
      ```
    - For each matchup, fetch vote counts via `getVoteCountsForMatchup(matchupId)` from vote DAL
    - Return JSON response with bracket data, matchups with vote counts, and viewingMode
    - Return 404 if bracket not found
    - Add this route as a public page in proxy.ts: `/api/brackets/` prefix should be accessible without auth (students use it for polling fallback). Add `if (pathname.startsWith('/api/brackets/')) return true` to the `isPublicPage` function.

    **Important implementation notes:**
    - Do NOT use `framer-motion` import path -- the library is now `motion` with `import from "motion/react"`
    - The `useRealtimeBracket` hook batches vote_update events but processes bracket_update events immediately (structural changes need instant UI feedback)
    - The `useVote` hook uses `startTransition` -- this is required for `useOptimistic` to work correctly with server actions
    - The polling fallback is a last resort -- most schools will have working WebSocket. Keep it simple.
