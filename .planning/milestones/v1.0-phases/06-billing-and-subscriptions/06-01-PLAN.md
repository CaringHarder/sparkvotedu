<objective>
Install the Stripe SDK, create the Subscription Prisma model, build the static pricing configuration, and implement the webhook route handler that receives Stripe lifecycle events and syncs subscription state to the database.

Purpose: This is the foundation of the billing system. The webhook handler is the critical piece -- it receives events from Stripe (checkout completed, subscription updated/deleted, payment succeeded/failed) and keeps the local database in sync. The pricing config provides the mapping between Stripe Price IDs and application tiers. The Subscription model stores the full Stripe subscription state for idempotent webhook processing.

Output: Stripe SDK singleton, Subscription model pushed to database, static pricing config with price-to-tier mapping, webhook route handler processing 5 core Stripe events.

<task type="auto">
  <name>Task 2: Webhook route handler with signature verification and event processing</name>
  <files>
    src/app/api/webhooks/stripe/route.ts
    src/app/proxy.ts
  </files>
  <action>
    1. Create `src/app/api/webhooks/stripe/route.ts` as a POST-only route handler:
       - Get raw body via `await request.text()` -- MUST be raw text, NOT `request.json()` (breaks signature verification)
       - Get signature from `request.headers.get('stripe-signature')`
       - Verify signature with `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)`
       - On verification failure: log error, return 400 with `{ error: 'Invalid signature' }`
       - On success: switch on `event.type` and handle 5 events:

       **checkout.session.completed:**
       - Extract `teacherId` from `session.metadata.teacherId`
       - If no teacherId, log error and return (non-fatal)
       - Retrieve full subscription from Stripe: `stripe.subscriptions.retrieve(session.subscription as string)`
       - Get `priceId` from `subscription.items.data[0].price.id`
       - Map to tier using `priceIdToTier(priceId)`
       - Upsert Subscription record (keyed on `teacherId`) with stripeSubscriptionId, stripePriceId, status, stripeCurrentPeriodEnd (convert Unix timestamp * 1000), cancelAtPeriodEnd
       - Update Teacher record: set `subscriptionTier` to mapped tier, set `stripeCustomerId` to `session.customer as string`

       **customer.subscription.updated:**
       - Extract `teacherId` from `subscription.metadata.teacherId`
       - Get priceId, map to tier
       - Upsert Subscription record with latest status, price, period end, cancelAtPeriodEnd
       - Update Teacher `subscriptionTier`:
         - If subscription status is `active` or `past_due`: set tier from priceId mapping (past_due = grace period, teacher keeps access)
         - If subscription status is `canceled`, `unpaid`, or `incomplete_expired`: set tier to `'free'`

       **customer.subscription.deleted:**
       - Extract `teacherId` from `subscription.metadata.teacherId`
       - Update Subscription record: set status to 'canceled'
       - Update Teacher: set `subscriptionTier` to `'free'`

       **invoice.paid:**
       - Extract subscription ID from `invoice.subscription`
       - If subscription exists in DB, update stripeCurrentPeriodEnd from the invoice's period_end
       - This ensures period end stays fresh after successful renewals

       **invoice.payment_failed:**
       - Log the failure for observability
       - Do NOT downgrade the teacher -- the subscription enters `past_due`, and the `customer.subscription.updated` event handles the status change
       - The 3-day grace period is managed by Stripe's dunning/retry settings (Dashboard config)

       - For any unhandled event type, log `Unhandled event type: ${event.type}` and continue
       - Always return `NextResponse.json({ received: true })` with 200 status

    2. Update `src/app/proxy.ts` to make the webhook route public:
       Add to the `isPublicPage` function:
       ```typescript
       if (pathname.startsWith('/api/webhooks/')) return true
       ```
       This ensures Stripe can reach the webhook endpoint without authentication.

    IMPORTANT: Use `request.text()` for raw body -- NOT `request.json()`. Wrap all DB operations in try/catch to prevent partial failures from blocking webhook acknowledgment. Use upsert for idempotency.
  </action>
