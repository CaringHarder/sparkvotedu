  <files>src/lib/dal/billing.ts</files>
  <action>
    Create `src/lib/dal/billing.ts` following the existing DAL patterns in `src/lib/dal/auth.ts` and `src/lib/dal/bracket.ts`.

    Functions to implement:

    1. **getTeacherSubscription(teacherId: string)**: Query `prisma.subscription.findUnique({ where: { teacherId } })` to get the full subscription record. Returns `Subscription | null`.

    2. **getTeacherWithSubscription(teacherId: string)**: Query `prisma.teacher.findUnique({ where: { id: teacherId }, include: { subscription: true } })` to get teacher with subscription in one query. Returns the Teacher with optional Subscription relation.

    3. **getSubscriptionByStripeId(stripeSubscriptionId: string)**: Query by Stripe subscription ID for webhook lookups. Returns `Subscription | null`.

    4. **getTeacherBillingOverview(teacherId: string)**: Returns an object with:
       - `tier`: teacher's current subscriptionTier
       - `subscription`: the Subscription record (or null for free)
       - `usage`: object with `liveBrackets` count (brackets where status='active' and teacherId matches), `draftBrackets` count (status='draft'), `totalPolls` count
       Uses a single Prisma query with `include` and `_count` where possible, or parallel queries.

    All functions import `prisma` from `@/lib/prisma`. No authentication checks in DAL -- callers (server actions, page components) handle auth.
  </action>

<task type="auto">
  <name>Task 2: Checkout and Portal server actions</name>
  <files>src/actions/billing.ts</files>
  <action>
    Create `src/actions/billing.ts` with `'use server'` directive at top. Follow the existing patterns in `src/actions/bracket.ts` for auth verification and error handling.

    1. **createCheckoutSession(priceId: string)**:
       - Call `getAuthenticatedTeacher()` -- return `{ error: 'Not authenticated' }` if null
       - Validate `priceId` is one of the known Price IDs from `PRICE_IDS` (imported from `@/config/pricing.ts`). Return `{ error: 'Invalid price' }` if not.
       - Call `stripe.checkout.sessions.create()` with:
         - `mode: 'subscription'`
         - `line_items: [{ price: priceId, quantity: 1 }]`
         - `success_url: \`\${process.env.NEXT_PUBLIC_SITE_URL}/billing/welcome?session_id={CHECKOUT_SESSION_ID}\``
         - `cancel_url: \`\${process.env.NEXT_PUBLIC_SITE_URL}/pricing\``
         - `customer: teacher.stripeCustomerId || undefined` (reuse existing Stripe customer if they have one)
         - `customer_email: !teacher.stripeCustomerId ? teacher.email : undefined` (only set email if no existing customer)
         - `metadata: { teacherId: teacher.id }`
         - `subscription_data: { metadata: { teacherId: teacher.id } }` (so subscription events also carry teacherId)
         - `allow_promotion_codes: true` (for future promo code support)
       - Call `redirect(session.url!)` to send teacher to Stripe Checkout

    2. **createPortalSession()**:
       - Call `getAuthenticatedTeacher()` -- return `{ error: 'Not authenticated' }` if null
       - If `!teacher.stripeCustomerId`, return `{ error: 'No active subscription' }`
       - Call `stripe.billingPortal.sessions.create()` with:
         - `customer: teacher.stripeCustomerId`
         - `return_url: \`\${process.env.NEXT_PUBLIC_SITE_URL}/billing\``
       - Call `redirect(session.url)` to send teacher to Stripe Portal

    IMPORTANT: Use `redirect()` from `next/navigation` for both actions. The redirect throws internally (Next.js pattern), so do NOT wrap it in try/catch that would swallow it. Place the redirect call AFTER the try/catch for the Stripe API call. Wrap the Stripe API call in try/catch and return `{ error: message }` on failure.
  </action>

  </verify>
