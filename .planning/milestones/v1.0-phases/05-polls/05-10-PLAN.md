  <action>
Modify the student poll page to subscribe to real-time poll events and show the winner reveal animation when the poll closes. This is a single-file change.

**Step 1 -- Add useRealtimePoll subscription:**

Import `useRealtimePoll` from `@/hooks/use-realtime-poll` and `PollReveal` from `@/components/poll/poll-reveal`.

In the `StudentPollVotingPage` component, after the page state is resolved to `ready`, call `useRealtimePoll(pollId)` to get `{ pollStatus, voteCounts, bordaScores }`. This hook must be called unconditionally (React rules of hooks) -- call it at the top level of the component with the pollId from params. It handles its own lifecycle.

IMPORTANT: The hook is already built and handles everything -- Supabase broadcast subscription, poll_closed/poll_activated events, transport fallback. We just need to consume its `pollStatus` output.

**Step 2 -- Track poll closure for reveal trigger:**

Add state: `const [showReveal, setShowReveal] = useState(false)` and `const [closedDetected, setClosedDetected] = useState(false)`.

Add a `useEffect` watching `pollStatus` from the real-time hook. When `pollStatus` changes to `'closed'` AND the current page state is `'ready'` (student was actively voting), set `setShowReveal(true)` and `setClosedDetected(true)`. Use a ref (`prevPollStatusRef`) to detect the transition -- only trigger when going FROM a non-closed status TO closed, not on initial mount when poll might already be closed.

Do NOT use the `forceReveal` prop pattern from the teacher page -- that pattern exists because the teacher explicitly clicks "Close Poll" and needs synchronous trigger before router.refresh(). On the student page, the trigger comes from the real-time hook detecting `poll_closed` broadcast, which is the correct async path.

**Step 3 -- Compute winner text for reveal:**

When showing the reveal, compute the winner text from the poll data available in state. For simple polls, find the option with the highest vote count from the hook's `voteCounts`. For ranked polls, use `bordaScores[0]` from the hook. Fall back to "Results are in!" if no votes exist. This mirrors the logic in `PollResults` component (see lines 72-90 of poll-results.tsx).

**Step 4 -- Render PollReveal overlay:**

When `showReveal` is true, render `<PollReveal winnerText={winnerText} onDismiss={() => setShowReveal(false)} />` as a sibling at the bottom of the JSX, outside the container div. The PollReveal component renders as a fixed overlay (z-50) so placement in DOM tree doesn't matter -- but keep it outside the container for cleanliness.

**Step 5 -- Transition to closed state after reveal:**

When `closedDetected` is true AND `showReveal` is false (reveal was dismissed), show a "poll closed" state instead of the voting UI. This replaces the voting components with a centered message showing the poll question, "This poll has been closed", and the winner text. This mirrors the existing `not-active` state for `status === 'closed'` but is reached via live transition rather than initial load.

Also: when `closedDetected` is true but `showReveal` is still true, keep rendering the voting UI underneath the overlay (the overlay is fixed position, so the UI behind is irrelevant).

**Step 6 -- Handle already-closed on mount:**

The existing `not-active` state handler (line 200-237) already handles polls that are closed when the student first loads the page. No change needed there -- `closedDetected` only activates when the real-time hook detects a live transition.

**What NOT to do:**
- Do NOT embed the full PollResults component (with charts, participation bar, etc.) on the student page. Students should see the reveal animation and a simple closed state -- the full results dashboard is teacher-only.
- Do NOT modify useRealtimePoll.ts -- it already handles poll_closed events correctly.
- Do NOT modify PollReveal.tsx -- it's a standalone overlay that works as-is.
- Do NOT add connectedStudents or participation tracking -- not needed for student view.
