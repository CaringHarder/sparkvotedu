    **Part A: Intercept pinch-to-zoom in BracketZoomWrapper (bracket-zoom-wrapper.tsx)**

    The current BracketZoomWrapper uses native `overflow: auto` scrolling with no wheel event interception. Trackpad pinch gestures (which browsers emit as `wheel` events with `ctrlKey: true`) are not intercepted, so the browser performs native page zoom.

    1. Add a `useEffect` that attaches a `wheel` event listener with `{ passive: false }` to the container ref. This listener intercepts ONLY pinch gestures (ctrlKey === true) and translates them into scale changes, while letting normal scroll wheel events pass through:

       ```typescript
       import { useRef, useState, useEffect } from 'react'

       // Inside BracketZoomWrapper, after containerRef and scale state:
       useEffect(() => {
         const el = containerRef.current
         if (!el) return
         const handleWheel = (e: WheelEvent) => {
           // Only intercept pinch-to-zoom (ctrlKey is set by browser for trackpad pinch)
           if (!e.ctrlKey) return  // Let normal scroll pass through
           e.preventDefault()
           const factor = e.deltaY > 0 ? 0.95 : 1.05
           setScale((s) => Math.max(minScale, Math.min(maxScale, s * factor)))
         }
         el.addEventListener('wheel', handleWheel, { passive: false })
         return () => el.removeEventListener('wheel', handleWheel)
       }, [minScale, maxScale])
       ```

       Use 0.95/1.05 factors (not 0.9/1.1) for smoother incremental pinch zoom that doesn't jump too aggressively.

    2. Add `touch-action: pan-x pan-y` CSS to the scrollable container div. This allows native scroll but prevents browser zoom on mobile pinch:
       ```tsx
       <div
         ref={containerRef}
         className="overflow-auto rounded-lg border bg-background"
         style={{ maxHeight: '70vh', touchAction: 'pan-x pan-y' }}
       >
       ```
