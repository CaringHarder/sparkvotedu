<feature>
  <name>Predictive Bracket Scoring Engine</name>
  <files>src/lib/bracket/predictive.ts, src/lib/bracket/__tests__/predictive.test.ts</files>
  <behavior>
  Pure functions. No DB calls.

  getPointsForRound(round: number) -> number:
  - Round 1 -> 1
  - Round 2 -> 2
  - Round 3 -> 4
  - Round 4 -> 8
  - Round 5 -> 16
  - Round 6 -> 32
  - Formula: Math.pow(2, round - 1)

  scorePredictions(
    predictions: Array of { participantId, matchupId, predictedWinnerId },
    resolvedMatchups: Array of { id, round, winnerId },
    totalRounds: number
  ) -> PredictionScore[]:

  Test cases:

  Case 1: Perfect bracket (8-team, 3 rounds, 7 matchups):
  - Student predicts all 7 matchups correctly
  - Score = 4*1 + 2*2 + 1*4 = 12 points, 7/7 correct
  - pointsByRound: { 1: {correct:4, total:4, points:4}, 2: {correct:2, total:2, points:4}, 3: {correct:1, total:1, points:4} }

  Case 2: All wrong:
  - Student predicts all 7 matchups incorrectly
  - Score = 0 points, 0/7 correct

  Case 3: Partial results:
  - Only round 1 resolved (4 matchups). Student got 3 right, 1 wrong.
  - Score = 3*1 = 3 points, 3/4 correct (out of 4 resolved)
  - Unresolved matchups (R2, R3) do not contribute to score

  Case 4: Multiple students:
  - Student A: 5 correct (R1: 3/4=3pts, R2: 2/2=4pts, R3: 0/1=0pts) = 7 total
  - Student B: 4 correct (R1: 4/4=4pts, R2: 0/2=0pts, R3: 0/1=0pts) = 4 total
  - Leaderboard: A rank 1, B rank 2

  Case 5: Tiebreaker:
  - Two students with same total points; one has more correct picks -> higher rank
  - If still tied: same rank

  Case 6: Empty predictions:
  - Returns empty array

  Case 7: Predictions for bye matchups should not exist (byes are skipped in prediction UI).
  The scoring engine should handle them gracefully (if a predicted matchup has no resolution, it's simply ignored).
  </behavior>
  <implementation>
  1. getPointsForRound: return Math.pow(2, round - 1)

  2. scorePredictions:
     - Build a Map of matchupId -> { round, winnerId } from resolvedMatchups
     - Iterate predictions, accumulate per-participant scores
     - For each prediction: look up matchup in map, if resolved, check if prediction matches winnerId
     - If correct: add points for that round
     - Build pointsByRound breakdown
     - Sort by totalPoints desc, then correctPicks desc
     - Return PredictionScore[] (without participantName -- caller fills that from DB)

  Note: PredictionScore.participantName will be empty string from this engine (the DAL layer fills it from the DB). The engine operates on IDs only.
  </implementation>
</feature>
