<objective>
Fix zoom control buttons and entrant click targets by preventing the usePanZoom hook's setPointerCapture from stealing pointer events from interactive child elements.

Purpose: Test 16 fails because usePanZoom's handlePointerDown calls setPointerCapture on EVERY pointerdown event within the container, including clicks on zoom buttons and SVG entrant rects. setPointerCapture redirects all subsequent pointer events to the container, preventing the browser from synthesizing 'click' events on child elements. Two-finger scroll zoom works because wheel events are independent of pointer capture.

Output: Zoom buttons and entrant clicks work. Drag-to-pan still works on empty background.
</objective>

    **Fix 1: bracket-zoom-wrapper.tsx -- Stop pointer capture on zoom controls**

    Add onPointerDown with stopPropagation to the floating controls container div (line 55). This prevents the pointerdown event from bubbling up to the containerRef div where usePanZoom's handlePointerDown would call setPointerCapture.

    Change line 55 from:
    ```
    <div className="absolute bottom-3 right-3 flex items-center gap-1 rounded-lg border bg-background/90 p-1 shadow-sm backdrop-blur-sm">
    ```
    To:
    ```
    <div
      className="absolute bottom-3 right-3 flex items-center gap-1 rounded-lg border bg-background/90 p-1 shadow-sm backdrop-blur-sm"
      onPointerDown={(e) => e.stopPropagation()}
    >
    ```

    This single change fixes all three zoom buttons (zoom in, zoom out, fit-to-screen) because they are all children of this div.

    **Fix 2: bracket-diagram.tsx -- Stop pointer capture on clickable entrant rects**

    For the clickable entrant SVG <rect> elements (the transparent click targets for voting), add onPointerDown with stopPropagation.
