  <name>Task 1: Prediction DAL and server actions</name>
  <files>src/lib/dal/prediction.ts, src/actions/prediction.ts</files>
  <action>
1. Create src/lib/dal/prediction.ts:

   a. submitPredictions(bracketId, participantId, predictions: Array of {matchupId, predictedWinnerId}):
      - Verify bracket exists and predictionStatus is 'predictions_open'
      - Verify participant exists and is not banned
      - Filter out predictions for bye matchups (isBye=true matchups should not have predictions)
      - Use prisma.prediction.createMany with skipDuplicates for bulk insert
      - For updates (student editing predictions): use upsert pattern -- delete existing predictions for this participant+bracket, then createMany
      - Return the count of predictions saved

   b. getParticipantPredictions(bracketId, participantId):
      - Query all predictions for this participant in this bracket
      - Include the predictedWinner entrant name
      - Return PredictionData[]

   c. scoreBracketPredictions(bracketId):
      - Query all predictions for this bracket
      - Query all resolved matchups (status='decided' and winnerId is not null)
      - Calculate totalRounds from bracket size
      - Call scorePredictions from predictive.ts engine
      - Enrich with participant names from StudentParticipant
      - Return PredictionScore[]

   d. getMatchupPredictionStats(bracketId, matchupId):
      - Count predictions grouped by predictedWinnerId
      - Return { entrant1Predictions: number, entrant2Predictions: number, totalPredictions: number }
      - Only return stats for resolved matchups (predictions are hidden until resolution per CONTEXT.md)

   e. updatePredictionStatusDAL(bracketId, teacherId, status):
      - Verify bracket ownership
      - Validate transition:
        - draft -> predictions_open (opens predictions)
        - predictions_open -> active (closes predictions, bracket begins resolving)
        - active -> completed (all matchups resolved)
      - Update bracket.predictionStatus
      - Broadcast prediction status change

2. Create src/actions/prediction.ts:
   'use server'

   a. submitPrediction action:
      - Validate input with submitPredictionSchema
      - This is an UNAUTHENTICATED student action (like castVote)
      - Validate: participant exists, not banned, bracket predictionStatus is 'predictions_open'
      - Call submitPredictions DAL
      - Broadcast prediction update
      - Return { success: true, count: N }

   b. updatePredictionStatus action:
      - Auth (teacher) -> validate -> call updatePredictionStatusDAL -> revalidate
      - Feature gate: canUseBracketType(tier, 'predictive')

   c. getLeaderboard action:
      - Auth (teacher or student via participantId) -> call scoreBracketPredictions -> return scores

   d. getMyPredictions action:
      - Student action (unauthenticated, uses participantId)
      - Call getParticipantPredictions
      - Return predictions for this participant

3. Wire predictive bracket creation into createBracketDAL:
   - When bracketType === 'predictive':
     - Use the same matchup generation as single-elimination (predictive uses single-elim structure)
     - Apply byes if non-power-of-two (use generateMatchupsWithByes)
     - Set predictionStatus to null initially (teacher sets to 'predictions_open' when ready)
     - Store predictiveMode and predictiveResolutionMode on the bracket
  </action>
