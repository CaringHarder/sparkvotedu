## Goal

Replace the current "render everything at once, zoom/scroll to navigate" approach for large brackets (16+ entrants) with a focused **region-based tab navigation** pattern:

- **One region at a time** shown at full readable size
- **Mini-map thumbnail** at top for visual context and quick navigation
- **Tab bar** below mini-map for region switching
- **Auto-focus** on the region containing active voting matchups

This applies to:
- **Student Advanced view** (16+ SE brackets)
- **Teacher Dashboard** (16+ SE/Predictive brackets, always uses this regardless of simple/advanced)
- **Bracket Detail page** (static view)

Student **Simple view** is unchanged — it already shows one matchup at a time with animated slides for all bracket sizes.

## Region Splitting Logic

| Bracket Size | Regions | Per Region | Connecting |
|-------------|---------|-----------|------------|
| 16 (4 rounds) | 2: Top Half, Bottom Half | 8 entrants, 3 rounds each | Championship (1 matchup) |
| 32 (5 rounds) | 2: Top Half, Bottom Half | 16 entrants, 4 rounds each | Championship (1 matchup) |
| 64 (6 rounds) | 4: Region 1-4 | 16 entrants, 4 rounds each | Final Four (2 rounds, 3 matchups) |

Algorithm: Trace any matchup's round/position back to its round-1 ancestor to determine which region it belongs to. Generalization of the existing `getQuadrant()` logic in `quadrant-bracket-layout.tsx`.

## Tasks

### Task 1: Create `RegionBracketView` component

**File**: `src/components/bracket/region-bracket-view.tsx` (new)

Core container component that orchestrates region-based navigation:

```
+---------------------------------------+
|  [Mini-map thumbnail]                 |
+---------------------------------------+
| [Top Half] [Bottom Half] [Championship]|  <- tab bar
+---------------------------------------+
|                                       |
|  Full-size BracketDiagram for the     |
|  selected region (scrollable if       |
|  needed, no zoom wrapper)             |
|                                       |
+---------------------------------------+
```

**Region splitting logic** (generalized from QuadrantBracketLayout):
- Compute `regionRounds` = number of rounds per region (varies by bracket size)
- For each matchup: trace back to round-1 ancestor, bucket into region by position range
- Normalize positions within each region (1-based)
- Connecting rounds (above regionRounds) go into a "Championship" or "Final Four" group

**Props**: Same as current BracketDiagram/QuadrantBracketLayout — pass through all interactive props:
- `matchups`, `totalRounds`, `bracketSize`
- `onEntrantClick`, `votedEntrantIds`, `allowPendingClick` (student voting)
- `voteLabels`, `onMatchupClick`, `selectedMatchupId` (teacher dashboard)

**State**:
- `activeRegion`: which tab is selected
- Auto-navigate: when realtime matchups update and a new region has voting matchups, switch to that region

**Region tab bar**:
- Horizontal row of tab buttons with region names
- Active tab highlighted with primary color underline
- Badge on each tab showing votable matchup count (if any)
- "Championship" / "Final Four" tab at the end

### Task 2: Create `BracketMiniMap` component

**File**: `src/components/bracket/bracket-mini-map.tsx` (new)

Small visual bracket thumbnail for contextual navigation:

- Renders the full bracket's `BracketDiagram` at ~12-15% scale inside a fixed-height container (~80px)
- Overlay: semi-transparent colored rectangles for each region boundary
- Active region rectangle has primary color border/fill
- Regions with voting matchups get a subtle animated indicator
- Click on any region rectangle → switches the active tab
- Container has `overflow: hidden`, `pointer-events: none` on the SVG, `pointer-events: auto` on the overlay rects

**Props**:
- `matchups: MatchupData[]`
- `totalRounds: number`
- `regions: RegionDef[]` (computed region boundaries)
- `activeRegion: string`
- `onRegionClick: (regionKey: string) => void`
- `votingRegions: Set<string>` (which regions have voting matchups)
