  <name>Task 1: Add ClassSession and StudentParticipant models to Prisma schema</name>
  <files>prisma/schema.prisma</files>
  <action>
Add two new models to the existing Prisma schema, AFTER the existing Teacher model. Also add a `classSessions` relation field to the Teacher model.

**Teacher model update** -- add relation field:
```
classSessions ClassSession[]
```

**ClassSession model:**
- `id` String @id @default(uuid())
- `code` String -- 6-digit numeric code (unique among active sessions, not globally unique)
- `name` String? -- optional display name for the session
- `status` String @default("active") -- values: active, paused, ended, archived
- `teacherId` String @map("teacher_id")
- `teacher` Teacher @relation(fields: [teacherId], references: [id])
- `createdAt` DateTime @default(now()) @map("created_at")
- `updatedAt` DateTime @updatedAt @map("updated_at")
- `endedAt` DateTime? @map("ended_at")
- `participants` StudentParticipant[] -- relation to participants
- `@@index([code, status])` -- fast lookup by active code
- `@@map("class_sessions")`

**StudentParticipant model:**
- `id` String @id @default(uuid())
- `funName` String @map("fun_name")
- `deviceId` String @map("device_id") -- localStorage UUID (primary identifier)
- `fingerprint` String? -- FingerprintJS hash (secondary signal, nullable)
- `recoveryCode` String? @map("recovery_code") -- personal recovery code (tertiary)
- `rerollUsed` Boolean @default(false) @map("reroll_used")
- `banned` Boolean @default(false)
- `lastSeenAt` DateTime @default(now()) @map("last_seen_at")
- `createdAt` DateTime @default(now()) @map("created_at")
- `sessionId` String @map("session_id")
- `session` ClassSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)
- `@@unique([sessionId, deviceId])` -- one participant per device per session
- `@@unique([sessionId, funName])` -- unique fun names within session
- `@@index([sessionId, fingerprint])` -- fingerprint lookup for returning students
- `@@index([recoveryCode])` -- recovery code lookup
- `@@map("student_participants")`

After adding models, run `npx prisma db push` to push schema to Supabase. Verify no errors.

IMPORTANT: Keep the existing `generator client` and `datasource db` blocks unchanged. Keep the existing Teacher model fields unchanged -- only add the `classSessions` relation field.

  <action>
Create four files in the student utility layer:

**1. `src/lib/student/class-codes.ts`** -- Class code generation utility
- Export `generateClassCode(): Promise<string>` function
- Uses `crypto.randomInt(100000, 1000000)` for secure random 6-digit code generation (NOT Math.random)
- Import prisma from `@/lib/prisma`
- Checks generated code against active/paused sessions in database to ensure uniqueness
- Retry loop with MAX_ATTEMPTS = 10, throws Error if all attempts fail
- Export `validateClassCode(code: string): boolean` -- validates format (exactly 6 digits)
- Follow the exact pattern from 02-RESEARCH.md Pattern 2

**2. `src/lib/student/fun-names-words.ts`** -- Curated word lists
- Export `ADJECTIVES: Record<string, string[]>` -- object keyed by uppercase letter (A-Z), each key maps to array of 15-25 classroom-appropriate adjectives starting with that letter
- Export `ANIMALS: Record<string, string[]>` -- object keyed by uppercase letter (A-Z), each key maps to array of 10-20 animals starting with that letter
- Cover ALL 26 letters. Some rare letters (Q, X, Z) can have fewer entries (5-8 minimum).
- Total should be 400+ adjectives and 250+ animals for sufficient pool size
- All words must be classroom-appropriate (no negative/scary/violent words)
- Examples: A: ['Amazing', 'Awesome', 'Adventurous', ...], B: ['Bold', 'Brave', 'Brilliant', ...]
- Animals examples: A: ['Armadillo', 'Antelope', 'Albatross', ...], D: ['Dragon', 'Dolphin', 'Deer', ...]

**3. `src/lib/student/fun-names.ts`** -- Fun name generator
- Import ADJECTIVES and ANIMALS from `./fun-names-words`
- Export `generateFunName(existingNames: Set<string>): string`
- Strategy: pick random letter that has both adjectives and animals, pick random adjective and animal from that letter, check uniqueness against existingNames Set
- MAX_ATTEMPTS = 100, fallback returns `Student ${random 4-digit number}`
- Result format: "Adjective Animal" (e.g., "Daring Dragon", "Mighty Moose")
- Follow the exact pattern from 02-RESEARCH.md Pattern 3

**4. `src/types/student.ts`** -- TypeScript types for the student subsystem
- Export interface `DeviceIdentity { deviceId: string; fingerprint: string; ready: boolean }`
- Export interface `ClassSessionData { id: string; code: string; name: string | null; status: string; teacherName: string | null }`
- Export interface `StudentParticipantData { id: string; funName: string; rerollUsed: boolean; recoveryCode: string | null; sessionId: string }`
- Export interface `JoinResult { participant?: StudentParticipantData; session?: ClassSessionData; returning?: boolean; error?: string }`

Create parent directories if they don't exist (`src/lib/student/`, `src/types/`).
