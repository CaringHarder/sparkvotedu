**2. `src/components/teacher/qr-code-display.tsx`** -- QR code display with toggle (Client Component)
- `'use client'`
- Import `QRCodeSVG` from `qrcode.react`
- Props: `code: string` (6-digit class code)
- Construct join URL: `${window.location.origin}/join/${code}`
- Toggle button: "Show QR Code" / "Hide QR Code"
- When visible: render QRCodeSVG with size={200}, level="M" (medium error correction)
- Display the join URL below the QR code as copyable text
- Follow the pattern from 02-RESEARCH.md "QR Code Display Component"

**3. `src/components/teacher/student-roster.tsx`** -- Student roster display (Client Component)
- `'use client'`
- Props: `sessionId: string, participants: StudentParticipantData[]`
- Header: "X of Y connected" -- X from Supabase Realtime Presence (live), Y from participants.length (total who ever joined)
- Use the `useSessionPresence` hook (created in Task 2) for live connected count
- List all participants with their fun name, joined date, and status (connected/disconnected/banned)
- Connected students shown with a green dot, disconnected with gray dot
- Banned students shown with strikethrough and red indicator

**4. `src/components/teacher/student-management.tsx`** -- Remove/ban controls (Client Component)
- `'use client'`
- Props: `participantId: string, funName: string, banned: boolean, onAction: () => void`
- For each student in roster: dropdown menu with "Remove" and "Ban Device" options
- Remove: calls removeStudent server action, removes participant entirely
- Ban: calls banStudent server action, marks participant as banned (device can't rejoin)
- Confirmation dialog before destructive actions: "Remove {funName} from this session?" / "Ban {funName}'s device from rejoining?"
- After action: call onAction callback to refresh roster data

**5. `src/app/(dashboard)/sessions/page.tsx`** -- Teacher sessions list page (Server Component)

- Below: list of existing sessions as cards with:
  - Session name (or "Unnamed Session"), code, status badge, participant count
  - Click to navigate to session detail page
  - Active sessions highlighted, ended sessions grayed

**6. `src/app/(dashboard)/sessions/[sessionId]/page.tsx`** -- Session detail page (Server Component)
- Import getAuthenticatedTeacher and getSessionWithParticipants from DAL
- Fetch session with participants, verify teacher ownership
- Display:

  - QRCodeDisplay component with the session code
  - "End Session" button (calls endSession server action) -- only for active sessions
  - StudentRoster component with participant list
- If session not found or unauthorized: redirect to /dashboard/sessions

Create the student session view components and Supabase Realtime hooks.

**1. `src/hooks/use-student-session.ts`** -- Presence tracking hook (Client Component hook)
- `'use client'` (if needed at module level, or just ensure it's used in client components)
- Import `useEffect`, `useState` from 'react'
- Import `createClient` from `@/lib/supabase/client`
- Export `useSessionPresence(sessionId: string, funName: string)` hook
- Subscribe to Supabase Presence on channel `session:${sessionId}`
- On 'presence' 'sync' event: read presenceState(), flatten values, set connectedStudents
- On subscribe success ('SUBSCRIBED'): track own presence with `{ funName, joinedAt: new Date().toISOString() }`
- Cleanup on unmount: untrack + removeChannel
- Return `{ connectedStudents, connectedCount }`
- Follow the EXACT pattern from 02-RESEARCH.md "Supabase Realtime Presence for Connection Tracking"

**2. `src/hooks/use-realtime-activities.ts`** -- Activity list hook (Client Component hook)
- Import `useEffect`, `useState`, `useCallback` from 'react'
- Import `createClient` from `@/lib/supabase/client`
- Define Activity interface: `{ id: string, name: string, type: 'bracket' | 'poll', participantCount: number, hasVoted: boolean, status: string }`
- Export `useRealtimeActivities(sessionId: string, participantId: string)` hook
- Initial fetch: load activities from a server action or API endpoint (for now, return empty array since brackets/polls don't exist yet -- this is a scaffold for Phase 3/4/5)
- Subscribe to Supabase Realtime channel `activities:${sessionId}` for updates
- On any postgres_changes event or broadcast: refetch activities
- Return `{ activities, loading }`
- NOTE: Since brackets and polls don't exist yet (Phase 3+), this hook will initially always return an empty array. The hook structure and Supabase channel setup is what matters -- the data source will be connected in future phases.
- Follow the pattern from 02-RESEARCH.md Pattern 4 (use-realtime-activities.ts)

**3. `src/components/student/empty-state.tsx`** -- Waiting state (Client Component)

- Branded empty state for when no activities are active

- Centered layout, warm and encouraging tone
- This is what students see between activities or when they join early

**4. `src/components/student/activity-card.tsx`** -- Activity card (Client Component)
- `'use client'`
- Props: `activity: Activity` (from the Activity interface)
- Card layout: activity name, type icon (bracket icon vs poll icon), participant count
