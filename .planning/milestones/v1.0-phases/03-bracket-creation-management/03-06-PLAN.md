  **Create src/components/bracket/bracket-form.tsx ('use client'):**

  A multi-step wizard with 3 steps:

  **Step 1: Bracket Info**
  - Name input (required, max 100 chars)
  - Description textarea (optional, max 500 chars)
  - Size selector: radio buttons or segmented control for 4, 8, 16
  - "Next" button (disabled until name and size selected)

  **Step 2: Add Entrants**
  - Three tabs or method selectors: "Manual", "CSV Upload", "Topic Lists"
  - **Manual tab:** Text input + "Add" button. Each added entrant appears in the EntrantList below. Teacher types name, clicks Add (or presses Enter), entrant appended to list.
  - **CSV tab:** Renders CSVUpload component. On successful parse, adds parsed entrants to the list.
  - **Topics tab:** Renders TopicPicker component. On selection, replaces entrant list with topic entries (sliced to match bracket size).
  - Below the tab content: EntrantList component showing current entrants with drag reorder, edit, and remove
  - Counter showing "X / {size} entrants" with color feedback (red if wrong count, green if exactly right)
  - "Next" button (disabled until entrants.length === size)
  - "Back" button to return to Step 1

  **Step 3: Review & Create**
  - Summary: bracket name, description, size, entrant list (numbered by seed)
  - Optional: Preview of bracket diagram using BracketDiagram component (render a preview with the engine-generated matchup structure -- call generateMatchups client-side for preview only)
  - "Create Bracket" submit button
  - "Back" button to return to Step 2
  - On submit: call `createBracket` server action with { bracket: { name, description, size }, entrants: [{ name, seedPosition }] }
  - Show loading state during submission
  - On success: redirect to `/brackets/${bracket.id}` using `useRouter().push()`
  - On error: display error message using toast or inline error

  **State management:**
  - `step`: 1 | 2 | 3
  - `name`: string
  - `description`: string
  - `size`: 4 | 8 | 16 | null
  - `entrants`: Array<{ id: string, name: string, seedPosition: number }>
  - Use `nanoid` for client-side temporary entrant IDs
  - When size changes and entrants.length > newSize, truncate entrants to newSize

  **UI framework:** Use shadcn/ui components (Button, Input, Textarea, Card, Tabs or custom tab UI). Use lucide-react icons (Trophy for header, Plus for add, Upload for CSV, List for topics).
  </action>

  Props:
  ```typescript
  interface EntrantListProps {
    entrants: Array<{ id: string; name: string; seedPosition: number }>
    onReorder: (entrants: Array<{ id: string; name: string; seedPosition: number }>) => void
    onRemove: (id: string) => void
    onEdit: (id: string, newName: string) => void
    disabled?: boolean  // true when bracket is not draft
  }
  ```

  Features:
  - Vertical list of entrants with seed number, name, and action buttons
  - Native HTML5 drag-and-drop for reorder (draggable attribute + onDragStart/onDragOver/onDragEnd)
  - Drag handle icon (GripVertical from lucide-react) on the left
  - Seed position number displayed as monospace badge
  - Entrant name (editable: double-click or pencil icon to enter edit mode, inline text input)
  - Remove button (X icon or Trash icon) on the right
  - Move up/down arrow buttons for accessibility and mobile (ChevronUp, ChevronDown from lucide-react)
  - On reorder/move: recalculate seedPosition for all entrants (1-indexed based on position)
  - When `disabled` prop is true, hide all edit/remove/reorder controls
  - Empty state: "No entrants added yet" message

  Follow the drag pattern from the research doc exactly (useState for dragIndex, handleDragStart, handleDragOver with preventDefault and splice, handleDragEnd to clear).

  Props:

  ```typescript
  interface CSVUploadProps {

  ```

  Features:
  - Hidden file input (`<input type="file" accept=".csv" />`) triggered by a styled button
  - Upload button with Upload icon: "Upload CSV"
  - On file select: call `parseEntrantCSV(file)` from `@/lib/bracket/csv-parser`

  - "Cancel" to discard and return to empty state
  - Error display for parse failures
  - Help text: "Upload a CSV file with a 'name' column. One entrant per row."
