  <action>
  Create a 'use client' React component at src/components/bracket/bracket-diagram.tsx.

  **Props interface:**
  ```typescript
  interface BracketDiagramProps {
    matchups: MatchupData[]
    totalRounds: number
    className?: string
  }
  ```

  **Layout constants:**
  ```typescript
  const MATCH_WIDTH = 200
  const MATCH_HEIGHT = 80
  const ROUND_GAP = 60       // horizontal gap between rounds
  const MATCH_V_GAP = 20     // vertical gap between matches in same round
  const PADDING = 20         // padding around the diagram
  ```

  **Position calculation function:**
  ```typescript
  function getMatchPosition(round: number, position: number, totalRounds: number): { x: number; y: number }
  ```
  - x = PADDING + (round - 1) * (MATCH_WIDTH + ROUND_GAP)
  - y calculation: Each successive round doubles the vertical spacing to center matchups between their feeder matchups
    - For round 1: evenly spaced, y = PADDING + (position - 1) * (MATCH_HEIGHT + MATCH_V_GAP)
    - For round 2+: center between the two feeder matchups from the previous round
    - Formula: Calculate the total height of round 1 first. For later rounds, the y position is the midpoint of the two feeder matchup positions.

  **SVG structure:**
  1. Outer scrollable container: `<div className="overflow-x-auto">` wrapping the SVG
  2. SVG element with computed width and height
  3. Round labels at the top of each column ("Round 1", "Quarterfinals", "Semifinals", "Final")
  4. For each matchup, render a `<g>` group at the computed position containing:
     - Rectangle (matchup box): rounded corners, themed with Tailwind CSS variables (fill: card bg, stroke: border)
     - Top entrant name (or "TBD"): text at y=28 inside the box
     - Divider line at MATCH_HEIGHT/2
     - Bottom entrant name (or "TBD"): text at y=60 inside the box
     - If winner is set: bold/highlight the winning entrant name, subtle background for winner half
  5. Connector lines between rounds: From the right edge of a matchup to the left edge of its nextMatchup
     - Use SVG `<path>` with horizontal-then-vertical-then-horizontal segments (bracket-style connector)
     - Path: move right from source center-right, then vertical to target center-left, then horizontal to target

  **Round label mapping:**
  - For totalRounds=2: ["Semifinals", "Final"]
  - For totalRounds=3: ["Quarterfinals", "Semifinals", "Final"]
  - For totalRounds=4: ["Round 1", "Quarterfinals", "Semifinals", "Final"]

  **Styling notes:**
  - Use CSS custom properties from the Tailwind theme: `hsl(var(--card))` for fill, `hsl(var(--border))` for strokes
  - Since SVG doesn't support Tailwind classes for fill/stroke directly, use inline styles referencing CSS variables
  - Text uses `hsl(var(--foreground))` for names, `hsl(var(--muted-foreground))` for TBD
  - Winner text gets `font-weight: bold` and optionally a subtle background highlight
  - The component wraps in a scrollable div for mobile support (brackets wider than viewport)

  **Edge cases:**
  - Empty matchups (no entrants assigned yet) show "TBD" for both slots
  - Matchups with only one entrant show name + "TBD"
  - Connector lines only render for matchups that have a nextMatchupId
