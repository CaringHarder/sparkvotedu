**Create `src/components/shared/card-context-menu.tsx`:**

Build a unified context menu using Radix DropdownMenu primitives from `@/components/ui/dropdown-menu`. This component will be consumed by both bracket-card.tsx and poll-card.tsx in Plan 02.

Props interface:
```typescript
interface CardContextMenuProps {
  itemId: string
  itemName: string
  itemType: 'poll' | 'bracket'
  status: string              // draft, active, closed, completed, archived
  editHref: string            // URL for the Edit action (e.g., /polls/{id} or /brackets/{id})
  sessionCode?: string | null // For Share/Copy Link -- only available on active brackets with sessions
  onRename: (newName: string) => void
  onDuplicated?: (newId: string) => void
  onArchived?: () => void
  onDeleted?: () => void
}
```

Implementation details:
- Wrap in `'use client'` directive
- Use `DropdownMenu`, `DropdownMenuTrigger`, `DropdownMenuContent`, `DropdownMenuItem`, `DropdownMenuSeparator` from `@/components/ui/dropdown-menu`
- Trigger button: `<MoreVertical className="h-4 w-4" />` wrapped in a button with `onClick={(e) => { e.stopPropagation(); e.preventDefault() }}` to prevent card navigation (per user decision and existing session-card-menu.tsx pattern)
- Trigger styling: `"rounded-md p-1 text-muted-foreground hover:bg-accent hover:text-foreground"` (matches existing bracket card triple-dot exactly)
- Include `<span className="sr-only">{itemType === 'poll' ? 'Poll' : 'Bracket'} options</span>` for accessibility

Menu items in order (per locked decision: Rename > Edit > Share > Duplicate > Archive > Delete):

1. **Rename** (`Pencil` icon) -- Triggers inline rename mode on the card. When selected, calls a callback that the parent card component handles (switching to an inline input). Uses `onSelect` handler that stops propagation and calls a local `handleRename` which opens a controlled state. For MVP, trigger an inline rename by calling the `onRename` callback with a window.prompt -- but actually, use a state-driven approach: when Rename is clicked, set `isRenaming=true`, render an inline input inside the menu trigger area. Actually, the simplest and most robust approach: use `onSelect` to set a local `isRenaming` state, then render a small controlled dialog/popover with an input field for the new name. But this adds complexity. Instead, expose `onStartRename` callback prop and let the parent card handle the inline edit UI (following editable-session-name.tsx pattern). Change the prop from `onRename` to `onStartRename: () => void`.

2. **Edit** (`SquarePen` icon) -- Uses `useRouter().push(editHref)`. Stop propagation.

3. **Share/Copy Link** (`Link` icon) -- Copy the join URL or poll URL to clipboard. For active brackets with sessionCode, copy `${origin}/join/${sessionCode}`. For polls and non-active brackets, copy `${origin}${editHref}`. Show "Copied!" text for 2 seconds using local state (matching existing bracket card pattern). No toast -- per user decision "no success toasts for any action."

4. **Duplicate** (`Copy` icon) -- Call the appropriate server action (`duplicateBracket` or `duplicatePoll`) via `useTransition`. On success, call `onDuplicated?.(newId)`.

5. `<DropdownMenuSeparator />` -- visual break before destructive actions

6. **Archive** (`Archive` icon) -- Call `archiveBracket` or `archivePoll` via `useTransition`. On success, call `onArchived?.()`. No confirmation dialog (per user decision: "Archive happens immediately with no confirmation").

7. **Delete** (`Trash2` icon, destructive variant) -- Opens DeleteConfirmDialog. Use `variant="destructive"` on the DropdownMenuItem if the shadcn wrapper supports it; otherwise apply red text manually.

Use `useTransition` for all async operations. Show `isPending` state by disabling menu items during operations.

DropdownMenuContent should use `align="end"` for right-alignment (matching session-card-menu.tsx).
