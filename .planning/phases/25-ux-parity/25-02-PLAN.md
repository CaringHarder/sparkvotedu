**Add AnimatePresence to card lists:**

For both the brackets page and the refactored polls page:

Create client-side wrapper components (`BracketCardList` and `PollCardList`) that manage local `items` state with `useState`, initialized from the server-provided data.

Pattern for AnimatePresence (from `motion/react` -- NOT `framer-motion`):
```tsx
import { motion, AnimatePresence } from 'motion/react'

// Wrap the grid:
<div className="grid grid-cols-1 gap-4 md:grid-cols-2 lg:grid-cols-3">
  <AnimatePresence mode="popLayout">
    {items.map(item => (
      <motion.div
        key={item.id}
        layout
        initial={{ opacity: 0, scale: 0.95 }}
        animate={{ opacity: 1, scale: 1 }}
        exit={exitAnimation}  // dynamic based on removal type
        transition={{ duration: 0.25 }}
      >
        <BracketCard bracket={item} onRemoved={(type) => handleRemove(item.id, type)} />
      </motion.div>
    ))}
  </AnimatePresence>
</div>
```

Card removal animations (per locked user decisions):
- **Delete:** `exit={{ opacity: 0 }}` with `duration: 0.25` -- fade out
- **Archive:** `exit={{ opacity: 0, x: -100 }}` with `duration: 0.3` -- slide left
- **Duplicate:** New card appears with `initial={{ opacity: 0, scale: 0.9 }}` and a brief ring highlight animation using `animate={{ boxShadow: ['0 0 0 0px rgba(59, 130, 246, 0.5)', '0 0 0 4px rgba(59, 130, 246, 0)', '0 0 0 0px rgba(59, 130, 246, 0)'] }}` with `transition: { duration: 0.6 }`

The simplest approach: track a `removalType` state per item (null, 'delete', 'archive') and choose exit animation dynamically. When CardContextMenu callbacks fire:
1. Set the item's `removalType` in local state
2. Remove the item from the `items` array (triggers AnimatePresence exit)
3. After animation completes, call `router.refresh()` to sync server state

For duplicate: after the server action returns the new item ID, call `router.refresh()` which will add the new card to the server data. The AnimatePresence `initial` animation handles the entrance.

Keep it simple -- if the animation state management becomes too complex, fall back to just `router.refresh()` with basic fade animations for all removals. The user wants feedback but not at the cost of reliability.

  <action>
Convert the SignOutButton from a form action pattern to a `useTransition` pattern for immediate pending state feedback.

Current implementation:
```tsx
<form action={signOut}>
  <Button type="submit" variant="ghost" size="sm">Sign Out</Button>
</form>
```

New implementation:
```tsx
'use client'

import { useTransition } from 'react'
import { signOut } from '@/actions/auth'
import { Button } from '@/components/ui/button'

export function SignOutButton() {
  const [isPending, startTransition] = useTransition()

  function handleSignOut() {
    startTransition(async () => {
      await signOut()
    })
  }

  return (
    <Button
      type="button"
      variant="ghost"
      size="sm"
      onClick={handleSignOut}
      disabled={isPending}
      className={isPending ? 'opacity-50' : ''}
    >
      {isPending ? 'Signing out...' : 'Sign Out'}
    </Button>
  )
}
```

Per locked user decisions:
- Button text changes to "Signing out..." on click
- Button becomes disabled AND visually dimmed (opacity-50 matches the project's Button base disabled:opacity-50 pattern)
- Prevents double-click during processing (disabled attribute)
- After sign-out completes: `signOut()` already calls `redirect('/login')` in the server action, so no additional redirect logic needed

The component is already a client component (`'use client'`), already imports `signOut` and `Button`. The only change is removing the `<form>` wrapper and adding `useTransition`.

Note: The `signOut` server action in `src/actions/auth.ts` calls `redirect('/login')` after `supabase.auth.signOut()`. Next.js `redirect()` in server actions throws a special error that `useTransition` handles correctly -- the redirect still works. No changes needed to the auth action itself.
