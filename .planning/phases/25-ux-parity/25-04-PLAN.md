<objective>
Create archive views for brackets and polls so teachers can see, recover, and permanently delete archived items.

Purpose: When a teacher archives a bracket or poll, it disappears from the main list (correct behavior), but there is currently no page to view archived items. Teachers need a way to find and recover archived brackets and polls. This follows the exact pattern established by the sessions archived page at `src/app/(dashboard)/sessions/archived/`.

Output: Two new archive pages (/brackets/archived, /polls/archived) with recover and permanent delete functionality, plus DAL functions and server actions to support unarchive transitions.
</objective>

**DAL layer -- bracket.ts:**

1. Add `getArchivedBracketsDAL(teacherId: string)` function that queries `prisma.bracket.findMany` with `where: { teacherId, status: 'archived' }`, `orderBy: { updatedAt: 'desc' }`, and `include: { _count: { select: { entrants: true } }, session: { select: { id: true, code: true } } }`. Place it near `getTeacherBrackets`.

2. Add `unarchiveBracketDAL(bracketId: string, teacherId: string)` function. This should:
   - Find the bracket with `prisma.bracket.findFirst({ where: { id: bracketId, teacherId, status: 'archived' } })`
   - If not found, return `{ error: 'Bracket not found or not archived' }`
   - Transition to 'completed' status (safest state after unarchive -- bracket was live at some point before archiving, and 'completed' is a valid terminal state that doesn't reactivate voting)
   - Use `prisma.bracket.update({ where: { id: bracketId }, data: { status: 'completed' } })`
   - Return the updated bracket
   - NOTE: Do NOT modify VALID_TRANSITIONS -- unarchive bypasses the transition map intentionally (like sessions do). The function handles its own validation by checking `status: 'archived'` in the where clause.

3. Add `deleteBracketPermanentlyDAL(bracketId: string, teacherId: string)` function:
   - Find with `prisma.bracket.findFirst({ where: { id: bracketId, teacherId, status: 'archived' } })`
   - If not found, return `{ error: 'Bracket not found or not archived' }`
   - Delete with `prisma.bracket.delete({ where: { id: bracketId } })` (cascade deletes entrants, matchups, votes, predictions via Prisma schema)
   - Return `{ success: true }`

**DAL layer -- poll.ts:**

1. Add `getArchivedPollsDAL(teacherId: string)` function that queries `prisma.poll.findMany` with `where: { teacherId, status: 'archived' }`, `orderBy: { updatedAt: 'desc' }`, and `include: { _count: { select: { votes: true } } }`.

2. Add `unarchivePollDAL(pollId: string, teacherId: string)` function:
   - Find with `prisma.poll.findFirst({ where: { id: pollId, teacherId, status: 'archived' } })`
   - If not found, return `{ error: 'Poll not found or not archived' }`
   - Transition to 'closed' status (equivalent to completed -- polls use closed as their terminal state)
   - Use `prisma.poll.update({ where: { id: pollId }, data: { status: 'closed' } })`
   - Return the updated poll
   - NOTE: Do NOT modify VALID_POLL_TRANSITIONS -- unarchive bypasses the map.

3. Add `deletePollPermanentlyDAL(pollId: string, teacherId: string)` function:
   - Find with `prisma.poll.findFirst({ where: { id: pollId, teacherId, status: 'archived' } })`
   - If not found, return `{ error: 'Poll not found or not archived' }`
   - Delete with `prisma.poll.delete({ where: { id: pollId } })` (cascade deletes options, votes via schema)
   - Return `{ success: true }`
