Content layout (per locked decisions):

   - Normal mode: "We sent a verification link to **{email}**" (show full email, bold)

   - Button text: "Resend verification email" when idle
   - Button text: "Sending..." when isPending
   - Button text: "Resend in {N}s" when cooldown > 0
   - Button disabled when isPending OR cooldown > 0
   - On successful resend (state.success), set cooldown to 60
   - On error, show error message in red text
4. **Success feedback** -- when resend succeeds, show "Verification email sent!" in green/muted text

Since we need just Google here, create a standalone Google sign-in button that calls `supabase.auth.signInWithOAuth({ provider: 'google' })` from the client-side Supabase client. 7. **"Sign out" link** -- a button/link that calls the signOut action, styled as a subtle text link at the bottom: "Sign out" in muted text

Styling:
- Match existing auth page styling (text-sm, muted-foreground, etc.)
- Email address displayed in font-medium
- Use the same Tailwind patterns as login-form.tsx and signup-form.tsx
- Resend button: full width, primary style (matching "Sign In" button pattern)
- Google button: variant="outline" full width (matching OAuthButtons)
- Sign out: text-sm text-muted-foreground hover:text-foreground cursor-pointer, centered

Cooldown timer implementation:
```typescript
const [cooldown, setCooldown] = useState(0)

useEffect(() => {
  if (cooldown <= 0) return
  const timer = setInterval(() => {
    setCooldown((prev) => {
      if (prev <= 1) return 0
      return prev - 1
    })
  }, 1000)
  return () => clearInterval(timer)
}, [cooldown])

// After successful resend:
useEffect(() => {
  if (state?.success) setCooldown(60)
}, [state?.success, state?.sentAt])
```

Important: The page needs 'use client' on the card component (not the page). The page is a server component that reads searchParams and passes them down.
