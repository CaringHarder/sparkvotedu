  <action>
In the student bracket page, the `loadBracket` function at lines 148-150 routes ALL completed brackets to `setState({ type: 'completed', bracket })`. This causes the "Completed bracket" rendering branch (lines 256-300) to render PredictiveBracket for predictive brackets, which shows "Predictions are closed" because predictionStatus is not 'predictions_open'.

The fix: Before the generic completed routing at line 148, add a special case for completed auto-mode predictive brackets. If `bracketData.status === 'completed' && bracketData.bracketType === 'predictive' && bracketData.predictiveResolutionMode === 'auto'`, route to the 'ready' state instead:

```typescript
if (bracketData.status === 'completed') {
  // Auto-mode predictive brackets: route through PredictiveStudentView for PredictionReveal (podium celebration)
  if (bracketData.bracketType === 'predictive' && bracketData.predictiveResolutionMode === 'auto') {
    setState({
      type: 'ready',
      bracket,
      participantId,
      initialVotes: {},
    })
    return
  }
  setState({ type: 'completed', bracket })
  return
}
```

This routes the completed auto-mode predictive bracket into the 'ready' state, which renders PredictiveStudentView (line 307-317). PredictiveStudentView then computes `effectiveStatus` from the bracket's predictionStatus. For a completed bracket, effectiveStatus will be 'completed', which hits the auto-mode revealing/completed branch at line 774 and renders PredictionReveal -- which has the podium celebration.

Note: The participantId is needed here but the current code only resolves participantId above the completed check. The participantId variable IS already available at that scope (declared at line 98, populated at lines 99-104). However, for the 'completed' -> 'ready' routing we need participantId to be non-null. The existing flow already returns early at line 109 if participantId is null, so by line 148 participantId is guaranteed non-null. Use a non-null assertion or the existing variable directly.

IMPORTANT: Keep the existing completed routing for all other bracket types (SE, DE, RR) unchanged. Only auto-mode predictive brackets get the special routing.
