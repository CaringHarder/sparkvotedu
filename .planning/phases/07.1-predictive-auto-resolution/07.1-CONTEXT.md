# Phase 7.1: Predictive Auto-Resolution Mode - Context

**Gathered:** 2026-02-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a third predictive bracket resolution mode where student predictions automatically count as votes, resolving the entire bracket without voting rounds. Teacher closes predictions, reviews/adjusts results, then releases them round-by-round to students. Prediction leaderboard scores accuracy against the prediction-derived outcomes.

This extends BRKT-05 (Predictive brackets). The existing manual and vote-based resolution modes are unchanged.

</domain>

<decisions>
## Implementation Decisions

### Resolution Reveal Experience
- Round-by-round progressive reveal, teacher-paced (teacher clicks "Reveal Next Round" to advance)
- 3-2-1 countdown animation between rounds while waiting for teacher to advance
- Student view has two toggleable tabs: Bracket and Leaderboard
- Default tab is Leaderboard when reveal starts
- Bracket diagram tab fills in progressively with winners as rounds are revealed
- Color-coded picks on bracket diagram: green highlight for correct predictions, red for wrong
- Wrong picks show the student's predicted winner with a strikethrough next to the actual winner
- "X/Y correct" counter displayed at top of bracket diagram tab
- Leaderboard updates live after each round is revealed (scores and ranks change in real time)
- Animated rank changes on leaderboard (names slide up/down as scores update each round)
- Finale is leaderboard-first (skip champion fanfare, prediction accuracy is the real payoff)
- Top 3 predictors celebration: gold/silver/bronze podium reveal after all rounds complete
- Presentation mode available for teacher to project bracket or leaderboard for classroom display
- Teacher dashboard has both tabs (Bracket + Leaderboard) plus "Reveal Next Round" button
- Release is committed -- once teacher clicks Release Results, reveal proceeds and cannot be undone

### Tie-Breaking & Vote Tabulation
- Predictions are tabulated as votes: majority pick wins each matchup
- Exact ties (50/50 split): teacher breaks the tie
- No predictions on a matchup (later rounds with sparse predictions): teacher decides
- Tie resolution happens round-by-round (not all at once) -- Round 1 ties first, then Round 2 (which may be affected by Round 1 decisions), etc.
- Prediction vote counts are never shown to students -- only the outcome matters

### Teacher Control Flow
- After predictions close, teacher clicks "Prepare Results" to trigger tabulation (explicit action, not auto)
- Teacher can reopen predictions before releasing results (safety net for premature close)
- Full preview of auto-resolved bracket shown to teacher before release
- Preview shows winners only (no prediction percentages/statistics)
- Teacher can override ANY matchup winner in preview, not just ties
- "Release Results" button triggers confirmation dialog: "Are you sure? This will begin the reveal."
- After full reveal, teacher manually clicks "Complete Bracket" to finalize (matches other bracket types)

### Student Prediction Accuracy Feedback
- Leaderboard shows rank + total points only (no per-round breakdown for students)
- Leaderboard displays top 10 students plus the viewing student's own rank if outside top 10
- No separate personal summary card -- leaderboard position is sufficient
- Top 3 predictors get gold/silver/bronze podium celebration after final reveal

### Claude's Discretion
- Whether students can revisit completed bracket and leaderboard after session (follow existing bracket completion patterns)
- Exact countdown animation implementation between rounds
- Presentation mode implementation details (reuse existing poll presentation mode patterns)
- Leaderboard rank animation implementation approach
- How the "Prepare Results" tabulation is communicated (loading state, progress indication)

</decisions>

<specifics>
## Specific Ideas

- Teacher controls pacing verbally in classroom while clicking "Reveal Next Round" -- the countdown between rounds builds anticipation
- Leaderboard as default tab puts the student competition front and center -- bracket diagram is available for those who want to see the visual
- Top 3 podium celebration gives more students recognition (Olympic vibe) rather than just a single winner
- The flow is: Open Predictions -> Close Predictions -> Prepare Results -> (Break ties/Override) -> Release Results -> Reveal Round 1 -> ... -> Reveal Final -> Celebration -> Complete Bracket

</specifics>
