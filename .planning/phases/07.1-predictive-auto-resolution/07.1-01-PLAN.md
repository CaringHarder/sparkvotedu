<feature>
  <name>Prediction Tabulation Engine</name>
  <files>
    src/lib/bracket/predictive.ts
    src/lib/bracket/__tests__/tabulate-predictions.test.ts
  </files>
  <behavior>
    tabulatePredictions takes predictions array, matchups array, and totalRounds.
    Returns TabulationResult[] sorted by round then position.

    Cases:
    - 4-entrant bracket, 3 students all pick same winners -> all 'resolved' with clear majority
    - 4-entrant bracket, 2v2 exact tie -> status 'tie', winnerId null
    - 4-entrant bracket, matchup with 0 predictions -> status 'no_predictions', winnerId null
    - 8-entrant bracket, Round 1 winners propagate to Round 2 entrants before Round 2 tabulation
    - Bye matchups in input -> excluded from output (filtered by isBye flag)
    - Round 2 prediction for eliminated entrant -> prediction ignored (counts only for actual entrants in that round matchup)
    - Empty predictions array -> empty results
    - Teacher override scenario: after override of Round 1 winner, Round 2 matchup entrants change (tested via re-tabulation)
  </behavior>
  <implementation>
    Follow the scorePredictions pattern in predictive.ts.

    Interface TabulationInput:
      matchupId: string, round: number, position: number, entrant1Id: string, entrant2Id: string, isBye: boolean, nextMatchupId: string | null

    Interface TabulationResult:
      matchupId: string, round: number, position: number, winnerId: string | null,
      entrant1Id: string, entrant2Id: string, entrant1Votes: number, entrant2Votes: number,
      totalVotes: number, status: 'resolved' | 'tie' | 'no_predictions'

    Algorithm:
    1. Group predictions by matchupId into Map<matchupId, Map<entrantId, count>>
    2. For each round (1 to totalRounds):
       a. Get non-bye matchups for this round, sorted by position
       b. For each matchup, count votes for entrant1Id and entrant2Id
       c. Majority -> resolved, equal -> tie, zero -> no_predictions
       d. If resolved, propagate winnerId to next matchup (odd position -> entrant1, even -> entrant2)
    3. Return all results sorted by round, position

    Export tabulatePredictions from predictive.ts alongside existing scorePredictions.
  </implementation>
</feature>
