## Decisions Made

1. **Inline SVG badges over separate SVG overlay**: Moving accuracy rendering inside BracketDiagram's own SVG eliminates coordinate misalignment when regions normalize matchup positions (the root cause of the bug). The badges share the exact same position coordinates as MatchupBox.

2. **accuracyMap as Record<string, union>**: Simple string-keyed record type propagates cleanly through RegionBracketView -> BracketDiagram without requiring position data or entrant names. BracketDiagram filters internally (matchup IDs not in the map return null).

3. **>= 32 conditional matching AdvancedVotingView**: Uses the identical `(bracket.maxEntrants ?? bracket.size) >= 32` pattern from the SE student voting view for consistency across all student-facing bracket rendering.
