<objective>
Fix 32+ entrant predictive bracket layout and accuracy badge alignment on the student reveal Bracket tab.

Purpose: BracketAccuracyView currently renders a bare BracketDiagram for all bracket sizes, causing 32+ entrant brackets to show a single scrollable column instead of the split regional layout used by SE student views. Additionally, the accuracy SVG overlay is a separate absolute-positioned SVG on top of BracketDiagram, which misaligns when regions normalize matchup positions. Moving accuracy rendering inside BracketDiagram's own SVG (via an accuracyMap prop) fixes alignment for all layout modes.

Output: Students on 32+ entrant predictive brackets see the same regional navigation (Top/Bottom or Quadrants) as SE brackets, with accuracy badges correctly overlaying each matchup card.

a) Replace the `accuracyData` useMemo with a simpler `accuracyMap` useMemo that returns `Record<string, 'correct' | 'incorrect' | null>`:

```typescript
// Build accuracy map: matchupId -> 'correct' | 'incorrect' | null
const accuracyMap = useMemo(() => {
  const map: Record<string, 'correct' | 'incorrect' | null> = {}

  for (const matchup of bracket.matchups) {
    if (matchup.isBye) continue

    // Only show accuracy for revealed, decided matchups
    if (matchup.status !== 'decided' || !matchup.winnerId) {
      map[matchup.id] = null
      continue
    }

    // Check if matchup round is within revealed range
    if (revealedUpToRound != null && matchup.round > revealedUpToRound) {
      map[matchup.id] = null
      continue
    }

    const predicted = predictionMap.get(matchup.id)
    if (!predicted) {
      map[matchup.id] = null
      continue
    }

    map[matchup.id] = predicted === matchup.winnerId ? 'correct' : 'incorrect'
  }

  return map
}, [bracket.matchups, predictionMap, revealedUpToRound])
```

b) Remove the `svgWidth`, `svgHeight` local computations (lines 264-270) — no longer needed since the accuracy overlay is no longer a separate SVG.

c) Remove the MATCH_WIDTH, MATCH_HEIGHT, ROUND_GAP, MATCH_V_GAP, PADDING, LABEL_HEIGHT, getMatchPosition imports from bracket-diagram. Keep only `BracketDiagram` imported (it is still used for < 32 entrants).
