## Decisions Made
- LiveDashboard delegates rendering to PredictiveBracket for auto mode rather than adding auto-mode controls inline -- cleaner separation of concerns since auto mode has a fundamentally different lifecycle
- Presentation mode uses a fixed position overlay with dark background (matching existing 05-05 presentation mode pattern) rather than CSS class toggle
- Override triggers a full re-prepare (refetch tabulation results) after override to reflect downstream recalculation -- simpler than optimistic local state update
- Auto mode hides SE-style round tabs, action buttons, and completion badge in the LiveDashboard top bar since PredictiveBracket manages its own lifecycle controls
