  <name>Task 1: Add admin data queries to DAL</name>
  <files>
    src/lib/dal/admin.ts

  <action>
Add the following functions to `src/lib/dal/admin.ts` (appending to existing admin auth functions from 17-01):

1. `getAdminStats()` -- Returns summary metrics for the stat bar:
   - Total teachers count
   - Teachers active today (teachers with classSessions updated today OR brackets updated today)
   - Teachers on free tier (count where subscriptionTier = 'free')
   - Teachers on paid tiers (count where subscriptionTier != 'free')
   - Use `Promise.all` for parallel queries

2. `getTeacherList(params)` -- Paginated teacher list with search and filters:
   - Parameters: `{ search?: string, tier?: string, status?: 'active' | 'inactive', page?: number, pageSize?: number }`
   - Default page=1, pageSize=20
   - Search: case-insensitive ILIKE on name OR email (use Prisma `contains` with `mode: 'insensitive'`)
   - Tier filter: exact match on subscriptionTier
   - Status filter: "active" means teachers with at least one classSession in the last 30 days; "inactive" means no sessions in 30 days. Use a subquery or conditional where clause.
   - Return each teacher with: id, name, email, subscriptionTier, createdAt, role, plus aggregated counts:
     - `_count.brackets` (bracket count)
     - Last active date: most recent classSession.createdAt or bracket.updatedAt
   - Return `{ teachers, total, page, pageSize, totalPages }`
   - Use `prisma.$transaction` for count + findMany in single round trip

3. `getTeacherDetail(teacherId: string)` -- Full teacher detail for the slide-out panel:
   - Return teacher with:
     - Basic info: name, email, subscriptionTier, createdAt, role
     - Usage counts via `_count`: brackets, polls, classSessions
     - Total students: count of unique StudentParticipant across all their sessions
   - Use `prisma.teacher.findUnique` with includes and counts
  </action>
