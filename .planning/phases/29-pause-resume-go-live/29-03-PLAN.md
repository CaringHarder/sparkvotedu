    Create `src/components/student/paused-overlay.tsx` -- a full-screen overlay component with cooking theme per user decisions.

    **Component API:**
    ```typescript
    interface PausedOverlayProps {
      visible: boolean  // Driven by bracket.status === 'paused' or poll.status === 'paused'
    }
    ```

    **Requirements from user decisions (LOCKED):**
    - Playful and fun cooking theme -- animated bubbling pot, steam, personality-driven
    - Dimmed overlay: voting UI stays visible but dimmed/blurred underneath, overlay sits on top
    - Looping subtle animation while paused: steam rising, pot bubbling, keeps the screen alive
    - Dual messaging: big "Let it cook!" headline with smaller "Voting will resume soon" subtext
    - Visual feedback only -- NO audio or haptic
    - Overlay appears immediately on pause (no grace period)
    - Energetic reveal on resume -- quick animation (slide/burst) that signals "go go go!"

    **Implementation (Claude's discretion on exact design):**
    - Use `motion/react` (already installed v12.29.2) for AnimatePresence entry/exit and looping animations
    - Use `useReducedMotion` from motion/react to respect accessibility preferences
    - Fixed inset-0 z-50 positioning following existing overlay patterns (CelebrationScreen, WinnerReveal, CountdownOverlay)
    - Background: `bg-background/80 backdrop-blur-sm` (matches existing project overlay patterns)
    - Cooking pot: SVG illustration of a cooking pot with lid. Use inline SVG (no external assets). Style with project colors. Pot should have a subtle bobble/bounce loop.
    - Steam: 2-3 animated steam wisps rising from the pot, using motion/react animate with y translation and opacity cycles. Looping with `repeat: Infinity`.
    - Headline: "Let it cook!" in large bold text (text-3xl or text-4xl font-bold). Optional cooking emoji (at discretion).
    - Subtext: "Voting will resume soon" in smaller muted text below
    - Entry animation: fade in with slight scale up (opacity 0->1, scale 0.95->1)
    - Exit animation: energetic burst -- scale up to 1.1 with quick fade out (0.3s), creating a "burst open" feel that signals resume. Use ease-out for the burst feel.
    - If reduced motion preferred: simple opacity fade for both entry and exit

    **Important notes:**
    - This is NOT an error state -- it should feel like a fun game pause
    - The overlay is a single shared component used by both brackets and polls
    - The `visible` prop is controlled by the parent page based on status from realtime hooks
    - Do NOT add any click handlers or dismiss buttons -- only the teacher controls this
