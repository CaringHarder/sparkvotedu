   - Add `lastInitial: string | null` field to the `DuplicateCandidate` interface.

2. **Add `findReturningByFirstName` DAL function** in `src/lib/dal/student-session.ts`:
   - New export: `findReturningByFirstName(teacherId: string, firstName: string)`.
   - Queries `StudentParticipant` with: `firstName` case-insensitive match, `banned: false`, `session.teacherId` match, `session.archivedAt: null`.
   - Returns array with `{ id, funName, emoji, lastInitial, sessionId, lastSeenAt }` (note: includes `lastInitial` unlike `findReturningStudent`).
   - Order by `lastSeenAt` desc.

3. **Add `lookupStudentByFirstName` server action** in `src/actions/student.ts`:
   - New zod schema: `lookupStudentByFirstNameSchema` with `code` (6-digit string) and `firstName` (existing `firstNameSchema`). No `lastInitial` field.
   - Logic mirrors existing `lookupStudent` but:
     a. Calls `findReturningByFirstName` instead of `findReturningStudent`.
     b. Single match: returns `{ candidates: [match], session, allowNew: true }` (NOT auto-reclaim). The single match goes to confirmation card in UI.
     c. Multiple matches: dedup by `funName|emoji` as before, return candidates with `lastInitial` included. Return `allowNew: true`.
     d. Current-session match: still auto-reclaim (already in this session).
     e. Zero matches: return `{ isNew: true, session }`.
   - Candidates must include `lastInitial` in the response: `candidates.map(m => ({ id: m.id, funName: m.funName, emoji: m.emoji, lastInitial: m.lastInitial }))`.

4. **Add `setNameViewDefault` server action** in `src/actions/student.ts`:
   - Authenticated via `getAuthenticatedTeacher()`.
   - Input: `{ value: 'fun' | 'real' }` validated with zod.
   - Updates `Teacher.nameViewDefault` via Prisma.
   - Returns `{ success: true }` or `{ error: string }`.

5. **Extend `teacherUpdateStudentName` server action**:
   - Add optional `lastInitial?: string` parameter.
   - Validate with existing `lastInitialSchema` (only if provided and non-empty).
   - If `lastInitial` provided, include in Prisma update `data` alongside firstName.
   - Keep existing broadcast call unchanged.

   - Import `lookupStudentByFirstName` instead of `lookupStudent`.

   - Use `motion/react` for card entrance animation (same pattern as ReturningDisambiguation).
