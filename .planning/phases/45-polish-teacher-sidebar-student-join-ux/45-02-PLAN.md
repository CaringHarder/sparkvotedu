  <action>
1. **Create participants API route** (`src/app/api/sessions/[sessionId]/participants/route.ts`):
   - GET endpoint. Authenticate via `getAuthenticatedTeacher()`.
   - Verify teacher owns the session.
   - Return `{ participants }` array with `{ id, funName, firstName, emoji, lastInitial }`, ordered by `funName` asc.
   - Return 401 if not authenticated, 404 if session not found or not owned.

2. **Create `useRealtimeParticipants` hook** (`src/hooks/use-realtime-participants.ts`):
   - Parameters: `(sessionId: string, initialParticipants: ParticipationSidebarProps['participants'])`.
   - Returns: `{ participants, newParticipantIds }`.
   - Maintains `participants` in useState, initialized from `initialParticipants`.
   - Maintains `newParticipantIds` Set state for tracking newly joined (for highlight animation).
   - Subscribes to Supabase broadcast channel `activities:{sessionId}`.
   - On `participant_joined` event: fetch `/api/sessions/{sessionId}/participants`, update state.
   - Compare new participants list with previous: any new IDs get added to `newParticipantIds`.
   - After 2 seconds, clear the ID from `newParticipantIds` (highlight pulse duration).
   - Use `useRef` for the fetch function to avoid stale closure issues (per RESEARCH pitfall #2).
   - Cleanup: unsubscribe on unmount.
   - **IMPORTANT:** Do NOT duplicate channel subscription if the page already subscribes to `activities:{sessionId}`. Check existing subscription patterns in `useRealtimePoll` and `useRealtimeBracket`. If those hooks already subscribe, add the participant refetch handler to the same channel subscription rather than creating a separate one. If creating a new subscription is cleaner, use a unique channel name or ensure no conflicts.

3. **Update `ParticipationSidebar`**:
   - Add optional `newParticipantIds?: Set<string>` prop.
   - For participant tiles in `newParticipantIds`: apply a brief green/highlight pulse CSS animation class (`animate-pulse-once` or use `motion/react` `AnimatePresence` with `layoutId`).
   - Add disconnect fade-out: Accept optional `disconnectTimestamps?: Map<string, number>` prop. For participants disconnected > 45 seconds, apply `opacity-30 transition-opacity duration-2000` class. For participants disconnected > 60 seconds, hide from list entirely (filter out in render).

4. **Wire poll live client** (`src/app/(dashboard)/polls/[pollId]/live/client.tsx`):
   - Import and use `useRealtimeParticipants(sessionId, participants)`.
   - Pass returned `participants` and `newParticipantIds` to `ParticipationSidebar`.
   - Avoid duplicate subscription: The poll live page already subscribes via `useRealtimePoll`. Either extend `useRealtimePoll` to expose a participant refetch callback, OR have `useRealtimeParticipants` check if the channel is already active. The cleanest approach: `useRealtimeParticipants` creates its own subscription since it targets a different concern (participants vs poll state). Supabase client handles multiple subscriptions to the same channel gracefully.

5. **Wire bracket live dashboard** (`src/components/teacher/live-dashboard.tsx`):
   - Import and use `useRealtimeParticipants(sessionId, participants)` where participants come from the bracket live page props.
   - Pass returned `participants` and `newParticipantIds` to `ParticipationSidebar`.
   - `sessionId` can be extracted from bracket data (bracket.session.id or similar -- check existing code).
