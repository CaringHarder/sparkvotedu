```typescript
const bracketDone = isDoubleElim
  ? deBracketDone

```

This SE-centric logic (currentRound/totalRounds/allRoundDecided) never evaluates true for RR brackets because RR matchups don't follow the SE round structure. Add an RR-specific condition.

Replace with:
```typescript
const rrAllDecided = isRoundRobin
  ? currentMatchups.length > 0 && currentMatchups.every((m) => m.status === 'decided')

  ? deBracketDone
  : isRoundRobin
```

This ensures the action bar shows "Bracket Complete!" for RR brackets when all matchups are decided, matching the behavior for SE and DE brackets.

The student poll page has a bug where the countdown animation never renders. At lines 89-90 inside the useEffect detecting active->closed transition:

```typescript
setShowCountdown(true)
setClosedDetected(true)
```

Both state variables are set simultaneously. On the next render, the early return guard at line 310:
```typescript
if (closedDetected && !showReveal) {
```
fires immediately, returning the static "This poll has been closed" div BEFORE the `showCountdown` check at line 345 can render the WinnerReveal overlay.

Fix: Add `!showCountdown` to the early return guard condition. This ensures that while the countdown is playing, the guard does not fire. Once the countdown completes (`handleCountdownComplete` sets `showCountdown=false, showReveal=true`), the guard still works correctly for the post-reveal state.

Change line 310 from:
```typescript
if (closedDetected && !showReveal) {
```
to:
```typescript
if (closedDetected && !showReveal && !showCountdown) {
```

This way:
1. When poll closes: `showCountdown=true, closedDetected=true` -- guard is `true && true && false` = false -- falls through to render voting UI + WinnerReveal overlay
2. When countdown completes: `showCountdown=false, showReveal=true` -- guard is `true && false && true` = false -- falls through to render PollReveal overlay
3. When reveal dismissed: `showCountdown=false, showReveal=false, closedDetected=true` -- guard is `true && true && true` = true -- shows static "closed" state

This is the minimal fix. The voting UI content underneath is harmless during the countdown because WinnerReveal is a `fixed inset-0 z-50` overlay.
