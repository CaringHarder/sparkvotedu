**IMPORTANT DETAIL on voting mechanism:** `MatchupVoteCard` uses the `useVote` hook internally which calls `castVote` server action. This is the same server action that `RRLiveView` uses via `handleStudentVote` -> `castVote`. However, `MatchupVoteCard` tracks its own vote state via `useVote`, so the parent `votedMatchups` state won't automatically update. To fix this, also call `onStudentVote` from `RRSimpleVoting` when a vote is submitted, by wrapping the `onVoteSubmitted` callback:

```tsx
onVoteSubmitted={() => {
  // Update parent's votedMatchups state so the matchup is tracked as voted
  onStudentVote(currentMatchup.id, 'voted') // The actual entrant ID doesn't matter for tracking
  setShowConfirmation(true)
  setTimeout(() => {
    setShowConfirmation(false)
    setCurrentIndex((i) => i + 1)
  }, 1200)
}}
```

Wait -- actually there is a conflict here. `MatchupVoteCard` uses `useVote` which calls `castVote` internally. But `handleStudentVote` in RRLiveView ALSO calls `castVote`. This would result in a double vote submission.

**Resolution:** In `RRSimpleVoting`, do NOT call `onStudentVote` for the server action. Instead, just update the parent's `votedMatchups` state to track that the matchup was voted on. The simplest approach: pass a separate `onVoteTracked` callback that only updates state without calling the server:

In RRLiveView, add:
```typescript
const handleVoteTracked = useCallback((matchupId: string, entrantId: string) => {
  setVotedMatchups((prev) => ({ ...prev, [matchupId]: entrantId }))
}, [])
```

Pass `onVoteTracked={handleVoteTracked}` to `RRSimpleVoting` instead of `onStudentVote`.
