<objective>
Add dual-channel broadcast to bracket and predictive bracket status changes so the student dashboard auto-updates when brackets are activated or completed.

Purpose: Brackets (round robin and predictive) currently require a manual page refresh to appear on the student dashboard because `updateBracketStatus` and `updatePredictionStatus` only write to the database -- they never call `broadcastActivityUpdate(sessionId)` to notify the `activities:{sessionId}` channel that `useRealtimeActivities` subscribes to. Polls already do this correctly (Phase 21 dual-channel pattern). This plan patches brackets to follow the same pattern.

Output: Two modified server action files with `broadcastActivityUpdate` calls after status changes.
</objective>

```typescript
// Dual-channel broadcast: notify session activity channel (Phase 21 pattern)
// This enables useRealtimeActivities on student dashboard to auto-update
if (['active', 'completed'].includes(parsed.data.status)) {
  const bracket = await prisma.bracket.findUnique({
    where: { id: parsed.data.bracketId },
    select: { sessionId: true },
  })

  }
}
```
