<objective>
Create a dedicated presentation-optimized results view for ranked polls that is readable on classroom projectors from 30+ feet away.

Purpose: Teachers project ranked poll results for the whole class to see. The current RankedLeaderboard uses text-sm which is unreadable on projectors. A dedicated PresentationResults component with large text and high-contrast medal colors on a dark background solves this.

Output: New PresentationResults component, enhanced PresentationMode wrapper
</objective>

<task type="auto">
  <name>Task 1: Create PresentationResults component with projector-optimized medal cards</name>
  <files>src/components/poll/presentation-results.tsx</files>
  <action>
Create a new `PresentationResults` component at `src/components/poll/presentation-results.tsx` that renders ranked poll results optimized for projector display on a dark background.

**Props:** Same as RankedLeaderboard: `options: PollOptionData[]`, `bordaScores: BordaLeaderboardEntry[]`, `totalVoters: number`.

**Medal color scheme (WCAG AAA on dark bg #0a0a0f):**
- Gold (1st): `bg-amber-400 text-amber-950 border-amber-300` -- bright gold with dark text
- Silver (2nd): `bg-gray-300 text-gray-900 border-gray-200` -- bright silver with dark text
- Bronze (3rd): `bg-orange-400 text-orange-950 border-orange-300` -- bright orange with dark text
- Rest (4th+): `bg-white/10 text-white/90 border-white/20` -- subtle on dark background

**Text sizes for projector readability:**
- Position number: `text-3xl` in a `h-14 w-14` circle
- Option name: `text-2xl font-bold` (minimum, `text-3xl` for 1st place)
- Points: `text-xl font-bold tabular-nums`
- Meta info (voter count): `text-sm text-white/60`
- Points bar: `h-3` (thicker than standard `h-2`)

**Layout:**
- `space-y-4` between entries (more spacing than standard `space-y-2`)
- Medal cards: `p-5 rounded-xl border-2` (larger padding, rounded corners, thicker border)
- Non-medal cards: `p-4 rounded-lg border`
- No Framer Motion animations needed in presentation mode (simplicity for projector rendering)

**Empty state:** "No votes yet" in `text-xl text-white/60`

Import types from `@/lib/poll/types` for `PollOptionData` and from `@/lib/poll/borda` for `BordaLeaderboardEntry`. Follow the same data merge pattern as RankedLeaderboard (merge bordaScores with option labels, append unscored options at bottom).

Mark as `'use client'` since it will be rendered inside PresentationMode (a client component).
  </action>
  <verify>
Run `npx tsc --noEmit` to confirm no type errors. Verify the file exists and exports PresentationResults.
  </verify>
  <done>
PresentationResults component exists with high-contrast medal styling, large text sizes (2xl+ for names, 3xl+ for position numbers), and dark-background-optimized color scheme. Types compile without errors.
  </done>
