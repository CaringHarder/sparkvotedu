<objective>
Create a dedicated presentation-optimized results view for ranked polls that is readable on classroom projectors from 30+ feet away.

Purpose: Teachers project ranked poll results for the whole class to see. The current RankedLeaderboard uses text-sm which is unreadable on projectors. A dedicated PresentationResults component with large text and high-contrast medal colors on a dark background solves this.

Output: New PresentationResults component, enhanced PresentationMode wrapper, PollResults integration
</objective>

**Medal color scheme (WCAG AAA on dark bg #0a0a0f):**
- Gold (1st): `bg-amber-400 text-amber-950 border-amber-300` -- bright gold with dark text

Import types from `@/lib/poll/types` for `PollOptionData` and from `@/lib/poll/borda` for `BordaLeaderboardEntry`. Follow the same data merge pattern as RankedLeaderboard (merge bordaScores with option labels, append unscored options at bottom).

**CRITICAL DATA FLOW:** The `bordaScores` data is only available inside `PollResults` via the `useRealtimePoll` hook. `PollLiveClient` does NOT have access to bordaScores. Therefore, the PresentationMode + PresentationResults rendering MUST happen inside `PollResults`, not in `PollLiveClient`.

**In `src/components/poll/poll-results.tsx`:**

1. Add new props to `PollResultsProps`:
   - `presenting?: boolean` -- whether presentation mode is active
   - `pollTitle?: string` -- the poll question for the PresentationMode title header
   - `onExitPresentation?: () => void` -- callback to exit presentation mode

2. Import `PresentationResults` from `@/components/poll/presentation-results`
3. Import `PresentationMode` from `@/components/poll/presentation-mode`

4. At the end of the return JSX (after the `{showReveal && ...}` block), add a conditional PresentationMode rendering:
```tsx
{presenting && (
  <PresentationMode
    title={pollTitle ?? ''}
    onExit={() => onExitPresentation?.()}
  >
    {poll.pollType === 'ranked' ? (
      <PresentationResults
        options={poll.options}
        bordaScores={leaderboardEntries}
        totalVoters={liveTotalVotes}
      />
    ) : (
      /* For simple polls, render the same chart content with CSS overrides */
      <div className="space-y-4">
        {chartType === 'bar' ? (
          <AnimatedBarChart data={chartData} total={liveTotalVotes} />
        ) : (
          <DonutChart data={chartData} total={liveTotalVotes} />
        )}
      </div>
    )}
  </PresentationMode>
)}
```

This approach works because `leaderboardEntries` and `bordaScores` are computed from `useRealtimePoll` data that is already in scope within PollResults. The live real-time data flows directly to PresentationResults without any prop-threading through PollLiveClient.

**In `src/app/(dashboard)/polls/[pollId]/live/client.tsx`:**

1. Remove the direct PresentationMode rendering block (lines ~167-173 where `{presenting && <PresentationMode>...`) since PollResults now handles it internally.
2. Keep the `presenting` state and the `setPresenting` setter -- they still control the toggle.
3. Keep the keyboard shortcut (F key) and the Present button -- they set `presenting`.
4. Pass new props to PollResults:
```tsx
const resultsContent = (
  <PollResults
    poll={poll}
    initialVoteCounts={initialVoteCounts}
    initialTotalVotes={initialTotalVotes}
    initialParticipantCount={initialParticipantCount}
    forceReveal={forceReveal}
    onRevealDismissed={() => setForceReveal(false)}

    pollTitle={poll.question}
```

5. Remove the `import { PresentationMode }` since PollLiveClient no longer renders it directly.

Keep the existing Fullscreen API integration, Escape handling, and Exit button unchanged.
