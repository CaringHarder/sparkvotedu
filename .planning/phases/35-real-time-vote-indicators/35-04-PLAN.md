  <name>Task 1: Verify real-time vote indicators across all activity types</name>
  <what-built>
    Real-time per-student green dot vote indicators on the teacher's live dashboard ParticipationSidebar for all activity types: SE brackets, DE brackets, RR brackets, predictive brackets, and polls. Not-voted students float to the top of the list. Indicators update live as votes come in without page refresh.
  </what-built>
  <how-to-verify>
    **Setup:** Open the app in two browser windows -- one as teacher, one as student (or use incognito).

    **Test 1 -- SE Bracket:**
    1. Create an SE bracket, assign to a session, go live
    2. Open matchup for voting
    3. On student window, cast a vote
    4. On teacher window, verify: green dot appears next to student name within seconds
    5. Verify: not-voted students are at the TOP of the sidebar list
    6. Verify: vote progress shows "1 of N voted"

    **Test 2 -- Poll:**
    1. Create a simple poll, assign to session, activate
    2. On teacher live page, verify ParticipationSidebar is visible
    3. On student window, cast a poll vote
    4. Verify: green dot appears within seconds on teacher sidebar
    5. Verify: not-voted students at top, vote progress accurate

    **Test 3 -- Round Robin (if easily available):**
    1. Create an RR bracket, go live, open a round
    2. Cast votes on SOME matchups (not all) for one student
    3. Verify: NO green dot (partial voting)
    4. Cast remaining matchup votes for that round
    5. Verify: green dot appears (all matchups voted)

    **Test 4 -- Round advancement reset:**
    1. On an SE bracket, advance to next round after all votes
    2. Verify: all green dots disappear (reset for new round)
    3. Verify: sidebar shows 0 of N voted for new round

    **Test 5 -- Predictive bracket (if easily available):**
    1. Create a predictive bracket, go live, open predictions
    2. Submit a complete prediction
    3. Verify: green dot appears for that student

    At minimum, Tests 1, 2, and 4 must pass. Tests 3 and 5 are high-priority but can be deferred if those bracket types take too long to set up.
  </how-to-verify>
