  <name>Task 1: Fix fullscreen/presentation mode auto-close</name>
  <files>src/components/poll/presentation-mode.tsx, src/app/(dashboard)/polls/[pollId]/live/client.tsx</files>
  <action>
The root cause is threefold (per research):
1. The `onExit` prop is an inline arrow function that creates a new reference every render, causing the useEffect to re-run
2. The useEffect cleanup calls `document.exitFullscreen()`, then re-requests fullscreen on re-run -- creating a flicker/close cycle
3. The `fullscreenchange` listener calls `onExit()` when `document.fullscreenElement` becomes null, which happens during the re-run cycle

**Fix in presentation-mode.tsx:**

1. **Decouple overlay from Fullscreen API:** The fixed-position overlay should persist independently of whether the browser's Fullscreen API succeeds or fails. The overlay is the presentation -- fullscreen is an enhancement.

2. **Use a ref for the onExit callback** instead of including it in useEffect dependencies:
   ```typescript
   const onExitRef = useRef(onExit)
   useEffect(() => { onExitRef.current = onExit }, [onExit])
   ```

3. **Simplify the fullscreen useEffect:**
   - Request fullscreen on mount (fire-and-forget, catch errors silently)
   - Do NOT call `onExit` when `fullscreenchange` fires with no fullscreenElement -- the overlay should stay visible regardless of fullscreen state
   - On cleanup, exit fullscreen if active (no onExit call)

4. **Only exit presentation on explicit user action:**
   - Esc key press: Add a `keydown` listener for Escape that calls `onExitRef.current()`
   - F key press: Add a `keydown` listener for 'f'/'F' that calls `onExitRef.current()`
   - Exit button click: Already calls `onExit`
   - No click-outside-to-exit (per user decision)
   - The Esc key should BOTH exit the browser fullscreen AND close the overlay. Handle the case where browser Esc exits fullscreen first -- the keydown handler should still fire onExit to close the overlay.

5. **Remove the `fullscreenchange` -> `onExit()` connection.** The fullscreenchange listener should only track whether the browser is in fullscreen for UI purposes (e.g., showing/hiding a "fullscreen" indicator), not trigger overlay closure.

**Fix in polls/[pollId]/live/client.tsx:**

6. **Stabilize the `onExitPresentation` callback** with `useCallback`:
   ```typescript
   const onExitPresentation = useCallback(() => setPresenting(false), [])
   ```
   Pass this stable reference to `PresentationMode`.
