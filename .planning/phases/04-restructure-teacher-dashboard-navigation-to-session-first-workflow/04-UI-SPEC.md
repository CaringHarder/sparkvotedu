## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 (20px) | Card metadata, student roster rows, badge text, tab trigger labels, sidebar nav items, dropdown labels |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.3 (26px) | Session name in workspace header, page titles |
| Display | 30px (`text-3xl`) | 600 (semibold) | 1.2 (36px) | Dashboard welcome message (existing) |

Declared weights: 400 (normal), 600 (semibold). No other weights permitted in this phase.

### Card Title Typography (D-12 fix)

Card titles (`BracketCard`, `PollCard`) use:
- Size: 14px (`text-sm`)
- Weight: 600 (`font-semibold`)
- Line height: 1.5
- Clamping: `line-clamp-2` (replaces `truncate`) -- allows up to 2 visible lines before ellipsis
- Min display: titles up to ~60 characters show fully on a standard card width (280px+)

### Join Code Typography (D-08)

- Size: 28px (`text-2xl` with `tracking-widest`)
- Weight: 600 (`font-semibold`) -- prominence comes from size, monospace, and letter-spacing, not weight
- Font: Geist Mono (`font-mono`) for clarity of alphanumeric codes
- Color: `text-foreground`

## Color

This phase does not introduce new colors. It uses the existing project palette defined in `globals.css` via oklch CSS custom properties.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (oklch(1 0 0) light / oklch(0.145 0 0) dark) | Page background, main content area |
| Secondary (30%) | `--card` / `--sidebar` | Session cards, sidebar navigation, tab content panels, dropdown popover |
| Accent (10%) | `--brand-blue` (oklch(0.68 0.15 230)) | Active tab indicator, session status "active" badge, join code highlight, "Create Bracket"/"Create Poll" CTA buttons |
| Destructive | `--destructive` (oklch(0.577 0.245 27.325)) | Delete actions in context menus, "End Session" confirmation button |

Accent reserved for:
- Active tab underline/highlight in session workspace tabs
- "Active" session status badge background
- Join code copy button hover state
- Primary CTA buttons within session workspace ("Create Bracket", "Create Poll")
- Dashboard session dropdown selected-item indicator

### Dashboard Session Dropdown (D-16)

- Position: below welcome section, above Plan & Usage card
- Component: shadcn `Select` with placeholder "Jump to a session..."
- Items: active sessions only (D-17), sorted by last activity
- On select: `router.push(/sessions/${sessionId})`
- Width: `max-w-sm` (384px)
- If no active sessions: do not render the dropdown at all

### Session Picker Dialog (D-14, D-15)

- Component: shadcn `Dialog` with scrollable session list
- Title: "Move to session" or "Duplicate to session" (passed as prop)
- List items: session name + status badge + student count
- Current session: shown but disabled/grayed with "(current)" label
- Confirm button: "Move Activity" or "Duplicate Activity" (matches action)
- Loading state: button shows spinner and disables during server action

## Interaction Contracts

### Tab Navigation (D-07, D-09)

| Interaction | Behavior |
|-------------|----------|
| Page load | Default tab = tab with most recently updated activity. Empty session defaults to "Brackets" |
| Tab click | Switch tab content, update URL query param `?tab=brackets\|polls\|students` |
| Browser back | Restore previous tab via query param |
| Tab with zero items | Show empty state inside tab content (not disabled) |

### Card Title Display (D-12)

| Scenario | Behavior |
|----------|----------|
| Title <= 1 line | Display fully, single line |
| Title 1-2 lines | Display fully, wraps to second line |
| Title > 2 lines | Clamp at 2 lines with ellipsis |
| Rename input active | Input uses `truncate` (single-line input is acceptable) |

### Context Menu Extensions (D-14, D-15)

| Menu Item | Trigger | Result |
|-----------|---------|--------|
| "Move to session..." | Click | Opens SessionPickerDialog. On confirm, calls `assignBracketToSession` / `assignPollToSession`. Card disappears from current tab via `router.refresh()`. |
| "Duplicate to session..." | Click | Opens SessionPickerDialog with title "Duplicate to session". On confirm, calls extended `duplicateBracket` / `duplicatePoll` with `targetSessionId`. Toast confirms success. |

### Dashboard Dropdown (D-16)

| Interaction | Behavior |
|-------------|----------|
| Click dropdown | Opens session list (active sessions only) |
| Select session | Navigates to `/sessions/{id}` |
| No active sessions | Dropdown not rendered; "Create Session" CTA is prominent |

### End Session Confirmation

| Interaction | Behavior |
|-------------|----------|
| Click "End Session" | Opens confirmation dialog before ending the session |
| Confirm | Calls end session action, redirects to sessions list |
| Cancel | Closes dialog, no action taken |

### Orphan Migration (D-13)

| Scenario | Behavior |
|----------|----------|
| Teacher has orphan activities | On dashboard load, auto-create "General" session, assign orphans. No user-visible UI -- silent migration. |
| Teacher has no orphans | No action taken |
| "General" session already exists | Reuse it for additional orphans |

## Accessibility Notes

- Tab navigation: shadcn Tabs component provides ARIA `role="tablist"`, `role="tab"`, `role="tabpanel"`, keyboard arrow navigation
- Session picker dialog: shadcn Dialog provides focus trap, Escape to close, ARIA `role="dialog"`
- Select dropdown: shadcn Select provides keyboard navigation, ARIA combobox pattern
- Card titles with `line-clamp-2`: full title available via `title` attribute on the `<h3>` for tooltip/screen reader access
- Context menu items: existing Radix DropdownMenu provides keyboard navigation and ARIA
- Icon-only buttons: Copy join code button must have `aria-label="Copy join code"`. QR code button must have `aria-label="Show QR code"`. Both use `Button` variant="ghost" size="icon".
