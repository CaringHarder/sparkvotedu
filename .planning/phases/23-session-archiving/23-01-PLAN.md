**DAL functions (src/lib/dal/class-session.ts):**

1. **Modify `getTeacherSessions`** -- add `archivedAt: null` to the where clause so archived sessions are excluded from the main list:
   ```typescript
   where: { teacherId, archivedAt: null },
   ```

2. **Add `archiveSession(sessionId: string, teacherId: string)`** -- Follows the existing `endSession` pattern:
   - Find session with ownership check (`findFirst where { id: sessionId, teacherId }`)
   - Throw `'Session not found or unauthorized'` if not found
   - Use `prisma.$transaction` to atomically:
     a. `updateMany` brackets where `{ sessionId, status: 'active' }` to `{ status: 'completed' }`
     b. `updateMany` polls where `{ sessionId, status: 'active' }` to `{ status: 'closed' }`
     c. Update the session: set `archivedAt: new Date()`, `status: 'ended'`, `endedAt: session.endedAt ?? new Date()` (preserve existing endedAt if already ended)
   - Return the updated session

3. **Add `unarchiveSession(sessionId: string, teacherId: string)`**:
   - Find session with `{ id: sessionId, teacherId, archivedAt: { not: null } }`
   - Throw `'Session not found, unauthorized, or not archived'` if not found
   - Update session: `{ archivedAt: null }` -- session returns to main list as "ended"
   - Return the updated session

4. **Add `deleteSessionPermanently(sessionId: string, teacherId: string)`**:
   - Find session with `{ id: sessionId, teacherId, archivedAt: { not: null } }` -- ONLY archived sessions can be permanently deleted (two-step safety net per locked decision)
   - Throw `'Session not found, unauthorized, or not archived'` if not found
   - Use `prisma.$transaction` to atomically:
     a. Find all brackets with `{ sessionId }`, then delete each by id (cascades to entrants, matchups, votes, predictions via schema cascade)
     b. Find all polls with `{ sessionId }`, then delete each by id (cascades to options, poll_votes via schema cascade)
     c. Delete the session by id (cascades to student_participants, which cascade to votes, poll_votes, predictions)
   - IMPORTANT: Brackets and polls have OPTIONAL sessionId with NO cascade delete -- they must be explicitly deleted or they'll become orphaned records

5. **Add `getArchivedSessions(teacherId: string, search?: string)`**:
   - Query `findMany` with `{ teacherId, archivedAt: { not: null } }`
   - If `search` is provided and non-empty, add `name: { contains: search, mode: 'insensitive' }` to the where clause
   - Order by `{ archivedAt: 'desc' }` (newest archived first, per locked decision)
   - Include `_count: { select: { participants: true, brackets: true, polls: true } }` for card info density
