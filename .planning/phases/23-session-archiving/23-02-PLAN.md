The sessions page (`src/app/(dashboard)/sessions/page.tsx`) currently renders sessions as `<Link>` cards directly. The three-dot menu needs to be overlaid on each card.

**Problem:** The sessions page is a server component. The SessionCardMenu is a client component. We need to extract the session card grid into a client component wrapper so the menu's `onArchived` callback can trigger `router.refresh()`.

Implementation approach:

1. **Create a client wrapper** -- either inline or as a separate small component. The simplest approach: Create a `SessionList` client component inline within the same file (or as a separate file `src/components/teacher/session-list.tsx` if needed for cleanliness). However, to minimize file creation and follow existing patterns, convert the grid rendering into a client component.

   **Preferred approach:** Create `src/components/teacher/session-list.tsx` as a client component that receives sessions data and renders the grid with menus.

   Actually, on reflection, to keep `files_modified` accurate and minimal, modify `sessions/page.tsx` to:
   - Keep the page as a server component for data fetching
   - Import a new `SessionCardGrid` client component that renders the card grid
   - Pass serialized sessions data to it

   But this adds a file. Instead, the SIMPLEST approach is:
   - Wrap each session card's Link in a `<div className="relative">`
   - Add the `<SessionCardMenu>` positioned absolutely in the top-right
   - The sessions page stays as a server component -- the SessionCardMenu is already a client component that handles its own state
   - For refresh after archive: the SessionCardMenu calls `archiveSessionAction` which calls `revalidatePath('/sessions')` server-side, and the server action return triggers Next.js to revalidate the page data automatically

   This is the cleanest approach -- no wrapper needed because `revalidatePath` in the server action already handles page data refresh.

2. **Update the session card rendering:**
   - Import `SessionCardMenu` from `@/components/teacher/session-card-menu`
   - Wrap each card in `<div className="relative" key={session.id}>`
   - Move the `key` from `<Link>` to the wrapper `<div>`
   - Keep the `<Link>` and `<Card>` as-is
   - Add padding-right to the card header's first child div to make room for the menu: change `<div className="flex items-start justify-between">` content so the title doesn't overlap with the menu button
   - Add `<div className="absolute right-3 top-3 z-10"><SessionCardMenu sessionId={session.id} sessionName={session.name} /></div>` after the `</Link>` closing tag but inside the wrapper div

3. The `SessionCardMenu` trigger button already has `stopPropagation`/`preventDefault` so clicking the three-dot icon won't navigate via the parent Link.
