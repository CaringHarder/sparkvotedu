<objective>
Wire server-side feature gating into all existing bracket and poll creation actions so that tier limits are enforced at the mutation layer. Reconcile the TIER_LIMITS analytics field with BILL-01 requirements. Add new gate functions for live and draft bracket count limits.

Purpose: This is the security backbone of the billing system -- server-side enforcement ensures that bypassing the UI (curl, devtools, etc.) does not grant access to paid features. The existing `canAccess()`, `canCreateBracket()`, `canUseBracketType()`, `canUseEntrantCount()`, `canUsePollType()`, and `canUsePollOptionCount()` functions are already built but not yet called in the server actions. This plan wires them in.

Output: Server actions for bracket and poll creation check tier limits before proceeding. New live/draft bracket count gate functions. TIER_LIMITS analytics field reconciled.
</objective>

    2. Update `src/lib/gates/features.ts` -- Add new gate functions:

       **canCreateLiveBracket(tier, currentLiveCount)**:
       - Check `currentLiveCount >= TIER_LIMITS[tier].maxLiveBrackets`
       - If at limit, return `{ allowed: false, reason: "Live bracket limit reached (${limit}). Complete or archive a bracket, or upgrade to ${upgradeTarget}.", upgradeTarget }`
       - Uses the same pattern as `canCreateBracket()`

       **canCreateDraftBracket(tier, currentDraftCount)**:
       - Check `currentDraftCount >= TIER_LIMITS[tier].maxDraftBrackets`
       - If at limit, return `{ allowed: false, reason: "Draft bracket limit reached (${limit}). Delete a draft or upgrade to ${upgradeTarget}.", upgradeTarget }`

       **Update canAccess()**: Since `analytics` no longer exists as a key, ensure the FeatureKey type updates automatically (it's derived from `keyof typeof TIER_LIMITS.free`). The type will now include `basicAnalytics` and `csvExport` instead of `analytics`. Existing callers of `canAccess(tier, 'analytics')` do not exist yet (analytics phase is future), so this is a clean change.

       **Update existing canCreateBracket()**: Keep as-is for total bracket count check (maxBrackets). The new live/draft functions complement it.

    IMPORTANT: Run `npx vitest run` after changes to ensure existing gate tests pass. The test file at `src/lib/gates/__tests__/features.test.ts` tests the existing gate functions -- update test expectations to reflect the analytics -> basicAnalytics/csvExport split if needed.

    3. Update `src/actions/bracket.ts` -- Add gate checks to bracket creation action:
       - At the top of the create bracket action (after auth check, before database mutation):
         a. Get teacher's subscription tier: `const tier = teacher.subscriptionTier as SubscriptionTier`
         b. Get bracket counts: `const counts = await getTeacherBracketCounts(teacher.id)`
         c. Check total bracket limit: `const totalCheck = canCreateBracket(tier, counts.total)` -- return `{ error: totalCheck.reason }` if not allowed
         d. Check live bracket limit (if creating an active bracket): `const liveCheck = canCreateLiveBracket(tier, counts.live)` -- return error if not allowed
         e. Check draft bracket limit (if creating a draft bracket): `const draftCheck = canCreateDraftBracket(tier, counts.draft)` -- return error if not allowed
         f. Check bracket type: `const typeCheck = canUseBracketType(tier, data.bracketType)` -- return error if not allowed
         g. Check entrant count: `const entrantCheck = canUseEntrantCount(tier, data.size)` -- return error if not allowed
       - Also add gate check to the "activate bracket" action (status change to 'active'):
         Check `canCreateLiveBracket(tier, counts.live)` before allowing activation

    4. Update `src/actions/poll.ts` -- Add gate checks to poll creation action:
       - At the top of the create poll action (after auth check, before database mutation):
         a. Get tier: `const tier = teacher.subscriptionTier as SubscriptionTier`
         b. Check poll type: `const typeCheck = canUsePollType(tier, data.pollType)` -- return error if not allowed
         c. Check poll option count: `const optionCheck = canUsePollOptionCount(tier, data.options.length)` -- return error if not allowed
       - Also add type check when updating poll type

    IMPORTANT: Gate checks must happen BEFORE the database mutation (create/update). Return `{ error: string }` matching the existing action error pattern. Do NOT throw -- return error objects that the UI can display. The gate functions are PURE (no async, no DB calls) -- the action provides the tier and counts, the gate function returns allowed/denied.
