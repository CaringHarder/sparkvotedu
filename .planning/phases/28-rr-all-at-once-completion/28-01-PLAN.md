In `updateBracketStatusDAL` (around lines 655-668), modify the round-robin activation block to be pacing-aware:

**Current code (buggy):**
```typescript
if (bracket.status === 'draft' && status === 'active' && updated.bracketType === 'round_robin') {
  await prisma.matchup.updateMany({
    where: {
      bracketId: bracket.id,
      roundRobinRound: 1,
      status: 'pending',
    },
    data: { status: 'voting' },
  })
  broadcastBracketUpdate(bracketId, 'round_advanced', { round: 1 }).catch(console.error)
}
```

**Replace with:**
```typescript
if (bracket.status === 'draft' && status === 'active' && updated.bracketType === 'round_robin') {
  const pacing = (updated.roundRobinPacing ?? 'round_by_round') as string
  if (pacing === 'all_at_once') {
    // All-at-once: open ALL matchups across ALL rounds simultaneously
    await prisma.matchup.updateMany({
      where: {
        bracketId: bracket.id,
        status: 'pending',
      },
      data: { status: 'voting' },
    })
  } else {
    // Round-by-round: only open round 1
    await prisma.matchup.updateMany({
      where: {
        bracketId: bracket.id,
        roundRobinRound: 1,
        status: 'pending',
      },
      data: { status: 'voting' },
    })
  }
  broadcastBracketUpdate(bracketId, 'round_advanced', { round: 1 }).catch(console.error)
}
```

Key points:
- Read `roundRobinPacing` from the `updated` Prisma return (full record, no select clause).
- Default to `'round_by_round'` if null (existing brackets without pacing set).
- The `all_at_once` branch omits the `roundRobinRound` filter so ALL matchups open.
- The `round_by_round` branch is exactly the old behavior -- no regression.
- The broadcast call stays unchanged (round_advanced with round: 1 is fine for both modes since it triggers a client refetch).
