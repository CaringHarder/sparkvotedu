<task type="auto">
  <name>Task 1: Enhance OAuth callback to sync teacher profile data</name>
  <files>
    src/app/auth/callback/route.ts
  </files>
  <action>
    The current OAuth callback at `src/app/auth/callback/route.ts` only exchanges the code for a session. Per user decision, OAuth should "request full profile data from all providers: email, name, and avatar."

    Supabase Auth already captures this data from providers and stores it in `auth.users.raw_user_meta_data`. The callback needs to sync this to the Teacher record in our database.

    Update the callback route to:
    1. After successful `exchangeCodeForSession`, call `supabase.auth.getUser()` to get the authenticated user
    2. Extract profile data from `user.user_metadata`: `full_name` (or `name`), `avatar_url` (or `picture`), `email`
    3. Upsert the Teacher record using the profile data

    Import `prisma` from `@/lib/prisma`. The Teacher model has these fields: `id`, `email`, `name` (nullable String), `avatarUrl` (nullable String, maps to `avatar_url`), `supabaseAuthId` (unique String, maps to `supabase_auth_id`).

    Handle the case where user_metadata field names vary by provider:
    - Google: `full_name`, `avatar_url`, `email`
    - Azure/Microsoft: `full_name` or `name`, `avatar_url` or `picture`, `email`
    - Apple: `full_name` (may be null on subsequent logins), no avatar, `email`

    Use a helper like:
    ```ts
    const metadata = user.user_metadata
    const email = user.email ?? metadata.email
    const name = metadata.full_name ?? metadata.name ?? null
    const avatarUrl = metadata.avatar_url ?? metadata.picture ?? null
    ```

    Use `prisma.teacher.upsert` with `where: { supabaseAuthId: user.id }`:
    - `create`: Set `email`, `name`, `avatarUrl`, `supabaseAuthId`
    - `update`: Only update `name` and `avatarUrl` if the new values are non-null (don't overwrite existing data with null from Apple on subsequent logins). Use conditional spread: `...(name ? { name } : {})` and `...(avatarUrl ? { avatarUrl } : {})`

    Wrap the prisma call in try/catch. Log errors but don't block the redirect -- the teacher should still reach the dashboard even if profile sync fails.

    The full flow: exchange code -> get user -> upsert teacher -> redirect to dashboard.
  </action>
  <verify>
    - `src/app/auth/callback/route.ts` imports prisma from `@/lib/prisma`
    - File contains `supabase.auth.getUser()` call after code exchange
    - File contains `prisma.teacher.upsert` with `supabaseAuthId`
    - Profile extraction handles `full_name`, `name`, `avatar_url`, `picture` variants
    - `npx tsc --noEmit` passes with no type errors
  </verify>
  <done>
    OAuth callback syncs teacher name and avatar from provider metadata to Teacher record via upsert on supabaseAuthId. New OAuth users get a Teacher record created automatically. Returning users get non-null profile data refreshed. Errors are logged but don't block sign-in.
  </done>
</task>
