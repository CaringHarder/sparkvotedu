<task type="auto">
  <name>Task 3: Create service health check endpoint</name>
  <files>
    src/app/api/health/route.ts
  </files>
  <action>
    Create `src/app/api/health/route.ts` -- a GET endpoint that checks the configuration status of all external services. This gives the user (and future admin panel) a single URL to verify everything is wired up.

    The endpoint should check:
    1. **Supabase Auth**: Verify `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are set. Use admin client to call `supabase.auth.admin.listUsers({ perPage: 1 })` as a connectivity test.
    2. **Supabase Storage**: Use admin client to check if `poll-images` bucket exists via `supabase.storage.getBucket('poll-images')`. Report `configured` or `missing`.
    3. **Stripe**: Verify `STRIPE_SECRET_KEY` and `STRIPE_WEBHOOK_SECRET` are set. Call `stripe.accounts.retrieve()` (retrieves own account) as a connectivity test.
    4. **SportsDataIO**: Verify `SPORTSDATAIO_API_KEY` is set. Do NOT make an API call (costs quota). Just report env var presence.
    5. **Cron**: Verify `CRON_SECRET` is set.
    6. **Site URL**: Report current `NEXT_PUBLIC_SITE_URL` value.

    Response shape:
    ```json
    {
      "status": "healthy" | "degraded" | "unhealthy",
      "timestamp": "ISO string",
      "services": {
        "supabase_auth": { "status": "ok" | "error", "detail": "..." },
        "supabase_storage": { "status": "ok" | "missing", "detail": "..." },
        "stripe": { "status": "ok" | "error", "detail": "..." },
        "sportsdata": { "status": "ok" | "not_configured", "detail": "..." },
        "cron_secret": { "status": "ok" | "not_configured" },
        "site_url": { "value": "..." }
      }
    }
    ```

    Overall status: "healthy" if all services ok, "degraded" if some missing, "unhealthy" if critical services (auth, stripe) failing.

    Use `export const dynamic = 'force-dynamic'` to prevent caching. Wrap each service check in its own try/catch so one failure doesn't prevent checking others.

    Import `createAdminClient` from `@/lib/supabase/admin` and `stripe` from `@/lib/stripe`. Import `getProvider` from `@/lib/sports/provider` but only for env var check (don't instantiate).

    Do NOT require authentication -- this endpoint is for deployment verification. It does not expose secrets, only status. In production, consider rate limiting or restricting to admin (but not now -- keep it simple for launch).
  </action>
  <verify>
    - File exists at `src/app/api/health/route.ts`
    - `curl http://localhost:3001/api/health` returns JSON with `status`, `timestamp`, `services` fields
    - Response includes checks for all 6 service areas
    - `npx tsc --noEmit` passes
  </verify>
  <done>
    GET /api/health returns a JSON report of all external service configuration status. Each service checked independently with error isolation. Overall status reflects aggregate health.
  </done>
