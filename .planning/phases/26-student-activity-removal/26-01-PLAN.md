Add `broadcastActivityUpdate(sessionId)` calls to 4 server actions that currently lack them.

**CRITICAL: For delete actions, read sessionId BEFORE calling the DAL delete function** because the DAL cascade removes the row and sessionId becomes unreadable afterward.

**src/actions/bracket.ts -- deleteBracket:**
Before the `try` block (after validation), add a pre-read of sessionId:
```typescript
// Read sessionId BEFORE delete (cascade removes the row)
const bracket = await prisma.bracket.findFirst({
  where: { id: parsed.data.bracketId, teacherId: teacher.id },
  select: { sessionId: true },
})
const preDeleteSessionId = bracket?.sessionId
```
After the `revalidatePath('/brackets')` line, add:
```typescript
// Broadcast removal to students
if (preDeleteSessionId) {
  broadcastActivityUpdate(preDeleteSessionId).catch(console.error)
}
```

**src/actions/bracket.ts -- archiveBracket:**
The `archiveBracketDAL` delegates to `updateBracketStatusDAL` which returns the updated bracket (it does NOT delete the row). So sessionId can be read after the DAL call. After `revalidatePath('/dashboard')`, add:
```typescript
// Read sessionId from the archived bracket for student broadcast
const archivedBracket = await prisma.bracket.findUnique({
  where: { id: parsed.data.bracketId },
  select: { sessionId: true },
})
if (archivedBracket?.sessionId) {
  broadcastActivityUpdate(archivedBracket.sessionId).catch(console.error)
}
```

After `revalidatePath('/activities')`, add:
