  <action>
    1. Create `src/lib/gates/tiers.ts`:
       - Export `type SubscriptionTier = 'free' | 'pro' | 'pro_plus'`.
       - Export `const TIER_LIMITS` as a const object with all three tiers:

       **Free tier:**
       - maxBrackets: 3
       - maxEntrantsPerBracket: 16
       - bracketTypes: ['single_elimination']
       - pollTypes: ['simple']
       - analytics: false
       - sportsIntegration: false
       - csvUpload: false
       - liveEventMode: false
       - maxLiveBrackets: 2
       - maxDraftBrackets: 2

       **Pro tier ($12/mo):**
       - maxBrackets: 25
       - maxEntrantsPerBracket: 64
       - bracketTypes: ['single_elimination', 'double_elimination', 'round_robin']
       - pollTypes: ['simple', 'ranked']
       - analytics: true
       - sportsIntegration: false
       - csvUpload: true
       - liveEventMode: true
       - maxLiveBrackets: 10
       - maxDraftBrackets: 15

       **Pro Plus tier ($20/mo):**
       - maxBrackets: Infinity
       - maxEntrantsPerBracket: 128
       - bracketTypes: ['single_elimination', 'double_elimination', 'round_robin', 'predictive']
       - pollTypes: ['simple', 'ranked']
       - analytics: true
       - sportsIntegration: true
       - csvUpload: true
       - liveEventMode: true
       - maxLiveBrackets: Infinity
       - maxDraftBrackets: Infinity

       Use `as const` on the entire object for TypeScript literal type inference.

    2. Create `src/lib/gates/features.ts`:
       - Import `TIER_LIMITS` and `SubscriptionTier` from `./tiers`.
       - Export `type FeatureKey = keyof typeof TIER_LIMITS.free`.
       - Export `interface AccessResult { allowed: boolean; reason?: string; upgradeTarget?: SubscriptionTier }`.

       - Export `function canAccess(tier: SubscriptionTier, feature: FeatureKey): AccessResult`:
         - Look up `TIER_LIMITS[tier][feature]`.
         - If the value is a boolean and false: return `{ allowed: false, reason: '${feature} requires ${upgradeTarget} plan', upgradeTarget }`.
         - Determine upgradeTarget: if tier is 'free', target is 'pro'. If 'pro', target is 'pro_plus'. If the feature is still not available on 'pro' (e.g., sportsIntegration), target should be 'pro_plus' regardless.
         - If the value is boolean and true, or is any other type (number, array): return `{ allowed: true }`.

       - Export `function canCreateBracket(tier: SubscriptionTier, currentCount: number): AccessResult`:
         - Check `currentCount >= TIER_LIMITS[tier].maxBrackets`.
         - If at limit, return `{ allowed: false, reason, upgradeTarget }`.
         - Otherwise return `{ allowed: true }`.

       - Export `function canUseBracketType(tier: SubscriptionTier, bracketType: string): AccessResult`:
         - Check if `bracketType` is in `TIER_LIMITS[tier].bracketTypes`.
         - If not, determine which tier first allows it and set as upgradeTarget.
         - Return appropriate AccessResult.

       - Export `function canUseEntrantCount(tier: SubscriptionTier, entrantCount: number): AccessResult`:
         - Check if `entrantCount <= TIER_LIMITS[tier].maxEntrantsPerBracket`.
         - If exceeded, return `{ allowed: false, reason, upgradeTarget }`.
         - Otherwise return `{ allowed: true }`.

       All functions are PURE -- no database calls, no side effects, no async. They take tier + context and return a decision. This makes them testable, callable anywhere (proxy, DAL, components), and fast.

    IMPORTANT: These functions must be pure. They do NOT look up the user's tier -- that's the caller's job. The caller gets the tier from the Teacher record, then calls canAccess(tier, feature).
    IMPORTANT: The upgradeTarget logic must be smart: if a feature requires pro_plus (not available on pro), then a free user's upgradeTarget should be 'pro_plus', not 'pro'.
  </action>
  <verify>

    - features.ts exports canAccess, canCreateBracket, canUseBracketType, canUseEntrantCount

    - No database imports in either file (pure functions)
  </verify>
  <done>TIER_LIMITS defines complete feature boundaries for Free/Pro/Pro Plus. canAccess(), canCreateBracket(), canUseBracketType(), and canUseEntrantCount() are pure functions that return AccessResult with allowed, reason, and upgradeTarget. No side effects, no database calls.</done>
