<task type="auto">
  <name>Task 1: Add password reset and sign-out server actions</name>
  <files>
    src/actions/auth.ts
  </files>
  <action>
    1. Open the existing `src/actions/auth.ts` (created in Plan 02) and ADD the following server actions. Do NOT remove or modify the existing signUp and signIn actions.

    2. Add `resetPassword(formData: FormData)`:
       - Validate email with `forgotPasswordSchema` from validation.ts.
       - Call `supabase.auth.resetPasswordForEmail(email, { redirectTo: \`${process.env.NEXT_PUBLIC_SITE_URL}/auth/callback?next=/update-password\` })`.
       - On success, return `{ success: 'Check your email for a password reset link.' }`.
       - On error, return `{ error: error.message }`.
       - IMPORTANT: The redirectTo sends the user to /auth/callback with `next=/update-password`. The callback route (Plan 02) exchanges the code for a session, then redirects to /update-password where the user sets a new password.

    3. Add `updatePassword(formData: FormData)`:
       - Validate with `updatePasswordSchema` from validation.ts (password + confirmPassword match).
       - Call `supabase.auth.updateUser({ password })`.
       - On success, `redirect('/dashboard')`.
       - On error, return `{ error: error.message }`.
       - IMPORTANT: This only works when the user has a valid session (established by the callback route after clicking the reset email link).

    4. Add `signOut()`:
       - Call `supabase.auth.signOut()`.
       - Call `redirect('/login')`.
       - No form data needed -- this is a simple action.

    All three new actions use `createClient` from `@/lib/supabase/server` (same as existing actions).
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - actions/auth.ts exports: signUp, signIn, resetPassword, updatePassword, signOut
    - resetPassword uses `resetPasswordForEmail` with redirectTo including `/auth/callback?next=/update-password`
    - updatePassword uses `supabase.auth.updateUser`
    - signOut calls `supabase.auth.signOut()` then `redirect('/login')`
  </verify>
  <done>Three new server actions added to auth.ts: resetPassword sends email via Supabase, updatePassword sets new password for authenticated user, signOut clears session and redirects to login. All use Supabase server client with proper validation.</done>
