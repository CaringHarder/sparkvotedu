  <action>
    1. Create `src/components/auth/oauth-buttons.tsx` as a client component:
       - 'use client' directive.
       - Import `createClient` from `@/lib/supabase/client`.
       - Create a function `handleOAuth(provider: 'google' | 'azure' | 'apple')`:
         - Get Supabase browser client.
         - Call `supabase.auth.signInWithOAuth({ provider, options: { redirectTo: \`${window.location.origin}/auth/callback\` } })`.
         - For the 'azure' provider, add `scopes: 'email'` to options (required for Microsoft to return email).
         - Handle errors by setting an error state.
       - Render three buttons in a vertical stack (flex col gap-2):
         - "Continue with Google" -- Google branded icon (use Lucide or inline SVG for brand icons).
         - "Continue with Microsoft" -- Microsoft branded icon.
         - "Continue with Apple" -- Apple branded icon.
       - Use shadcn/ui Button component with variant="outline" for OAuth buttons.
       - Add loading state: disable buttons and show spinner while OAuth redirect is in progress.
       - Display error message if OAuth initiation fails.

       IMPORTANT: The Microsoft provider in Supabase is called `'azure'` (NOT 'microsoft'). Use `provider: 'azure'`.
       IMPORTANT: The OAuth flow happens client-side (browser initiates redirect to provider). The callback route (Plan 02) handles the server-side code exchange.
       IMPORTANT: Brand icons for Google/Microsoft/Apple should be inline SVGs or simple icons. Do NOT install a heavy icon library just for these 3 icons.

    2. Update `src/app/(auth)/login/page.tsx`:
       - Add the OAuthButtons component BELOW the login form.
       - Add a visual Separator (from shadcn/ui) between the form and OAuth buttons with "or" text.
       - Layout: Login form -> Separator("or") -> OAuth buttons.

    3. Update `src/app/(auth)/signup/page.tsx`:
       - Add the OAuthButtons component BELOW the signup form.
       - Add a visual Separator between the form and OAuth buttons with "or" text.
       - Layout: Signup form -> Separator("or") -> OAuth buttons.

    The OAuth flow works as follows:
    1. User clicks "Continue with Google" on login/signup page
    2. Browser calls supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: '/auth/callback' } })
    3. Supabase redirects user to Google's consent screen
    4. Google redirects back to Supabase, which redirects to /auth/callback?code=xxx
    5. The callback route (from Plan 02) calls exchangeCodeForSession(code)
    6. Session is established, user is redirected to /dashboard
    7. The DAL (from Plan 02) auto-creates the Teacher record on first OAuth login

    No additional server-side code is needed -- Plan 02's callback route and DAL handle everything.
