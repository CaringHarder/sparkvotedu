<task type="auto">
  <name>Task 1: Create Poll DAL with CRUD, voting, and aggregation functions</name>
  <files>src/lib/dal/poll.ts</files>
  <action>
Create src/lib/dal/poll.ts following the exact pattern of src/lib/dal/bracket.ts and src/lib/dal/vote.ts.

**CRUD functions:**

1. `createPollDAL(teacherId, data, options)` -- Creates poll + options in a transaction. data: {question, description?, pollType, allowVoteChange?, showLiveResults?, rankingDepth?}. options: {text, imageUrl?, position}[]. Returns created poll with options.

2. `getPollByIdDAL(pollId)` -- Returns poll with options (include: {options: {orderBy: {position: 'asc'}}}). Returns null if not found.

3. `getPollsByTeacherDAL(teacherId, statusFilter?)` -- Returns polls filtered by teacher. Optional status filter array. Include _count of votes. Order by updatedAt desc.

4. `getPollsBySessionDAL(sessionId)` -- Returns polls for a session (active + closed). Include options and _count of votes.

5. `updatePollDAL(pollId, teacherId, data)` -- Updates poll fields. Verifies teacher ownership via findFirst with teacherId. Returns updated poll or null.

6. `deletePollDAL(pollId, teacherId)` -- Deletes poll. Verifies teacher ownership. Cascade handles options and votes. Returns boolean.

7. `updatePollStatusDAL(pollId, teacherId, newStatus)` -- Updates status with forward-only transition validation. Valid transitions: draft->active, active->closed, closed->archived, closed->draft (reopen). Returns updated poll or throws error for invalid transition.

8. `assignPollToSessionDAL(pollId, teacherId, sessionId)` -- Assigns poll to a session. Verifies teacher ownership.

9. `duplicatePollDAL(pollId, teacherId)` -- Creates a copy of an existing poll (with options, without votes). New poll is always draft status. Appends " (Copy)" to question.

**Vote functions:**

10. `castSimplePollVoteDAL(pollId, participantId, optionId)` -- Uses upsert with compound unique (pollId_participantId_rank where rank=1). Returns upserted vote.

11. `castRankedPollVoteDAL(pollId, participantId, rankings)` -- Delete-then-insert in a transaction. rankings: {optionId, rank}[]. Returns void.

**Aggregation functions:**

12. `getSimplePollVoteCounts(pollId)` -- Uses groupBy on optionId where rank=1. Returns Record<optionId, count>.

13. `getRankedPollVotes(pollId)` -- Returns raw vote data ({optionId, rank, participantId}[]) for Borda computation. Also returns totalUniqueVoters count.

14. `getPollParticipantVote(pollId, participantId)` -- Returns existing votes for a participant (for vote restoration). Returns array of {optionId, rank}.

**Pattern notes:**
- Import prisma from '@/lib/prisma'
- All teacher mutations verify ownership via findFirst with teacherId filter (same as bracket DAL pattern from 04-03)
- Read-only queries skip $transaction (04-02 decision)
- Use the DAL alias pattern if needed for name conflicts (02-03 decision)

**Create src/actions/poll.ts** with 'use server' directive, following the pattern of src/actions/bracket.ts and src/actions/vote.ts:

1. `createPoll(input)` -- Teacher action. Validates with createPollWithOptionsSchema (z.object({poll: createPollSchema, options: z.array(pollOptionSchema).min(2).max(32)})). Checks teacher auth via getAuthenticatedTeacher(). Checks feature gate: canUsePollType(teacher.subscriptionTier, pollData.pollType). Checks option count gate: canUsePollOptionCount(teacher.subscriptionTier, options.length). Calls createPollDAL. revalidatePath('/activities'). Returns {poll: {id, question}} or {error}.

2. `updatePoll(input)` -- Teacher action. Validates poll fields. Calls updatePollDAL. revalidatePath.

3. `deletePoll(input)` -- Teacher action. Validates with deletePollSchema. Calls deletePollDAL. revalidatePath('/activities').

4. `updatePollStatus(input)` -- Teacher action. Validates with updatePollStatusSchema. Calls updatePollStatusDAL. When activating (status='active'), also calls broadcastActivityUpdate(sessionId) if poll has a sessionId. When closing, calls broadcastPollUpdate(pollId, 'poll_closed'). revalidatePath.

5. `assignPollToSession(input)` -- Teacher action. z.object({pollId: z.string().uuid(), sessionId: z.string().uuid()}). Calls assignPollToSessionDAL. revalidatePath.

6. `duplicatePoll(input)` -- Teacher action. Validates pollId. Calls duplicatePollDAL. revalidatePath. Returns {poll: {id, question}}.

7. `castPollVote(input)` -- Student action (NO teacher auth). Validates with castPollVoteSchema or castRankedPollVoteSchema based on input shape. Checks: poll exists + status='active', participant exists + not banned, if !allowVoteChange then check no existing vote. For simple: call castSimplePollVoteDAL. For ranked: call castRankedPollVoteDAL. After vote: compute updated counts and broadcastPollVoteUpdate(pollId, voteCounts, totalVotes).catch(console.error). Returns {success: true} or {error}.

**Extend src/lib/realtime/broadcast.ts** -- Add two new functions:

1. `broadcastPollVoteUpdate(pollId, voteCounts, totalVotes)` -- Broadcasts to topic `poll:${pollId}` with event 'poll_vote_update'.

2. `broadcastPollUpdate(pollId, type, payload?)` -- Broadcasts to topic `poll:${pollId}` with event 'poll_update'. Types: 'poll_activated' | 'poll_closed' | 'poll_archived'.

**Extend src/lib/gates/features.ts** -- Add two new gate functions:

1. `canUsePollType(tier, pollType)` -- Checks TIER_LIMITS[tier].pollTypes array. Same pattern as canUseBracketType. Returns AccessResult with smart upgradeTarget.
