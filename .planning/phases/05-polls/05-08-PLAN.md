  <name>Task 1: Make assignPollToSession schema nullable and support unlinking</name>
  <files>src/actions/poll.ts</files>
  <action>
  In `src/actions/poll.ts`, the `assignPollToSessionSchema` (around line 202) currently requires a non-nullable UUID for sessionId:

  ```
  const assignPollToSessionSchema = z.object({
    pollId: z.string().uuid(),
    sessionId: z.string().uuid(),
  })
  ```

  This must be changed to match the bracket pattern where sessionId is nullable (allows unlinking):

  ```
  const assignPollToSessionSchema = z.object({
    pollId: z.string().uuid(),
    sessionId: z.string().uuid().nullable(),
  })
  ```

  Then update the `assignPollToSession` function body to handle null sessionId:
  - When sessionId is null, skip the session ownership check (no session to verify)
  - Pass sessionId directly to `assignPollToSessionDAL` -- but the DAL currently requires a string, not string | null

  So also update the DAL call. The DAL function `assignPollToSessionDAL` in `src/lib/dal/poll.ts` accepts `sessionId: string`. The Prisma update `data: { sessionId }` already handles null correctly at the Prisma level (sets the field to null). So update the DAL function signature to accept `string | null`:

  In `src/lib/dal/poll.ts`, change `assignPollToSessionDAL` signature from:
  ```
  export async function assignPollToSessionDAL(pollId: string, teacherId: string, sessionId: string)
  ```
  to:
  ```
  export async function assignPollToSessionDAL(pollId: string, teacherId: string, sessionId: string | null)
  ```

  Back in `src/actions/poll.ts`, update the `assignPollToSession` function:
  - After parsing, destructure `{ pollId, sessionId }`
  - Only verify session ownership if sessionId is NOT null (wrap the `prisma.classSession.findFirst` in `if (sessionId) { ... }`)
  - Call `assignPollToSessionDAL(pollId, teacher.id, sessionId)` -- sessionId can now be null

  This matches exactly how `assignBracketToSession` works in `src/actions/bracket.ts` (line 156-158: `sessionId: z.string().uuid().nullable()`).
  </action>

  </verify>
