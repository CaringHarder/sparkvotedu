**Create src/hooks/use-realtime-poll.ts** -- Following use-realtime-bracket.ts pattern exactly:
- useMemo for Supabase client (prevent duplicate subscriptions, per 02-05 decision)
- Subscribe to channel `poll:${pollId}`
- Listen for 'poll_vote_update' events: payload has {voteCounts: Record<string,number>, totalVotes: number}
- Listen for 'poll_update' events: payload has {type: 'poll_closed' | 'poll_activated' | 'poll_archived'}
- useRef accumulator for batching vote updates (per 04-03 pattern): accumulate pending vote counts, flush every batchIntervalMs (default 2000ms)
- poll_update events trigger immediate full refetch via /api/polls/${pollId}/state
- Transport fallback: 5s WebSocket timeout, fall back to 3s polling interval (per 04-03 decision)
- fetchPollState callback: GET /api/polls/${pollId}/state, sets voteCounts, totalVotes, pollStatus, bordaScores (if ranked)
- Initial fetch on SUBSCRIBED
- Cleanup: removeChannel, clearInterval, clearTimeout
- Returns: {voteCounts, totalVotes, pollStatus, bordaScores, transport, refetch}
- For ranked polls: bordaScores in state (fetched from state endpoint which computes them server-side)

**Create src/components/poll/bar-chart.tsx** -- Animated horizontal bar chart:
- 'use client' component, import { motion, AnimatePresence } from 'motion/react'
- Props: data ({optionId, label, count, color}[]), total (total votes)
- Each bar: motion.div with spring animation on width change
- Spring config: stiffness: 200, damping: 15, mass: 0.8 (bouncy feel)
- Bar width proportional to max count (not total): widest bar fills container, others scale relative
- Show percentage label at right end of each bar
- Show count in bar (if bar wide enough) or beside bar
- Staggered entry: delay each bar by index * 0.05s
- Color palette: array of 8 distinct colors (indigo, emerald, amber, rose, sky, violet, orange, teal), cycling for options
- Responsive: labels truncate on mobile, full text on desktop

**Create src/components/poll/donut-chart.tsx** -- Animated donut/pie chart:
- 'use client' component, import { motion } from 'motion/react'
- Props: data ({optionId, label, count, color}[]), total
- SVG-based: viewBox "0 0 200 200", center at (100, 100), radius 80, inner radius 50 (donut hole)
- Each segment: motion.path with SVG arc commands
- Calculate start/end angles from cumulative percentages
- Spring animation on pathLength (from 0 to segment arc)
- Center text: total votes count
- Legend below chart: colored dots with labels and percentages
- Handle 0 votes: show empty ring with "No votes yet" text

**Create src/components/poll/ranked-leaderboard.tsx** -- Borda count leaderboard:

- AnimatePresence for position changes (options may reorder as votes come in)
- motion.div with layout animation for smooth position transitions
- Show "X of Y possible points" for each option
- Show voter count per option
