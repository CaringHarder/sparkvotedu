3. Create `src/actions/profile.ts` with three server actions following the existing useActionState pattern (see `src/actions/auth.ts` for reference):

   **updateDisplayName** -- accepts FormData with `name` field:
   - Call `getAuthenticatedTeacher()` for auth check
   - Validate with Zod: `name` must be string, min 1, max 100, trimmed
   - Update via `prisma.teacher.update({ where: { id: teacher.id }, data: { name: parsed.data.name.trim() } })`
   - Call `revalidatePath('/profile')`
   - Return `{ success: 'Display name updated.' }` on success, `{ error: '...' }` on failure

   **changePassword** -- accepts FormData with `currentPassword`, `newPassword`, `confirmPassword`:
   - Call `getAuthenticatedTeacher()` for auth check
   - Validate with Zod: currentPassword required, newPassword min 8 characters (no complexity rules per user decision), confirmPassword must match via `.refine()`
   - Verify current password by calling `supabase.auth.signInWithPassword({ email: teacher.email, password: parsed.data.currentPassword })`. If error, return `{ error: 'Current password is incorrect' }`
   - Update password via `supabase.auth.updateUser({ password: parsed.data.newPassword })`
   - Return `{ success: 'Password changed successfully.' }` -- stay logged in (Supabase updateUser keeps session valid per user decision: lean toward least disruptive)

   **forceSetPassword** -- accepts FormData with `newPassword`, `confirmPassword` (no current password needed for forced reset):
   - Call `getAuthenticatedTeacher()` for auth check
   - Validate with Zod: newPassword min 8 characters, confirmPassword must match
   - Update password via `supabase.auth.updateUser({ password: parsed.data.newPassword })`
   - Clear the forced reset flag: `prisma.teacher.update({ where: { id: teacher.id }, data: { mustChangePassword: false } })`
   - IMPORTANT: Clear the flag BEFORE redirect to avoid race condition (proxy re-checking flag). Use `await` on both operations.
   - Call `redirect('/dashboard')` -- teacher lands directly on dashboard with no re-login (per user decision)

  All actions use `'use server'` directive. Import `getAuthenticatedTeacher` from `@/lib/dal/auth`, `prisma` from `@/lib/prisma`, `createClient` from `@/lib/supabase/server`, `z` from `zod`, `revalidatePath` from `next/cache`, `redirect` from `next/navigation`.
