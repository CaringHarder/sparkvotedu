- Single localStorage key: `sparkvotedu_identities`
- Schema version field `v: 1` for future migration

Exports:
- `StoredIdentity` interface: `{ participantId, funName, emoji, firstName, lastInitial, sessionId, joinedAt }`

- `setStoredIdentity(identity: StoredIdentity): void` -- upserts identity into store by sessionId, prunes after write
- `clearStoredIdentity(sessionId: string): void` -- removes single session entry from store

Pruning logic runs on every `getStoredIdentity` call: filter out entries older than TTL_MS, sort by joinedAt descending, slice to MAX_ENTRIES, write pruned store back.

**2c. Add confirmation handlers:**

```typescript
const handleConfirmIdentity = useCallback(async () => {
  if (step.type !== 'localStorage-confirm') return
  setLoading(true)
  setError(null)

  const result = await rejoinWithStoredIdentity({
    participantId: step.stored.participantId,
    sessionId: sessionInfo.id,
  })

  if (result.error) {
    // Stored identity is stale -- clear it, fall through to wizard
    clearStoredIdentity(sessionInfo.id)
    setError(null)  // Don't show error, just start fresh
    dispatch({ type: 'DENY_IDENTITY' })
    setLoading(false)
    return
  }

  if (result.participant) {
    // Set sessionStorage for in-session use
    setSessionParticipant(sessionInfo.id, {
      participantId: result.participant.id,
      firstName: result.participant.firstName,
      funName: result.participant.funName,
      sessionId: sessionInfo.id,
      rerollUsed: result.participant.rerollUsed,
      emoji: result.participant.emoji,
      lastInitial: result.participant.lastInitial,
    })

    // Update localStorage with fresh server data
    setStoredIdentity({
      participantId: result.participant.id,
      funName: result.participant.funName,
      emoji: result.participant.emoji,
      firstName: result.participant.firstName,
      lastInitial: result.participant.lastInitial,
      sessionId: sessionInfo.id,
      joinedAt: Date.now(),
    })

    // Navigate directly to session (skip welcome screen -- they already know who they are)
    router.push(`/session/${sessionInfo.id}`)
  }

  setLoading(false)
}, [step, sessionInfo.id, router])

const handleDenyIdentity = useCallback(() => {
```
