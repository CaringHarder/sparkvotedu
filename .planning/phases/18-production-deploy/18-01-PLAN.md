  <name>Task 1: Pre-deploy build verification and schema sync</name>
  <files>next.config.ts</files>
  <action>
Verify the codebase is production-ready before deploying:

1. **Run the build locally** to catch any TypeScript or build errors:
   ```
   npm run build
   ```
   The build script runs `prisma generate && next build`. Fix any build errors if found.

2. **Check Prisma schema sync**: The project uses `prisma db push` (no migrations directory). Since both old and new sites share the same Supabase database, the schema should already be in sync. Verify by running:
   ```
   npx prisma db push --dry-run 2>&1
   ```
   - If it reports "no changes", schema is in sync -- no action needed.
   - If it reports pending changes, those changes were likely already applied during development. Document what the dry-run reports but do NOT run `db push` against production without explicit confirmation.

3. **Check next.config.ts for Supabase Storage hostname**: Poll images are served from Supabase Storage. If any `next/image` components reference Supabase Storage URLs, the Supabase hostname (*.supabase.co) needs to be in `remotePatterns`. Search for Image components that use Supabase URLs:
   ```
   grep -r "supabase.*storage\|getPublicUrl" src/ --include="*.tsx" --include="*.ts"
   ```
   If Supabase image URLs are used with next/image, add the hostname to next.config.ts `remotePatterns`. If they use plain `<img>` tags or aren't rendered via next/image, no change needed.

4. **Verify git is clean and on main**: Ensure all changes are committed and the working tree is clean for deployment.
  </action>
  <verify>
- `npm run build` completes successfully with exit code 0
- `npx prisma db push --dry-run` reports status (document output)
- Git working tree is clean on main branch
  </verify>
  <done>Build succeeds locally, schema sync status is documented, codebase is ready for Vercel deployment.</done>
