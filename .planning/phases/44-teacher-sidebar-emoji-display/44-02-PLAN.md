    - Uses shadcn `Dialog` (already installed, NOT Popover -- per research recommendation to avoid adding dependencies)
    - Props: `open: boolean`, `onOpenChange: (open: boolean) => void`, `participantId: string`, `currentFirstName: string`, `funName: string`, `emoji: string | null`

    - Save button calls `teacherUpdateStudentName` server action

    **C. Create teacherUpdateStudentName server action** (add to `src/actions/student.ts`):
    - Signature: `export async function teacherUpdateStudentName(input: { participantId: string; firstName: string }): Promise<{ error?: string; success?: boolean }>`
    - Auth: call `getAuthenticatedTeacher()` -- return error if null
    - Validate firstName with `validateFirstName`
    - Fetch participant with `prisma.studentParticipant.findUnique({ where: { id }, include: { session: { select: { teacherId: true } } } })`
    - Verify `participant.session.teacherId === teacher.id` -- return error if not
    - Update: `prisma.studentParticipant.update({ where: { id }, data: { firstName } })`
    - Broadcast update: call `broadcastParticipantJoined(participant.sessionId)` so student sees update in real time (reuse existing broadcast -- semantic purity is less important than shipping, per research)
    - Return `{ success: true }`

    **D. Update ParticipationSidebar** (`src/components/teacher/participation-sidebar.tsx`):
    - Import `NameViewToggle` and `TeacherEditNameDialog` and `shortcodeToEmoji` from emoji-pool
    - Add local state: `const [nameView, setNameView] = useState<'fun' | 'real'>(teacherNameViewDefault ?? 'fun')`
    - Add state for edit dialog: `const [editingParticipant, setEditingParticipant] = useState<{ id: string; firstName: string; funName: string; emoji: string | null } | null>(null)`
    - In the sidebar header area (after "Student Activity" heading, before vote summary), render `<NameViewToggle value={nameView} onChange={setNameView} />`
    - Update each student tile to be clickable: wrap in `<button onClick={() => setEditingParticipant({ id, firstName, funName, emoji })}>` (styled as current div, cursor-pointer added)
    - **Fun name view** (nameView === 'fun'): Show status dot + emoji character (via `shortcodeToEmoji`) + fun name. Use sparkles fallback when emoji is null. Remove the firstName subtitle.
    - **Real name view** (nameView === 'real'): Show status dot + firstName + lastInitial (e.g., "Sarah T."). No emoji. If firstName is empty, show funName as fallback.
    - Render `<TeacherEditNameDialog>` at bottom of component, controlled by `editingParticipant` state
    - Keep sorting logic unchanged (voted first, then connected, then disconnected)
