<feature>
  <name>Placement Pure Functions</name>
  <files>src/lib/bracket/placement.ts, src/lib/bracket/__tests__/placement.test.ts</files>
  <behavior>
    The placement module provides these pure functions:

    **buildSeedOrder must be exported from engine.ts first** -- it is currently a private function. Export it without changing its implementation.

    1. **buildSlotMap(bracketSize: number): number[]**
       Returns the seed order array from buildSeedOrder. This is the canonical mapping:
       - slotIndex -> seedPosition: `seedOrder[slotIndex]`
       - seedPosition -> slotIndex: `seedOrder.indexOf(seedPosition)`
       Example: buildSlotMap(8) = [1, 8, 4, 5, 2, 7, 3, 6]
         Slot 0 -> Seed 1, Slot 1 -> Seed 8, Slot 2 -> Seed 4, etc.

    2. **seedToSlot(seedPosition: number, bracketSize: number): number**
       Returns the bracket slot index (0-based) for a given seedPosition.
       Example: seedToSlot(4, 8) = 2 (seed 4 is in slot index 2)

    3. **slotToSeed(slotIndex: number, bracketSize: number): number**
       Returns the seedPosition for a given bracket slot index (0-based).
       Example: slotToSeed(2, 8) = 4 (slot index 2 holds seed 4)

    4. **swapSlots(entrants: PlacementEntrant[], slotA: number, slotB: number, bracketSize: number): PlacementEntrant[]**
       Swaps the entrants in two bracket slots by updating their seedPositions.
       Returns a new array (immutable). Works for entrant-entrant swaps, entrant-bye swaps, and bye-bye swaps.
       PlacementEntrant = { id: string; name: string; seedPosition: number }

    5. **autoSeed(entrants: PlacementEntrant[]): PlacementEntrant[]**
       Returns entrants with seedPosition = index + 1 (natural list order).
       This is the "auto-seed" button behavior.

    6. **getByeSlots(entrantCount: number, bracketSize: number): number[]**
       Returns the slot indices (0-based) that are bye positions.
       Uses calculateBracketSizeWithByes to find bye seeds, then maps to slots.
       Example: getByeSlots(5, 8) returns the slot indices where seeds 6, 7, 8 would be.

    7. **placeEntrantInSlot(entrants: PlacementEntrant[], entrantId: string, targetSlot: number, bracketSize: number): PlacementEntrant[]**
       Places an entrant from the pool into a specific slot. If the slot is occupied (by another entrant or bye), swap them.
       The entrant's seedPosition is updated to the seedPosition that maps to the target slot.

    8. **removeEntrantFromSlot(entrants: PlacementEntrant[], entrantId: string): PlacementEntrant[]**
       No-op for this system since entrants are always in the list (seedPosition always maps somewhere).
       Instead, this resets the entrant's seedPosition to its original value (pre-placement).
       Actually -- removing an entrant from a slot in visual mode means "send back to pool". Since the pool IS the entrant list minus placed entrants, this function is unnecessary. The UI handles pool vs placed state by checking if the entrant is in a "placed" set.

    Actually, simplify: the entrant list IS the source of truth. Every entrant always has a seedPosition. Visual placement is a different VIEW of the same data. There is no separate "pool" data structure -- the pool is entrants whose seedPosition hasn't been manually placed yet.

    Define a **PlacementEntrant** interface:
    ```typescript
    interface PlacementEntrant {
      id: string
      name: string
      seedPosition: number
    }
    ```

    - slotToSeed(0, 8) = 1

    - swapSlots where one slot is a bye position
