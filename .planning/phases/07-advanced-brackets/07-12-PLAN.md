Create src/components/bracket/prediction-leaderboard.tsx:

Props:
```typescript
interface PredictionLeaderboardProps {
  bracketId: string
  scores: PredictionScore[]
  totalRounds: number
  isTeacher: boolean
  participantId?: string  // current student's ID (to highlight their row)
  matchupStats?: Record<string, { entrant1Predictions: number; entrant2Predictions: number; total: number }>
}
```

1. Student view (isTeacher=false):
   - Table with columns: Rank, Name, Score
   - Current student's row highlighted with primary color background
   - Gold/silver/bronze styling for top 3 (matching poll leaderboard from 05-04: amber-400, gray-300, amber-600)
   - No per-round breakdown visible (just rank + total)
   - Show total participants count at bottom

2. Teacher view (isTeacher=true):
   - Table with columns: Rank, Name, Score, Correct Picks, Accuracy %
   - Each row is expandable (click to expand)
   - Expanded row shows per-round breakdown:
     - For each round: "Round {N} ({pointsPerRound}pts each): {correct}/{total} = {points}pts"
     - Correct picks highlighted green (bg-green-50 text-green-700)
     - Wrong picks highlighted red (bg-red-50 text-red-700)
   - Optional: show the actual predictions for each matchup:
     - "Match: [A] vs [B] - Predicted: [A] - Actual: [B]" with green/red indicator
     - This is in a collapsible sub-section within the expanded row

3. Per-matchup prediction stats (shown in both views, but only for resolved matchups):
   - Below the leaderboard or as a separate section
   - For each resolved matchup: "23 of 30 predicted [Winner]" or a visual bar showing prediction distribution
   - Only show for matchups that have been resolved (not upcoming ones -- predictions are hidden)

4. Live updates:
   - Use the usePredictions hook to receive real-time leaderboard updates
   - When scores change, animate rank changes (subtle slide-up/slide-down using motion library)
   - New resolved matchups trigger a leaderboard recalculation

5. Empty state:
   - Before any matchups are resolved: "No matchups resolved yet. Leaderboard will appear as matches are decided."
   - If no predictions exist: "No predictions submitted yet."

Styling:
- Use Tailwind classes consistent with existing components
- Card wrapper matching existing bracket detail page cards
- Responsive: table on desktop, stacked cards on mobile
