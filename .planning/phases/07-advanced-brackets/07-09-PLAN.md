   ```typescript
     matchups: MatchupData[]

   }
   ```

2. Filter matchups by bracketRegion:
   ```typescript
   const winnersMatchups = matchups.filter(m => m.bracketRegion === 'winners')
   const losersMatchups = matchups.filter(m => m.bracketRegion === 'losers')
   const grandFinalsMatchups = matchups.filter(m => m.bracketRegion === 'grand_finals')
   ```

3. Tab state management:
   - 4 tabs: Winners, Losers, Grand Finals, Overview
   - Grand Finals tab hidden (not just disabled) until both GF entrants are set
   - Each tab shows a badge with active matchup count (status === 'voting')
   - Use a segmented control UI matching the existing bracket form tab pattern (custom tabs, not Radix)

4. Winners tab content:
   - Render winnersMatchups using the existing BracketDiagram component
   - Pass only winners matchups and the entrants they reference
   - Label: "Winners Bracket"

5. Losers tab content:
   - Render losersMatchups using BracketDiagram
   - The losers bracket has a different round structure (minor/major alternation)
   - Use the same BracketDiagram component but with losers-specific matchup data
   - Label: "Losers Bracket"
   - Entrants in the losers bracket who dropped from winners should have a visual indicator:
     - Add a small badge or color tint (e.g., amber/orange border or a "dropped" label)
     - Detect "dropped" entrants: they appear in LB but have no LB R1 seed (they were placed dynamically)

6. Grand Finals tab content:
   - Show the grand finals matchup(s) in a centered card layout (not a full bracket diagram)
   - If only match 1: show a single matchup card with WB Champion vs LB Champion
   - If reset match exists (match 2): show both matchup cards stacked vertically
   - Label: "Grand Finals"

7. Overview tab content:
   - Show a list of remaining entrants (those not yet eliminated from either bracket)
   - Group by "Still in Winners" and "In Losers Bracket"
   - Eliminated entrants shown dimmed at the bottom
   - This is a table/list view, not an SVG diagram

8. Styling: Use Tailwind classes matching existing bracket UI. Tabs should be visually consistent with the bracket form's segmented control tabs.

9. Wrap the Winners and Losers tab SVG diagrams with BracketZoomWrapper if the bracket has 32+ entrants.
