<task type="auto">
  <name>Task 2: Double-elimination advancement with loser drop and grand finals</name>
  <files>src/lib/bracket/advancement.ts, src/actions/bracket-advance.ts</files>
  <action>
1. Add advanceDoubleElimMatchup function to advancement.ts:

   This function wraps the existing advanceMatchupWinner with double-elim-specific logic:

   a. Call advanceMatchupWinner normally (sets winner, propagates to next matchup in same region)
   b. ADDITIONALLY, if the matchup is in the winners bracket (bracketRegion='winners'):
      - The LOSER must drop to the losers bracket
      - Determine which LB matchup to drop into:
        - Use the seedLosersFromWinnersRound function from double-elim.ts to determine the correct LB position
        - WB round R losers drop into LB major round (the round that accepts WB dropdowns)
        - The specific LB slot depends on the WB matchup position and the split-and-reverse seeding
      - Place the loser into the correct entrant slot of the target LB matchup

   c. If the matchup is the final matchup in losers bracket (LB champion):
      - Place the LB champion into grand finals matchup entrant2 slot

   d. If the matchup is the final matchup in winners bracket (WB champion):
      - Place the WB champion into grand finals matchup entrant1 slot

   e. If the matchup is grand finals match 1:
      - If WB champion wins: bracket is complete (no reset needed). Set bracket status to 'completed'.
      - If LB champion wins: create a RESET matchup dynamically:
        - Create a new matchup with bracketRegion='grand_finals', round=2, position=1
        - Set entrant1=WB champion, entrant2=LB champion
        - This is the "if necessary" match. It was NOT created upfront (per CONTEXT.md decision).

   f. If the matchup is grand finals match 2 (reset): winner is the bracket champion. Set bracket status to 'completed'.

   All mutations in a single $transaction.

2. Add server actions to bracket-advance.ts:
   - advanceDoubleElimMatchup action: auth -> validate -> call advanceDoubleElimMatchup -> broadcast -> revalidate
   - This action is distinct from the existing advanceMatchup action to avoid cluttering single-elim logic
   - The action checks bracketType and routes to the correct advancement function

   Alternative approach (preferred): Update the existing advanceMatchup action to detect bracketType and call the appropriate advancement function. This keeps a single action endpoint:
   ```typescript
   if (bracket.bracketType === 'double_elimination') {
     result = await advanceDoubleElimMatchup(matchupId, winnerId, bracketId)
   } else {
     result = await advanceMatchupWinner(matchupId, winnerId, bracketId)
   }
   ```

3. Add a helper to find the correct LB target matchup for a WB loser:
   This needs to:
   - Know which WB round the loser came from
   - Map WB round R to the LB major round that accepts WB R losers
   - Map WB matchup position to LB matchup position using split-and-reverse
   - Query the DB for the target LB matchup by bracketId + bracketRegion='losers' + round + position
  </action>
