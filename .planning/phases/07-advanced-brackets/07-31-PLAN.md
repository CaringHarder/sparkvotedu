In the component body, after computing all positions, if `mirrorX` is true, flip X coordinates:
```typescript
// If mirrorX, flip all X positions so rounds go right-to-left
const effectiveGetMatchPosition = useCallback((round: number, position: number, totalRounds: number) => {
  const pos = getMatchPosition(round, position, totalRounds)
  if (!mirrorX) return pos
  const svgWidth = PADDING * 2 + totalRounds * MATCH_WIDTH + (totalRounds - 1) * ROUND_GAP
  return {
    x: svgWidth - pos.x - MATCH_WIDTH,
    y: pos.y,
  }
}, [mirrorX])
```

Use `effectiveGetMatchPosition` wherever `getMatchPosition` is called for rendering matchup boxes and connectors.

The simplest fix: `getConnectorPath` already takes coordinates, so as long as the x1/x2 values are mirrored, the connector paths will naturally draw in the correct direction. Key implementation:

1. **Split matchups into quadrants.** For a 64-entrant bracket (6 rounds total), rounds 1-4 are quadrant rounds and rounds 5-6 are connecting rounds.

Split round 1 matchups (32 total, positions 1-32) into 4 groups of 8:
- TL: round 1 positions 1-8 (and their downstream matchups in rounds 2-4)
- TR: round 1 positions 9-16 (and downstream)
- BL: round 1 positions 17-24 (and downstream)
- BR: round 1 positions 25-32 (and downstream)

For rounds 2+, a matchup belongs to a quadrant based on which round-1 matchups feed into it. Use the `nextMatchupId` chain: trace each round-1 matchup forward through rounds 2, 3, 4 to determine quadrant membership.

Simpler alternative: Use position ranges. In a standard bracket, round N positions P feed into round N+1 positions based on `Math.ceil(P/2)`. So:
- Round 1 positions 1-8 -> Round 2 positions 1-4 -> Round 3 positions 1-2 -> Round 4 position 1
- Round 1 positions 9-16 -> Round 2 positions 5-8 -> Round 3 positions 3-4 -> Round 4 position 2
- Round 1 positions 17-24 -> Round 2 positions 9-12 -> Round 3 positions 5-6 -> Round 4 position 3
- Round 1 positions 25-32 -> Round 2 positions 13-16 -> Round 3 positions 7-8 -> Round 4 position 4

Build a quadrant assignment function:
```typescript
function getQuadrant(round: number, position: number): 'TL' | 'TR' | 'BL' | 'BR' | 'connecting' {
  if (round > 4) return 'connecting' // Rounds 5-6 are semifinals/finals
  // Trace back to round 1 position
  let r1Position = position
  for (let r = round; r > 1; r--) {
    r1Position = (r1Position - 1) * 2 + 1 // First feeder position
  }
  // r1Position is now the FIRST round-1 position that feeds into this matchup
  if (r1Position <= 8) return 'TL'
  if (r1Position <= 16) return 'TR'
  if (r1Position <= 24) return 'BL'
  return 'BR'
}
```

2. **Filter and normalize matchups per quadrant.** Each quadrant gets matchups from rounds 1-4 only, with positions renormalized to 1-based within the quadrant:
```typescript
function normalizeQuadrant(quadrantMatchups: MatchupData[]): MatchupData[] {
  // Find min position per round and subtract to normalize to 1-based
  const byRound = new Map<number, MatchupData[]>()
  for (const m of quadrantMatchups) {
    const arr = byRound.get(m.round) || []
    arr.push(m)
    byRound.set(m.round, arr)
  }

  return quadrantMatchups.map((m) => {
    const roundMatchups = byRound.get(m.round)!
    const minPos = Math.min(...roundMatchups.map((r) => r.position))
    return { ...m, position: m.position - minPos + 1 }
  })
}
```

Helper to filter voteLabels to only matchups in a quadrant:
```typescript
function filterVoteLabels(
  labels: Record<string, Record<string, number>> | undefined,
  quadrantMatchups: MatchupData[]
): Record<string, Record<string, number>> | undefined {
  if (!labels) return undefined
  const ids = new Set(quadrantMatchups.map((m) => m.id))
  const filtered: Record<string, Record<string, number>> = {}
  for (const [k, v] of Object.entries(labels)) {
    if (ids.has(k)) filtered[k] = v
  }
  return Object.keys(filtered).length > 0 ? filtered : undefined
}
```
