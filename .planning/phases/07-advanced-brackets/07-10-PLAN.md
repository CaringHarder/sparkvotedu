  <name>Task 1: Round-robin DAL and server actions</name>
  <files>src/lib/dal/round-robin.ts, src/actions/round-robin.ts</files>
  <action>
1. Create src/lib/dal/round-robin.ts:

   a. createRoundRobinBracketDAL(teacherId, data, entrants):
      - Import generateRoundRobinRounds from '@/lib/bracket/round-robin'
      - Create Bracket record with bracketType='round_robin', roundRobinPacing, roundRobinVotingStyle, roundRobinStandingsMode from data
      - Create entrant records
      - Generate rounds using generateRoundRobinRounds(entrants.length)
      - Create Matchup records for each round-robin matchup:
        - round = the generated round number
        - position = index within the round
        - roundRobinRound = round number (for filtering)
        - entrant1Id and entrant2Id mapped from seeds via entrantIdBySeed
        - No nextMatchupId (round-robin matchups are independent, not chained)
        - bracketRegion = null (not applicable for round-robin)
        - status = 'pending'
      - Return full bracket with matchups

   b. recordRoundRobinResult(matchupId, winnerId, teacherId):
      - Verify matchup ownership (bracket.teacherId = teacherId)
      - Set matchup.winnerId = winnerId (or null for tie -- but the Matchup model uses winnerId as the winner indicator)
      - For ties: set winnerId = null AND status = 'decided' (a tie is still a decided matchup)
      - For wins: set winnerId = winner's entrantId AND status = 'decided'
      - Broadcast bracket update
      - Return updated matchup

   c. getRoundRobinStandings(bracketId):
      - Query all decided matchups for the bracket
      - Map to RoundRobinResult[] format: { entrant1Id, entrant2Id, winnerId }
      - Call calculateRoundRobinStandings from round-robin.ts engine
      - Enrich with entrant names from bracket entrants
      - Return RoundRobinStanding[]

   d. advanceRoundRobinRound(bracketId, roundNumber, teacherId):
      - For round-by-round pacing: open matchups for the next round
      - Find all matchups where roundRobinRound = roundNumber and status = 'pending'
      - Set their status to 'voting'
      - Broadcast bracket update

2. Create src/actions/round-robin.ts:
   'use server'

   a. recordResult action:
      - Auth -> validate (recordRoundRobinResultSchema) -> call recordRoundRobinResult DAL -> revalidate
      - Return { success: true } or { error: string }

   b. advanceRound action:
      - Auth -> validate -> call advanceRoundRobinRound DAL -> revalidate
      - For round-by-round: opens next round's matchups for voting
      - For all-at-once: opens all matchups at once

   c. getRoundRobinStandingsAction:
      - Auth -> call getRoundRobinStandings DAL -> return standings data
      - Used by client components via server action

3. Also update src/lib/dal/bracket.ts > createBracketDAL:
   - Add bracketType branching: when bracketType === 'round_robin', call createRoundRobinBracketDAL instead of the single-elim path
   - Similarly for double_elimination (already done in 07-08) and predictive (done in 07-11)
  </action>
