<feature>
  <name>Bye Placement Algorithm</name>
  <files>src/lib/bracket/byes.ts, src/lib/bracket/__tests__/byes.test.ts</files>
  <behavior>
  Pure functions with deterministic outputs:

  calculateBracketSizeWithByes(entrantCount):
  - 3 entrants -> { bracketSize: 4, numByes: 1, byeSeeds: [1] }
  - 5 entrants -> { bracketSize: 8, numByes: 3, byeSeeds: [1, 2, 3] }
  - 6 entrants -> { bracketSize: 8, numByes: 2, byeSeeds: [1, 2] }
  - 7 entrants -> { bracketSize: 8, numByes: 1, byeSeeds: [1] }
  - 8 entrants -> { bracketSize: 8, numByes: 0, byeSeeds: [] }
  - 10 entrants -> { bracketSize: 16, numByes: 6, byeSeeds: [1,2,3,4,5,6] }
  - 4 entrants -> { bracketSize: 4, numByes: 0, byeSeeds: [] }
  - 64 entrants -> { bracketSize: 64, numByes: 0, byeSeeds: [] }

  generateMatchupsWithByes(entrantCount):
  - Returns MatchupSeedWithBye[] (extends MatchupSeed with isBye field)
  - For 5 entrants: generates 8-team bracket structure (7 matchups)
  - Round 1 matchups where one seed > entrantCount have that seed replaced with null and isBye=true
  - For 5 entrants in 8-bracket: seeds 6, 7, 8 become BYE slots
    - Match R1P1: seed 1 vs BYE(8) -> isBye=true, entrant2Seed=null
    - Match R1P2: seed 4 vs seed 5 -> isBye=false, both seeds present
    - Match R1P3: seed 2 vs BYE(7) -> isBye=true, entrant2Seed=null
    - Match R1P4: seed 3 vs BYE(6) -> isBye=true, entrant2Seed=null
  - Power-of-two inputs (4, 8, 16, 32, 64): no byes, all matchups have isBye=false
  - All matchups preserve nextMatchupPosition for advancement chain
  - The existing generateMatchups function is reused for the full bracket structure
  </behavior>
  <implementation>
  1. calculateBracketSizeWithByes: Math.pow(2, Math.ceil(Math.log2(entrantCount))) for bracket size. numByes = bracketSize - entrantCount. byeSeeds = range 1..numByes (top seeds get byes).

  2. generateMatchupsWithByes: Call generateMatchups(bracketSize) from engine.ts. Map over round 1 matchups: if either seed > entrantCount, replace that seed with null and set isBye=true. Return full array with isBye field on all matchups (false for non-bye, false for all later rounds).

  Both functions must be PURE (no DB, no side effects). Import generateMatchups from engine.ts.
  </implementation>
</feature>
