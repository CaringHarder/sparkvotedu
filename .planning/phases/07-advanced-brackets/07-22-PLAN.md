    3. **Fix round advancement visibility:**
       The current `canAdvanceRoundRobin` logic checks if all current-round matchups are decided AND a next round exists. This is correct.

       But the issue is the `currentRoundRobinRound` calculation: it finds the FIRST non-decided matchup's roundRobinRound. If all matchups in round N are decided but round N+1 has pending matchups, it jumps to N+1. The `canAdvanceRoundRobin` then checks if all N+1 matchups are decided (they're pending) and returns false.

       Fix: `currentRoundRobinRound` should be the LOWEST roundRobinRound that has ANY non-decided matchup. But also, after all matchups in round N are decided, we need to show the "Open Round N+1" button even before the round is formally advanced.

       Actually, review the existing logic more carefully:
       - `needsRound1Open` handles the R1 edge case.
       - `canAdvanceRoundRobin` checks if all current-round matchups are decided AND next round exists. This should trigger the "Open Round N+1" button.
       - The button is at line 448: `isRoundRobin && canAdvanceRoundRobin` -> shows "Open Round {N+1}".

       The bug may be that `currentRoundRobinRound` jumps ahead when it shouldn't. If round 1 is all decided, `currentRoundRobinRound` becomes the round of the first non-decided matchup. If round 2 matchups are still pending, it would be round 2. Then `canAdvanceRoundRobin` checks if all round 2 matchups are decided -- they're not (they're pending), so it returns false.

       Fix `currentRoundRobinRound` to be:
       ```typescript
       const currentRoundRobinRound = useMemo(() => {
         if (!isRoundRobin) return 1
         // Find the lowest RR round where ANY matchup has been opened (voting or decided)
         const activeRounds = currentMatchups
           .filter(m => m.roundRobinRound != null && m.status !== 'pending')
           .map(m => m.roundRobinRound!)
         if (activeRounds.length === 0) return 1
         // Current round is the max active round
         return Math.max(...activeRounds)
       }, [isRoundRobin, currentMatchups])
       ```

       This way: if round 1 has voting/decided matchups, currentRoundRobinRound = 1. After all are decided and round 2 is opened, it becomes 2.

       Also ensure `canAdvanceRoundRobin` works correctly: after all round 1 matchups are decided, it should detect that round 2 exists and has pending matchups, showing the advance button.

       Actually, re-analyze the logic chain:
       - After round 1 all decided, `currentRoundRobinRound` (with fix) = 1 (highest active round).
       - `canAdvanceRoundRobin`: checks if all round 1 matchups are decided (yes) AND next round exists (yes) -> true.
       - Button shows "Open Round 2" and calls `handleAdvanceRoundRobin` which calls `advanceRound({ bracketId, roundNumber: 2 })`.
       - This sets round 2 matchups from pending to voting.

       This chain should work correctly with the `currentRoundRobinRound` fix. Apply the fix.
