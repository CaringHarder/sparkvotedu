<task type="auto">

  <files>src/actions/bracket-advance.ts</files>
  <action>
Add a new exported async function `undoRoundAdvancement` to `src/actions/bracket-advance.ts`. Follow the existing pattern of `advanceMatchup` and `batchAdvanceRound`.

Add imports at the top:
- Import `undoRoundSE, undoRoundRR, undoRoundDE, undoRoundPredictive, getMostRecentAdvancedRound` from `@/lib/bracket/advancement`
- Import `undoRoundSchema` from `@/lib/utils/validation`
- Import `updateBracketStatusDAL` from `@/lib/dal/bracket` (for auto-pause)

**Function: `undoRoundAdvancement(input: unknown)`**

Flow:
1. **Auth:** `getAuthenticatedTeacher()` -- return error if not authenticated
2. **Validate:** `undoRoundSchema.safeParse(input)` -- return error if invalid
3. **Ownership + status check:** Query bracket:
   ```typescript
   const bracket = await prisma.bracket.findFirst({
     where: { id: bracketId, teacherId: teacher.id },
     select: { id: true, sessionId: true, bracketType: true, status: true, predictiveResolutionMode: true },
   })
   ```
   Return error if not found.
4. **Status guard:** Only allow undo on `active` or `paused` brackets. If bracket.status is `draft`, `completed`, or `archived`, return `{ error: 'Can only undo rounds on active or paused brackets' }`. EXCEPTION: if the bracket was JUST completed by the most recent advancement (i.e., the round being undone is the final round), allow it -- check by calling `getMostRecentAdvancedRound`. If getMostRecentAdvancedRound returns null for a completed bracket, that means it's truly done (Phase 31 scope). But if the bracket is completed and the final round matchups are all decided with no further state, allow undo by treating it as if it's still in progress.

   Simplification: allow `completed` status ONLY if `getMostRecentAdvancedRound` returns a non-null result for the bracket. This handles the "just completed" case.

5. **Auto-pause:** If bracket.status is `active`, pause it first:
   ```typescript
   if (bracket.status === 'active') {
     await updateBracketStatusDAL(bracketId, teacher.id, 'paused')
     broadcastBracketUpdate(bracketId, 'bracket_paused', {}).catch(console.error)
   }
   ```
   If bracket was `completed`, transition to `paused` (completed -> paused is not in VALID_TRANSITIONS, so use a direct prisma update here instead of the DAL, similar to `unarchiveBracketDAL` pattern which bypasses VALID_TRANSITIONS):
   ```typescript
   if (bracket.status === 'completed') {
     await prisma.bracket.update({
       where: { id: bracketId },
       data: { status: 'paused' },
     })
     broadcastBracketUpdate(bracketId, 'bracket_paused', {}).catch(console.error)
   }
   ```

6. **Validate round:** Call `getMostRecentAdvancedRound(bracketId, bracket.bracketType)`. Verify the requested round matches the returned undoable round. If mismatch, return `{ error: 'Round X is not the most recently advanced round' }`.

7. **Dispatch to type-specific engine:** Based on `bracket.bracketType`:
   - `'single_elimination'`: `await undoRoundSE(bracketId, round)`
   - `'round_robin'`: `await undoRoundRR(bracketId, round)`
   - `'double_elimination'`: `await undoRoundDE(bracketId, round, region!)` -- region is required for DE; return error if missing
   - `'predictive'`: `await undoRoundPredictive(bracketId, round)`
   - Any other type: return `{ error: 'Undo not supported for this bracket type' }`

8. **Broadcast:** `broadcastBracketUpdate(bracketId, 'round_undone', { round, region, bracketType: bracket.bracketType }).catch(console.error)`

9. **Session activity broadcast:** If bracket has sessionId, broadcast activity update: `broadcastActivityUpdate(bracket.sessionId).catch(console.error)`

10. **Revalidate:** `revalidatePath(\`/brackets/${bracketId}\`)` and `revalidatePath(\`/brackets/${bracketId}/live\`)`

11. **Return:** `{ success: true, ...undoResult }` where undoResult is the stats from the engine function.

Wrap the entire try block in try/catch, returning `{ error: message }` on failure (same pattern as existing actions).
