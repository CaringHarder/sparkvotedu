  <action>
Add three new exported functions at the bottom of `advancement.ts` (after the existing `advanceDoubleElimMatchup` function):

**1. `getMostRecentAdvancedRound(bracketId, bracketType)`**
- Queries all matchups for the bracket
- For SE/Predictive: finds the highest round where ALL matchups are `decided`, AND the next round (if any) has NO `voting` or `decided` matchups. If the next round has active voting, return null (undo window passed). If no decided rounds exist, return null.
- For DE: determines per-region. Query all matchups, group by region. For each region (winners, losers, grand_finals), find the highest round where all matchups are decided and the next round in that same region has no voting/decided matchups. Return `{ region, round }` for the most recently advanced region-round. Priority order: grand_finals > losers > winners (GF is the "latest" action in DE flow). If the bracket has a GF reset match that was just decided, that's the undoable target.
- For RR: finds the highest `roundRobinRound` where ALL matchups are `decided` AND the next `roundRobinRound` (if any) has no voting/decided matchups. Use `roundRobinRound` field, NOT `round`.
- Returns: `{ round: number, region?: string } | null`
- This is a read-only function (no transaction needed).

**2. `undoRoundSE(bracketId, round)`**
- Wrapped in `prisma.$transaction` with 30-second timeout (`{ timeout: 30000 }`).
- Steps inside transaction:
  1. Get all matchups in the target round with `status: 'decided'`
  2. Delete all votes for those matchups: `tx.vote.deleteMany({ where: { matchupId: { in: ids } } })`
  3. Clear winnerId and reset status to `'pending'` on those matchups
  4. For each matchup with `nextMatchupId`: clear the propagated entrant slot using `getSlotForPosition(matchup.position)` -> set that slot to null on the next matchup
  5. Cascade to downstream rounds: get all matchups with `round > targetRound`. Delete all their votes. Clear winnerId, entrant1Id, entrant2Id, set status to `'pending'` on all downstream matchups.
- Returns: `{ undoneMatchups: number, clearedVotes: number, cascadedMatchups: number }`

**3. `undoRoundRR(bracketId, roundRobinRound)`**
- Wrapped in `prisma.$transaction` with 30-second timeout.
- Steps inside transaction:
  1. Get all matchups where `bracketId` matches AND `roundRobinRound` equals the target round AND `status: 'decided'`
  2. Delete all votes for those matchups
  3. Clear winnerId and set status to `'pending'` on those matchups
  4. No cascade needed (RR matchups are independent -- no nextMatchupId, no entrant propagation between rounds)
- Returns: `{ undoneMatchups: number, clearedVotes: number }`
- NOTE: RR undo is simpler than SE because there's no entrant propagation between rounds.

Use the existing `TransactionClient` type alias already defined at the top of the file. Use the existing `getSlotForPosition` helper for SE entrant slot determination.

Add two more exported functions to `advancement.ts`:

**1. `undoRoundDE(bracketId, round, region)`**
- Wrapped in `prisma.$transaction` with 30-second timeout.
- Accept `region: 'winners' | 'losers' | 'grand_finals'` to identify which region's round is being undone.
- Fetch ALL matchups for the bracket (needed for cross-region cleanup).
- Compute `wbMaxRound`, `losersRoundOffset` (= wbMaxRound), `lbMinRound`, `lbMaxRound` from matchup data (same pattern as `advanceDoubleElimMatchup`).

**Region = 'winners':**
  1. Get WB matchups in target round with `status: 'decided'`
  2. Delete votes from those matchups
  3. Clear winnerId, set status to `'pending'`
  4. Clear propagated entrants in next WB round (via nextMatchupId + getSlotForPosition)
  5. Reverse loser placements in LB:
     - Compute `lbEngineRound = wbRoundToLbEngineRound(wbEngineRound)` where `wbEngineRound = round` (WB has no offset)
     - Compute `lbDbRound = lbEngineRound + losersRoundOffset`
     - Get LB matchups at `lbDbRound`
     - For WB round 1 losers: clear both entrant1Id and entrant2Id on LB round 1 matchups (losers filled both slots)
     - For WB round > 1 losers: clear entrant2Id on the LB major round matchups (WB dropdowns always go to entrant2Id)
  6. Check if undoing WB final (`round === wbMaxRound`): if so, clear entrant2Id on the LB final matchup (WB final loser goes to LB final entrant2)
  7. Cascade: delete votes and clear all matchups in LB rounds >= lbDbRound that were affected, and all GF matchups. Delete any GF reset matchup records (`tx.matchup.deleteMany` for GF matchups that are dynamically created reset matches -- these have no nextMatchupId and round > the base GF round).
  8. Cascade: delete votes and clear all WB matchups in rounds > target round.

**Region = 'losers':**
  1. Get LB matchups in target round with `status: 'decided'`
  2. Delete votes, clear winnerId, set status to `'pending'`
  3. Clear propagated entrants in next LB round via nextMatchupId
  4. If undoing LB final (round === lbMaxRound): clear entrant2Id on the GF matchup (LB champion goes to GF entrant2)
  5. Cascade: clear all LB matchups in rounds > target round, and all GF matchups. Delete any GF reset matchup.

**Region = 'grand_finals':**
  1. Get GF matchups sorted by round ascending
  2. If multiple GF matchups exist (reset match was created):
     - If undoing the reset match (highest round): DELETE the reset matchup record entirely (`tx.matchup.delete`), delete its votes. Clear GF match 1 winner if also decided? No -- only delete the reset match. The reset match IS the target round.
     - If undoing GF match 1: clear winnerId, delete votes, set status to 'pending'. Also delete any existing reset match.
  3. If single GF matchup: clear winnerId, delete votes, set status to 'pending'

- Returns: `{ undoneMatchups: number, clearedVotes: number, cascadedMatchups: number, deletedMatchups: number }`

**2. `undoRoundPredictive(bracketId, round)`**
- Wrapped in `prisma.$transaction` with 30-second timeout.
- Pattern follows `reopenPredictionsDAL` in `src/lib/dal/prediction.ts` but scoped to a single round rather than full bracket reopen.
- Steps inside transaction:
  1. Get matchups in the target round with `status: 'decided'` (exclude bye matchups: `isBye: false` or `isBye: { not: true }`)
  2. Clear winnerId and set status to `'pending'` on those matchups
  3. IMPORTANT: Do NOT delete from the `predictions` table. Predictions are preserved per CONTEXT.md locked decision.
  4. Clear propagated entrants in next round (same as SE: use nextMatchupId + getSlotForPosition)
  5. Cascade to downstream rounds: clear winnerId, entrant1Id, entrant2Id, set status to 'pending' for all non-bye matchups in rounds > target round
  6. Check if bracket has `predictionStatus` of 'revealing' or 'completed': if `revealedUpToRound >= round`, set `revealedUpToRound` to `round - 1` (or null if round === 1). If predictionStatus is 'completed', set it back to 'revealing' (the teacher was in the middle of reveal flow).
- Note: For vote_based predictive mode, also delete votes (same as SE). For manual mode, no votes to delete. Check bracket's `predictiveResolutionMode`: if 'vote_based', delete votes from target round and downstream matchups; if 'manual' or 'auto', skip vote deletion.
- Returns: `{ undoneMatchups: number, clearedVotes: number, cascadedMatchups: number }`

Reuse `wbRoundToLbEngineRound` (already exists in the file) and `getSlotForPosition` (already exists) for DE undo logic.
