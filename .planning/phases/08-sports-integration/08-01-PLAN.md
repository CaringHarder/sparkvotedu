  <action>
    Create `src/lib/sports/sportsdataio/client.ts` with a `SportsDataIOClient` class:
    - Constructor takes `{ apiKey: string }` (required). Base URL constants: `CBB_BASE = 'https://api.sportsdata.io/v3/cbb'` for men's, `WCBB_BASE = 'https://api.sportsdata.io/v3/wcbb'` for women's.
    - Private `get<T>(baseUrl: string, endpoint: string): Promise<T>` method that:
      - Builds URL as `${baseUrl}/json/${endpoint}`
      - Sets `Ocp-Apim-Subscription-Key` header with API key
      - Uses `next: { revalidate: 60 }` for non-live endpoints and `cache: 'no-store'` for live endpoints
      - Throws descriptive error on non-OK response
    - Public methods:
      - `getCurrentSeason(gender: SportGender): Promise<number>` -- `/CurrentSeason`
      - `areGamesInProgress(gender: SportGender): Promise<boolean>` -- `/AreAnyGamesInProgress`
      - `getSchedule(season: string, gender: SportGender): Promise<SportsDataIOGame[]>` -- `/Games/${season}` (e.g., "2026POST")
      - `getGamesByDate(date: string, gender: SportGender): Promise<SportsDataIOGame[]>` -- `/GamesByDate/${date}` (format: "2026-MAR-20")
      - `getTeams(gender: SportGender): Promise<SportsDataIOTeam[]>` -- `/TeamsBasic`
    - Each method selects the correct base URL based on gender parameter.
    - API key sourced from `process.env.SPORTSDATAIO_API_KEY` -- NEVER expose to client. This file should NOT be imported from any 'use client' component.

    Create `src/lib/sports/sportsdataio/provider.ts` implementing `SportsDataProvider`:
    - Constructor takes a `SportsDataIOClient` instance.
    - `getActiveTournaments()`: Calls `getCurrentSeason` for both men's and women's. For each, calls `getSchedule` with `{season}POST` to get tournament games. Determines tournament status from game dates and statuses. Returns `SportsTournament[]` with one entry per gender. Only returns tournaments where games exist (handles case where women's data isn't available yet).
    - `getTournamentGames(tournamentId, season)`: Parse gender from tournamentId (convention: `ncaa_mens_{season}` or `ncaa_womens_{season}`). Calls `getSchedule` + `getTeams`, builds teamsMap, maps all games using `mapGame`.
    - `getGamesByDate(date)`: Calls both men's and women's `getGamesByDate`, combines and maps results.
    - `getGameById(gameId)`: Fetches games by today's date and filters for the gameId (SportsDataIO has no single-game endpoint for CBB; alternative is to cache).
    - `areGamesInProgress()`: Calls both men's and women's `areGamesInProgress`, returns true if either is true.

    Create `src/lib/sports/provider.ts` with factory:
    ```typescript
    import { SportsDataIOClient } from './sportsdataio/client'
    import { SportsDataIOProvider } from './sportsdataio/provider'
    import type { SportsDataProvider } from './types'

    let cachedProvider: SportsDataProvider | null = null

    export function getProvider(): SportsDataProvider {
      if (cachedProvider) return cachedProvider
      const apiKey = process.env.SPORTSDATAIO_API_KEY
      if (!apiKey) throw new Error('SPORTSDATAIO_API_KEY environment variable is required')
      const client = new SportsDataIOClient({ apiKey })
      cachedProvider = new SportsDataIOProvider(client)
      return cachedProvider
    }
    ```
  </action>
