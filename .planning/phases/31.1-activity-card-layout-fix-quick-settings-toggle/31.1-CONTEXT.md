# Phase 31.1: Activity Card Layout Fix & Quick Settings Toggle - Context

**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix card title truncation at mid-width viewports across all card locations (Activities page, Brackets page, Polls page), and add an inline simple/advanced toggle that replaces the current read-only viewing mode badge on activity cards.

</domain>

<decisions>
## Implementation Decisions

### Title truncation fix
- Switch from single-line `truncate` to 2-line wrapping with `line-clamp-2` (matches student activity card pattern)
- Keep current grid breakpoints (md:2-col, lg:3-col) — the 2-line wrap handles the mid-width viewport issue
- Apply fix to all card locations: activities-list.tsx, bracket-card.tsx, poll-card.tsx — consistent everywhere
- Titles wrap to 2 lines max, then truncate with ellipsis if still too long

### Toggle placement & style
- Replace the current read-only viewing mode badge with a clickable badge in the same position
- Same visual style as current badge (teal/slate pill with Eye icon) but clickable — click toggles between Simple and Advanced
- Toggle visible on all card states (draft, active, paused, completed)
- Subtle hover state to indicate interactivity — cursor change, slight color shift — same shape as other badges

### Toggle behavior
- Toggle applies to Single Elimination brackets (viewingMode) and Predictive brackets (predictiveMode) — both already have the concept
- Round Robin pacing and polls do not get a toggle
- Instant save — click toggles and saves immediately, no confirmation dialog
- Instant live sync — student views update in real time when teacher toggles (uses existing broadcast infrastructure)
- Mid-vote safety: switching mode only changes the visual presentation, no vote data is affected

### Card information density
- Status and type badges stay read-only and unchanged — only viewing mode badge becomes interactive
- Other badges (Draft, Active, SE, DE, etc.) keep their current position and style

### Claude's Discretion
- Badge layout adjustment if 2-line titles need different badge positioning
- Card height behavior (uniform vs content-sized — whatever CSS grid naturally does)
- Overall density adjustments if the taller titles create spacing issues
- Visual feedback level on toggle (toast vs badge-change-only)

</decisions>

<specifics>
## Specific Ideas

- The clickable badge should feel like a natural extension of the existing badge pattern — teachers shouldn't need instructions to discover it
- The hover effect on the toggle badge should be subtle enough to not distract but clear enough that teachers discover the interactivity

</specifics>
