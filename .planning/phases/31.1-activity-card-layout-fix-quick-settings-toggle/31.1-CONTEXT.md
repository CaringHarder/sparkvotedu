# Phase 31.1: Activity Card Layout Fix & Quick Settings Toggle - Context

**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix card title/description truncation at mid-width viewports and all screen sizes when many activities exist. Add inline simple/advanced viewing mode toggle for brackets, and quick toggles for poll display settings (show live results, allow vote change). Toggle available on both detail/edit pages and live dashboards.

</domain>

<decisions>
## Implementation Decisions

### Card layout fix
- Both titles AND descriptions truncate, even short ones — fundamental card sizing issue, not just long titles
- Problem exists at all screen sizes when there are many brackets/polls, not just mid-width breakpoints
- Enforce minimum card width so grid drops to fewer columns rather than squeezing cards below readable size
- Descriptions: 2-line preview with ellipsis is acceptable for long descriptions
- Metadata items (badges, settings icons, participant counts) should also be investigated for squishing/misalignment

### Quick settings toggle placement
- Toggle lives on detail/edit page AND live dashboard (both pages, consistent experience)
- NOT on the activity card itself — card stays clean
- Mode badge already shows on bracket cards (verify and ensure it's working correctly)
- Brackets only for mode badge — polls don't have a viewing mode concept
- Inline on the live dashboard — no drawer or gear icon needed

### Toggle behavior & scope
- Bracket: Just viewing mode toggle (simple ↔ advanced) — other settings (seeds, vote counts, timer) stay on full settings page
- Poll: Both "show live results" and "allow vote change" are quick-togglable
- Works on all bracket/poll states (draft, active, paused, completed)
- Toggling "allow vote change" on a live poll takes effect immediately for all students
- Both pages get the same toggles: detail/edit page for pre-live config, live dashboard for mid-session changes

### Visual treatment
- Labels always visible next to toggles (not just icons with tooltips)
- Toggles blend into existing page layout — no separate "Quick Settings" section

### Claude's Discretion
- Responsive breakpoint strategy and exact number of cards per row at each width
- Whether cards maintain fixed size (page scrolls) or fill width with a floor
- Toggle control style (Radix Switch vs segmented control) — pick what fits the design system
- Visual consistency of toggle styles across brackets and polls
- Whether switching simple→advanced auto-adjusts related settings (seeds, etc.)
- Whether students see viewing mode changes in real time or on next refresh
- Confirmation UX after toggle (toast, no feedback, etc.)
- Exact placement on detail/edit page (near top vs in settings section)

</decisions>
