  <action>
    Replace the hardcoded responsive grid columns on all three card list containers:

    **Before (all 3 files):**
    ```
    grid grid-cols-1 gap-4 md:grid-cols-2 lg:grid-cols-3
    ```

    **After:**
    ```
    grid gap-4 grid-cols-[repeat(auto-fill,minmax(320px,1fr))]
    ```

    This gives the grid a 320px minimum card width. The browser auto-adjusts column count:
    - < 640px: 1 column
    - 640-959px: 2 columns
    - 960-1279px: 3 columns
    - 1280px+: 4 columns

    Files to update:
    1. `src/components/bracket/bracket-card-list.tsx` (line ~91) - bracket card grid
    2. `src/components/poll/poll-card-list.tsx` (line ~83) - poll card grid
    3. `src/app/(dashboard)/activities/activities-list.tsx` (line ~163) - combined activities grid

    **Important:** If the Tailwind v4 arbitrary value `grid-cols-[repeat(auto-fill,minmax(320px,1fr))]` does not compile correctly (cards render in single column or don't respond to width), fall back to an inline style: `style={{ gridTemplateColumns: 'repeat(auto-fill, minmax(320px, 1fr))' }}` and keep `grid gap-4` as Tailwind classes.

    Additionally, fix bracket card badge overflow:
    - In `src/components/bracket/bracket-card.tsx`, find the badges container (around line 165) which uses `flex items-center gap-1.5` WITHOUT `flex-wrap`
    - Add `flex-wrap` to the className so badges wrap to a second line on narrow cards instead of overflowing
    - Note: PollCard and ActivityItemCard already have `flex-wrap` -- only BracketCard is missing it

    Also investigate metadata items (participant counts, settings icons) for squishing/misalignment on bracket cards. If any are clipped or misaligned at narrow widths, add appropriate wrapping or min-width to prevent it.
