<feature>
  <name>Single-Elimination Bracket Engine</name>
  <files>src/lib/bracket/engine.ts, src/lib/bracket/__tests__/engine.test.ts</files>
  <behavior>
  The bracket engine provides three pure functions:

  **calculateRounds(entrantCount: number): number**
  - Input: entrant count (4, 8, or 16)
  - Output: number of rounds
  - Cases:
    - calculateRounds(4) -> 2
    - calculateRounds(8) -> 3
    - calculateRounds(16) -> 4

  **getStandardSeed(position: number, bracketSize: number, slot: 1 | 2): number**
  - Input: matchup position within first round, bracket size, which slot (1 or 2)
  - Output: seed number for that slot
  - Standard single-elimination seeding ensures top seeds face bottom seeds:
    - For size=4: Match 1 = Seed 1 vs Seed 4, Match 2 = Seed 2 vs Seed 3
    - For size=8: Match 1 = 1v8, Match 2 = 4v5, Match 3 = 2v7, Match 4 = 3v6
    - For size=16: Match 1 = 1v16, Match 2 = 8v9, Match 3 = 4v13, Match 4 = 5v12, Match 5 = 2v15, Match 6 = 7v10, Match 7 = 3v14, Match 8 = 6v11

  **generateMatchups(bracketSize: number): MatchupSeed[]**
  - Input: bracket size (4, 8, or 16)
  - Output: array of MatchupSeed objects
  - Properties:
    - Total matchups = bracketSize - 1 (e.g., 4->3, 8->7, 16->15)
    - Round 1 has bracketSize/2 matchups
    - Each subsequent round has half the matchups of the previous
    - Final round has exactly 1 matchup (the championship)
    - First-round matchups have entrant1Seed and entrant2Seed set via getStandardSeed
    - Later-round matchups have null seeds (filled by winners)
    - Every non-final matchup has nextMatchupPosition pointing to the correct next-round matchup
    - Final matchup has nextMatchupPosition = null

  The seeding algorithm uses the standard bracket pattern where:
  1. Build seed list recursively: start with [1], then for each round double the list by pairing each seed with (roundSize + 1 - seed)
  2. This produces: [1,4,2,3] for size 4, [1,8,4,5,2,7,3,6] for size 8, etc.
  3. Pair adjacent seeds for first-round matchups
  </behavior>
