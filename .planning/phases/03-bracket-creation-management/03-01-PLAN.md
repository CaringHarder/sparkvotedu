  Add three new models to prisma/schema.prisma, following the exact patterns from the research doc:

  **Bracket model:**
  - id: String @id @default(uuid())
  - name: String
  - description: String? (optional)
  - bracketType: String @default("single_elimination") @map("bracket_type")
  - size: Int (4, 8, or 16)
  - status: String @default("draft") -- values: draft, active, completed
  - teacherId: String @map("teacher_id") with @relation to Teacher
  - sessionId: String? @map("session_id") with optional @relation to ClassSession
  - createdAt: DateTime @default(now()) @map("created_at")
  - updatedAt: DateTime @updatedAt @map("updated_at")
  - Relations: entrants BracketEntrant[], matchups Matchup[]
  - Indexes: @@index([teacherId]), @@index([sessionId]), @@index([status])
  - @@map("brackets")

  **BracketEntrant model:**
  - id: String @id @default(uuid())
  - name: String
  - seedPosition: Int @map("seed_position")
  - bracketId: String @map("bracket_id") with @relation to Bracket (onDelete: Cascade)
  - createdAt: DateTime @default(now()) @map("created_at")
  - Relations: matchupsAsEntrant1 Matchup[] @relation("entrant1"), matchupsAsEntrant2 Matchup[] @relation("entrant2"), matchupsAsWinner Matchup[] @relation("winner")
  - @@unique([bracketId, seedPosition])
  - @@index([bracketId])
  - @@map("bracket_entrants")

  **Matchup model:**
  - id: String @id @default(uuid())
  - round: Int (1-indexed)
  - position: Int (1-indexed within round)
  - bracketId: String @map("bracket_id") with @relation to Bracket (onDelete: Cascade)
  - entrant1Id: String? @map("entrant1_id") with optional @relation("entrant1") to BracketEntrant
  - entrant2Id: String? @map("entrant2_id") with optional @relation("entrant2") to BracketEntrant
  - winnerId: String? @map("winner_id") with optional @relation("winner") to BracketEntrant
  - nextMatchupId: String? @map("next_matchup_id") with self-referential @relation("advancement")
  - previousMatchups: Matchup[] @relation("advancement")
  - createdAt: DateTime @default(now()) @map("created_at")
  - @@unique([bracketId, round, position])
  - @@index([bracketId])
  - @@map("matchups")

  **Also update existing models:**

  - Add `brackets Bracket[]` relation to ClassSession model

  **Create src/lib/bracket/types.ts:**

  Define the following TypeScript types for the bracket domain:

  ```typescript
  // Bracket sizes allowed in Phase 3 (single-elimination only)
  export type BracketSize = 4 | 8 | 16

  // Bracket lifecycle statuses
  export type BracketStatus = 'draft' | 'active' | 'completed'

  // Serialized bracket data (matches Prisma model shape for client use)
  export interface BracketData {
    id: string
    name: string
    description: string | null
    bracketType: string
    size: number
    status: BracketStatus
    teacherId: string
    sessionId: string | null
    createdAt: string  // ISO string for serialization
    updatedAt: string
  }

  // Serialized entrant data
  export interface BracketEntrantData {
    id: string
    name: string
    seedPosition: number
    bracketId: string
  }

  // Serialized matchup data
  export interface MatchupData {
    id: string
    round: number
    position: number
    bracketId: string
    entrant1Id: string | null
    entrant2Id: string | null
    winnerId: string | null
    nextMatchupId: string | null
    entrant1: BracketEntrantData | null
    entrant2: BracketEntrantData | null
    winner: BracketEntrantData | null
  }

  // Input type for bracket engine (before database persistence)
  export interface MatchupSeed {
    round: number
    position: number
    entrant1Seed: number | null  // seed position of entrant 1 (first round only)
    entrant2Seed: number | null  // seed position of entrant 2 (first round only)
    nextMatchupPosition: { round: number; position: number } | null
  }

  // Full bracket with relations for detail view
  export interface BracketWithDetails extends BracketData {
    entrants: BracketEntrantData[]
    matchups: MatchupData[]
  }
  ```

  **Extend src/lib/utils/validation.ts:**

  Add these Zod schemas after the existing auth schemas (do NOT remove existing schemas):

  ```typescript
  // Bracket validation schemas
  export const bracketSizeSchema = z.union([z.literal(4), z.literal(8), z.literal(16)])

  export const createBracketSchema = z.object({
    name: z.string().min(1, 'Bracket name is required').max(100, 'Bracket name must be 100 characters or less'),
    description: z.string().max(500, 'Description must be 500 characters or less').optional(),
    size: bracketSizeSchema,
    sessionId: z.string().uuid().optional(),
  })

  export const entrantSchema = z.object({
    name: z.string().min(1, 'Entrant name is required').max(100, 'Entrant name must be 100 characters or less'),
    seedPosition: z.number().int().positive(),
  })

  export const updateEntrantsSchema = z.object({
    bracketId: z.string().uuid(),
    entrants: z.array(entrantSchema).min(4, 'At least 4 entrants required').max(16, 'Maximum 16 entrants'),
  })

  export const updateBracketStatusSchema = z.object({
    bracketId: z.string().uuid(),
    status: z.enum(['draft', 'active', 'completed']),
  })

  export const deleteBracketSchema = z.object({
    bracketId: z.string().uuid(),
  })
  ```

  Also export the inferred types: `CreateBracketInput`, `EntrantInput`, `UpdateEntrantsInput`, `UpdateBracketStatusInput`, `DeleteBracketInput`.
