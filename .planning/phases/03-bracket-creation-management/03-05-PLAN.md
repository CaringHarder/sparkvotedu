<task type="auto">
  <name>Task 1: Create bracket DAL with ownership authorization</name>
  <files>src/lib/dal/bracket.ts</files>
  <action>
  Create src/lib/dal/bracket.ts following the pattern established in src/lib/dal/class-session.ts.

  **Import dependencies:**
  - `prisma` from `@/lib/prisma`
  - `generateMatchups` from `@/lib/bracket/engine`
  - Types from `@/lib/bracket/types` as needed

  **Functions to implement:**

  1. **createBracketDAL(teacherId: string, data: { name: string, description?: string, size: number, sessionId?: string }, entrants: { name: string, seedPosition: number }[])**
     - Validate entrants.length === data.size (must match bracket size)
     - Generate matchups using `generateMatchups(data.size)` from the engine
     - Use `prisma.bracket.create()` with nested writes:
       - Create bracket record
       - Create all BracketEntrant records (from entrants array)
       - Create all Matchup records (from generated matchups)
       - For first-round matchups: look up entrant IDs by seedPosition and set entrant1Id/entrant2Id
       - For later-round matchups: leave entrant1Id/entrant2Id as null
       - For each matchup: set nextMatchupId by finding the matchup in (round+1, ceil(position/2))
     - Note: This requires a transaction or multi-step creation. Use `prisma.$transaction()` if needed:
       1. Create bracket (get bracketId)
       2. Create entrants (get entrant IDs mapped by seedPosition)
       3. Create matchups with entrant references and nextMatchup references
     - Return the created bracket with entrants and matchups included

  2. **getBracketWithDetails(bracketId: string, teacherId: string)**
     - `prisma.bracket.findFirst({ where: { id: bracketId, teacherId } })` -- ownership enforced
     - Include: entrants (orderBy seedPosition asc), matchups (include entrant1, entrant2, winner; orderBy round asc, position asc)
     - Return null if not found (ownership check implicitly fails)

  3. **getTeacherBrackets(teacherId: string)**
     - `prisma.bracket.findMany({ where: { teacherId }, orderBy: { createdAt: 'desc' } })`
     - Include: `_count: { select: { entrants: true } }` for entrant count on list cards
     - Return array of brackets (without entrants/matchups for list performance)

  4. **updateBracketStatusDAL(bracketId: string, teacherId: string, status: string)**
     - First, find bracket with ownership check: `prisma.bracket.findFirst({ where: { id: bracketId, teacherId } })`
     - Validate status transition:
       - draft -> active: allowed
       - draft -> completed: allowed
       - active -> completed: allowed
       - All others: return error (cannot go backwards)
     - Additional validation for draft -> active: bracket must have entrants.length === size (fully populated)
     - Update: `prisma.bracket.update({ where: { id: bracketId }, data: { status } })`
     - Return updated bracket

  5. **updateBracketEntrantsDAL(bracketId: string, teacherId: string, entrants: { name: string, seedPosition: number }[])**
     - Find bracket with ownership check
     - Guard: if bracket.status !== 'draft', return error "Cannot edit entrants of non-draft bracket"
     - Guard: entrants.length must match bracket.size
     - In a transaction:
       1. Delete all existing matchups for this bracket
       2. Delete all existing entrants for this bracket
       3. Create new entrants from the provided array
       4. Generate new matchups using `generateMatchups(bracket.size)`
       5. Create new matchup records with correct entrant references (same logic as createBracketDAL)
     - Return updated bracket with new entrants and matchups

  6. **deleteBracketDAL(bracketId: string, teacherId: string)**
     - Find bracket with ownership check
     - `prisma.bracket.delete({ where: { id: bracketId } })` -- cascade handles entrants and matchups
     - Return { success: true }

  **Key patterns (match existing DAL style):**
  - Every function that accesses a specific bracket takes `teacherId` and filters by it
  - Return null or `{ error: string }` for failures, not exceptions
  - Use Prisma's `include` for eager loading relations
  </action>
  <verify>Run `npx tsc --noEmit` -- no type errors. All functions compile with correct Prisma types.</verify>
  <done>Bracket DAL exports all 6 functions with teacherId ownership enforcement on every operation. createBracketDAL and updateBracketEntrantsDAL generate matchups via the bracket engine. Status transitions are validated server-side.</done>
</task>

  **Server actions to implement:**

  1. **createBracket(input: unknown)**
     - Auth: `getAuthenticatedTeacher()` -- return `{ error: 'Not authenticated' }` if null
     - Validate: `createBracketSchema.safeParse(input)` -- return `{ error: 'Invalid bracket data' }` with issues on failure
     - Also validate entrants in input (if provided). The input shape for creation should include entrants:
       ```typescript
       const createBracketWithEntrantsSchema = z.object({
         bracket: createBracketSchema,
         entrants: z.array(entrantSchema),
       })
       ```

     - Validate: entrants.length must equal bracket.size
     - Call `createBracketDAL(teacher.id, parsed.data.bracket, parsed.data.entrants)`
     - On success: `revalidatePath('/brackets')` and return `{ bracket: { id, name } }`
     - On error: return `{ error: 'Failed to create bracket' }`

  2. **updateBracketStatus(input: unknown)**
     - Auth check
     - Validate with `updateBracketStatusSchema`
     - Call `updateBracketStatusDAL(parsed.data.bracketId, teacher.id, parsed.data.status)`
     - `revalidatePath('/brackets')` and `revalidatePath(`/brackets/${parsed.data.bracketId}`)`

     - `revalidatePath(`/brackets/${parsed.data.bracketId}`)`
     - Return result or error

  4. **deleteBracket(input: unknown)**
     - Auth check
     - Validate with `deleteBracketSchema`
     - Call `deleteBracketDAL(parsed.data.bracketId, teacher.id)`
     - `revalidatePath('/brackets')`
     - Return result or error

  - Error returns use `{ error: string }` shape (not thrown exceptions)

  - wrap DAL calls in try/catch to handle unexpected Prisma errors gracefully
