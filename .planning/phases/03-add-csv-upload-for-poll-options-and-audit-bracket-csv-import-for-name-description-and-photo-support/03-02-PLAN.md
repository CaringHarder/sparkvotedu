**src/components/poll/poll-csv-upload.tsx** -- New component following the bracket CSVUpload pattern closely:

Props interface:
```typescript
interface PollCSVUploadProps {
  onImportComplete: (options: Array<{ text: string; imageUrl?: string }>) => void
  maxOptions: number
  pollId?: string  // for image upload endpoint scoping, uses 'draft' fallback
}
```

States: isLoading, error, preview (ParsedPollOption[] | null), truncated (boolean), isUploading (boolean), uploadProgress ({ completed: number; total: number } | null), imageWarnings (Array<{ text: string; error: string }>)

Flow:
1. **File select**: Hidden file input (.csv), Upload CSV button triggers it. Parse with `parsePollOptionCSV()`. If empty, show error. If >maxOptions, truncate and set truncated=true. Show preview.
2. **Preview display**: List of parsed options in a scrollable max-h-48 container. Each row shows position number + option text. If entry has imageUrl, show a small Camera icon (from lucide-react) next to the text as an indicator. Show truncation warning if applicable (amber banner matching bracket pattern).
3. **Confirm ("Use These Options")**: Set isUploading=true. Filter preview items that have imageUrl. If any have images:
   - Generate temp IDs for each preview item (nanoid)
   - Call `processCSVImages()` with items mapped to `{ id, pendingImageUrl: item.imageUrl }`, upload endpoint `/api/polls/${pollId ?? 'draft'}/upload-url`, progress callback (updates uploadProgress state), warning callback (appends to imageWarnings)
   - Show progress bar/text: "Uploading images... N/M"
   - After complete, build final options array: `{ text, imageUrl: imageMap.get(id) || undefined }`
   - If imageWarnings exist, show them briefly in an amber warning box before calling onImportComplete
4. If no images in CSV, skip upload step, call onImportComplete immediately with `{ text }` objects.
5. **Cancel**: Reset all state.
6. Helper text below button: 'Upload a CSV with a "text" column. Optional "image" column for image URLs.'
